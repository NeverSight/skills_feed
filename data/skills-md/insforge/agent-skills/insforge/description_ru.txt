Создавайте приложения с помощью InsForge Backend-as-a-Service. Используйте, когда разработчикам необходимо:
(1) Настройка внутренней инфраструктуры (создание таблиц, сегментов хранения, функций развертывания, настройка аутентификации/ИИ)
(2) Интегрируйте InsForge SDK во внешние приложения (CRUD базы данных, потоки аутентификации, загрузка файлов, операции искусственного интеллекта, обмен сообщениями в реальном времени).
(3) Развертывание интерфейсных приложений на хостинге InsForge.

ВАЖНО. Перед любой работой с серверной частью вы ДОЛЖНЫ иметь URL-адрес проекта пользователя и ключ API. Если это не предусмотрено, сначала спросите пользователя.

Ключевое отличие: в конфигурации серверной части используются вызовы HTTP API к URL-адресу проекта InsForge. Интеграция клиента использует @insforge/sdk в коде приложения.
