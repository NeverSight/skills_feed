InsForge Backend-as-a-Service를 사용하여 애플리케이션을 구축하세요. 개발자가 다음을 수행해야 할 때 사용하세요.
(1) 백엔드 인프라 설정(테이블 생성, 스토리지 버킷, 기능 배포, 인증/AI 구성)
(2) InsForge SDK를 프런트엔드 애플리케이션(데이터베이스 CRUD, 인증 흐름, 파일 업로드, AI 작업, 실시간 메시징)에 통합합니다.
(3) InsForge 호스팅에 프런트엔드 애플리케이션 배포

중요: 백엔드 작업을 수행하기 전에 사용자의 프로젝트 URL과 API 키가 있어야 합니다. 제공되지 않은 경우 먼저 사용자에게 물어보세요.

주요 차이점: 백엔드 구성은 InsForge 프로젝트 URL에 대한 HTTP API 호출을 사용합니다. 클라이언트 통합은 애플리케이션 코드에서 @insforge/sdk를 사용합니다.
