أنشئ تطبيقات باستخدام InsForge Backend-as-a-Service. يُستخدم عندما يحتاج المطورون إلى:
(1) إعداد البنية الأساسية للواجهة الخلفية (إنشاء الجداول ومجموعات التخزين ونشر الوظائف وتكوين المصادقة/الذكاء الاصطناعي)
(2) دمج InsForge SDK في تطبيقات الواجهة الأمامية (قاعدة البيانات CRUD، وتدفقات المصادقة، وتحميل الملفات، وعمليات الذكاء الاصطناعي، والمراسلة في الوقت الفعلي)
(3) نشر تطبيقات الواجهة الأمامية لاستضافة InsForge

هام: قبل أي عمل في الواجهة الخلفية، يجب أن يكون لديك عنوان URL الخاص بمشروع المستخدم ومفتاح واجهة برمجة التطبيقات. إذا لم يتم تقديمه، اسأل المستخدم أولاً.

الفرق الرئيسي: يستخدم تكوين الواجهة الخلفية مكالمات HTTP API إلى عنوان URL لمشروع InsForge. يستخدم تكامل العميل @insforge/sdk في كود التطبيق.
