InsForge Backend-as-a-Service を使用してアプリケーションを構築します。開発者が次のことを行う必要がある場合に使用します。
(1) バックエンドインフラストラクチャのセットアップ (テーブルの作成、ストレージバケット、機能のデプロイ、認証/AI の構成)
(2) InsForge SDK をフロントエンド アプリケーションに統合 (データベース CRUD、認証フロー、ファイル アップロード、AI 操作、リアルタイム メッセージング)
(3) フロントエンド アプリケーションを InsForge ホスティングにデプロイする

重要: バックエンドの作業を行う前に、ユーザーのプロジェクト URL と API キーを取得する必要があります。提供されていない場合は、まずユーザーに問い合わせてください。

主な違い: バックエンド構成では、InsForge プロジェクト URL への HTTP API 呼び出しを使用します。クライアント統合は、アプリケーション コードで @insforge/sdk を使用します。
