設定和管理 Depot CI，它是 GitHub Actions 的直接替代品，完全在 Depot 內運行工作流程。在將 GitHub Actions 工作流程遷移到 Depot CI、執行「depot ci migrate」、管理 Depot CI 機密和變數、使用「depot ci run」運行工作流程、調試 Depot CI 運行、檢查工作流程相容性或了解 Depot CI 當前測試版限制時使用。當使用者提到 .depot/ 目錄、depot ci 命令或詢問有關在 Depot 的基礎設施上運行 GitHub Actions 工作流程而無需 GitHub 託管的運行器時，也可以使用。注意：Depot CI 目前處於測試階段，可用性有限。
