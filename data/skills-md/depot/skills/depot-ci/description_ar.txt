يقوم بتكوين وإدارة Depot CI، وهو بديل مباشر لـ GitHub Actions الذي يقوم بتشغيل سير العمل بالكامل داخل Depot. يُستخدم عند ترحيل مسارات عمل GitHub Actions إلى Depot CI، أو تشغيل "depot ci migrate"، أو إدارة أسرار ومتغيرات Depot CI، أو تشغيل مسارات العمل باستخدام "depot ci run"، أو تصحيح أخطاء تشغيل Depot CI، أو التحقق من توافق سير العمل، أو فهم قيود الإصدار التجريبي الحالية لـ Depot CI. يُستخدم أيضًا عندما يذكر المستخدم دليل .depot/، أو أوامر depot ci، أو يسأل عن تشغيل مهام سير عمل GitHub Actions على البنية التحتية لـ Depot بدون برامج تشغيل مستضافة على GitHub. ملاحظة: Depot CI حاليًا في مرحلة تجريبية مع توفر محدود.
