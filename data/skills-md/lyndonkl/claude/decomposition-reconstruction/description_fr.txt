À utiliser pour traiter des systèmes complexes qui nécessitent une simplification, identifier les goulots d'étranglement ou les points de défaillance critiques, repenser l'architecture ou les processus pour de meilleures performances, résoudre les problèmes qui semblent accablants, analyser les dépendances pour comprendre les effets d'entraînement, l'utilisateur mentionne "c'est trop complexe", "où est le goulot d'étranglement", "comment repenser cela", "quels sont les composants clés", ou lorsque l'optimisation nécessite de comprendre comment les pièces interagissent.
