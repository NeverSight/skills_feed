Úselo al iniciar un trabajo técnico que requiera un enfoque estructurado: escribir pruebas antes del código (TDD), planificar la exploración de datos (EDA), diseñar análisis estadísticos, aclarar los objetivos del modelado (causal versus predictivo) o validar resultados. Invocar cuando el usuario menciona "escribir pruebas para", "explorar este conjunto de datos", "analizar", "modelar", "validar" o cuando el trabajo técnico necesita un andamiaje sistemático antes de la ejecución.
