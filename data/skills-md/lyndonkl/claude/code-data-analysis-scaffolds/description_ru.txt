Используйте при начале технической работы, требующей структурированного подхода — написания тестов перед написанием кода (TDD), планирования исследования данных (EDA), разработки статистического анализа, уточнения целей моделирования (каузального или прогнозного) или проверки результатов. Вызывайте, когда пользователь упоминает «написать тесты», «изучить этот набор данных», «проанализировать», «моделировать», «проверить» или когда техническая работа требует систематической подготовки перед выполнением.
