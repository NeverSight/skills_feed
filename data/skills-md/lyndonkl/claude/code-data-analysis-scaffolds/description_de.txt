Verwenden Sie es, wenn Sie mit technischen Arbeiten beginnen, die einen strukturierten Ansatz erfordern – Tests vor dem Code schreiben (TDD), Datenexploration planen (EDA), statistische Analysen entwerfen, Modellierungsziele klären (kausal vs. prädiktiv) oder Ergebnisse validieren. Wird aufgerufen, wenn der Benutzer „Tests schreiben für“, „diesen Datensatz erkunden“, „analysieren“, „modellieren“, „validieren“ erwähnt oder wenn technische Arbeiten vor der Ausführung ein systematisches Gerüst erfordern.
