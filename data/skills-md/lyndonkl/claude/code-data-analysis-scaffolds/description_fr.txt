À utiliser lors du démarrage de travaux techniques nécessitant une approche structurée : rédaction de tests avant code (TDD), planification de l'exploration de données (EDA), conception d'analyses statistiques, clarification des objectifs de modélisation (causaux ou prédictifs) ou validation des résultats. À utiliser lorsque l'utilisateur mentionne « écrire des tests pour », « explorer cet ensemble de données », « analyser », « modéliser », « valider » ou lorsque le travail technique nécessite un échafaudage systématique avant l'exécution.
