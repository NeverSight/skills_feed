Créez un inventaire complet d’une base de code. Structure de la carte, points d'entrée, services, infrastructure, modèles de domaine et flux de données. Documentation pure – pas d’opinions ni de recommandations. À utiliser lors de l'intégration d'une base de code inconnue, de la documentation de l'architecture existante avant les modifications, de la préparation des révisions d'architecture ou de la planification de la migration, ou de la création d'une référence pour l'équipe. Déclencheurs sur des requêtes telles que « mapper cette base de code », « documenter l'architecture », « créer un inventaire » ou « que contient cette base de code ».
