Umfassendes 6-Check-Verifizierungs-Framework zur Validierung der Implementierungsqualität in Bezug auf Build, Typen, Lint, Tests, Sicherheit und Diff-Überprüfung. Diese Fähigkeit stellt sicher, dass der Code alle Qualitätstore erfüllt, bevor die Phase abgeschlossen ist. Wird bei „Implementierung überprüfen“, „Verifizierung ausführen“, „/verification-loop“ oder automatisch als Teil der Implementierungsphase Schritt 2 ausgelöst.
