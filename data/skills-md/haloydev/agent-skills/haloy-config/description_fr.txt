Créez un fichier de configuration haloy.yaml pour déployer des applications avec haloy. À utiliser lorsque l'utilisateur dit « créer une configuration haloy », « ajouter haloy.yaml », « configurer pour haloy », « configurer le déploiement de haloy » ou « préparer pour haloy ». Prend en charge les déploiements à cible unique et multi-cibles avec des bases de données auto-hébergées en option. Pas pour la création de fichiers Dockerfile (utilisez la compétence dockerize), les déploiements multi-environnements ou les configurations docker-compose/Kubernetes.
