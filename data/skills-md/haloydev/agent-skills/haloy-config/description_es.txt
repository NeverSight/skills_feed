Cree un archivo de configuración haloy.yaml para implementar aplicaciones con haloy. Úselo cuando el usuario diga "crear configuración de haloy", "agregar haloy.yaml", "configurar para haloy", "configurar la implementación de haloy" o "prepararse para haloy". Admite implementaciones de destino único y de destino múltiple con bases de datos autohospedadas opcionales. No para crear Dockerfiles (use la habilidad dockerize), implementaciones en múltiples entornos o configuraciones de docker-compose/Kubernetes.
