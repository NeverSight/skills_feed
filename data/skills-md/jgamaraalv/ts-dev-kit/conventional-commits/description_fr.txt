Rédigez, révisez et validez les messages de validation conformément à la spécification Conventional Commits v1.0.0. À utiliser lorsque : (1) créer un message de commit git pour toute modification, (2) réviser ou corriger un message de commit existant, (3) choisir le bon type de commit pour un changement, (4) décider comment marquer une modification avec rupture, (5) écrire des commits multilignes avec corps et pieds de page, ou (6) comprendre comment les commits sont mappés aux bosses SemVer (PATCH/MINOR/MAJOR). Couvre tous les types standards : feat, fix, docs, corvée, refactor, perf, test, build, ci, style, revert.
