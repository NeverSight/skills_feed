Schreiben, überprüfen und validieren Sie Commit-Nachrichten gemäß der Spezifikation „Conventional Commits v1.0.0“. Wird verwendet, wenn: (1) eine Git-Commit-Nachricht für eine Änderung erstellt wird, (2) eine vorhandene Commit-Nachricht überprüft oder korrigiert wird, (3) der richtige Commit-Typ für eine Änderung ausgewählt wird, (4) entschieden wird, wie eine wichtige Änderung markiert wird, (5) mehrzeilige Commits mit Text und Fußzeilen geschrieben werden oder (6) verstanden wird, wie Commits SemVer-Bumps (PATCH/MINOR/MAJOR) zugeordnet werden. Deckt alle Standardtypen ab: feat, fix, docs, chore, refactor, perf, test, build, ci, style, revert.
