Standardisieren Sie Anforderungs-/Funktionsänderungen in einer vorhandenen Codebasis (insbesondere Chrome-Erweiterungen), indem Sie „改需求/需求变更/调整交互/改功能/重构流程“ in eine wiederholbare Schleife umwandeln: Akzeptanzkriterien klären, aktuelles Verhalten aus dem Code bestätigen, Auswirkungen/Risiko bewerten, neue Logik entwerfen, mit kleinen Unterschieden implementieren, eine feste Schleife ausführen Regressionscheckliste und Aktualisierungsdokumente/Entscheidungsprotokoll. Verwenden Sie diese Option, wenn der Benutzer das Gefühl hat, dass der Änderungsprozess chaotisch ist, wenn sich Bearbeitungen tendenziell über mehrere Dateien erstrecken oder wenn Änderungen Manifest/Service Worker/OAuth/Speicher/UI berühren und eine zuverlässige Überprüfung und Rollback-Planung erfordern.
