Standardisez les modifications d'exigences/fonctionnalités dans une base de code existante (en particulier les extensions Chrome) en transformant "改需求/需求变更/调整交互/改功能/重构流程" en une boucle reproductible : clarifier les critères d'acceptation, confirmer le comportement actuel à partir du code, évaluer l'impact/risque, concevoir la nouvelle logique, implémenter avec de petites différences, exécuter une régression fixe liste de contrôle et mise à jour des documents/journal de décision. À utiliser lorsque l'utilisateur estime que le processus de modification est chaotique, lorsque les modifications ont tendance à s'étendre sur les fichiers, ou lorsque les modifications touchent le manifeste/le technicien de service/OAuth/le stockage/l'interface utilisateur et nécessitent une vérification fiable + une planification de restauration.
