팀 협업의 일관성과 버전 제어 모범 사례를 보장하기 위한 Git 브랜치 명명 및 워크플로 지침입니다. 브랜치 명명 형식, 커밋 표준, 풀 요청 절차 및 기본 브랜치에 대한 직접적인 변경을 방지하는 보호 장치가 포함되어 있습니다. 파일 수정, 새로운 기능, 버그 수정과 관련된 모든 개발 작업에 적용 가능합니다.

⚠️ 주요 규칙 - 준수해야 할 사항:
1. 메인 브랜치에서 직접 변경하거나 커밋하지 마세요.
2. 모든 변경 사항은 먼저 기능 분기를 생성해야 합니다(형식: <유형>/<이름>/<설명>)
3. 완료 후 Pull Request를 통해 Merge
4. 이 지침은 파일 수정, 새로운 기능 또는 버그 수정과 관련된 모든 작업에 적용됩니다.

트리거: 프로젝트 파일 변경(파일 추가, 편집, 삭제 포함)이 필요한 모든 작업은 이 지침을 따라야 합니다.
