팀 협업의 일관성과 버전 제어 모범 사례를 보장하기 위한 Git 브랜치 명명 및 워크플로 표준입니다. 브랜치 명명 형식, 커밋 규칙, 풀 요청 흐름 및 기본 브랜치에 대한 직접 변경을 방지하는 안전 메커니즘이 포함됩니다. 파일 수정, 기능 추가, 버그 수정과 관련된 모든 개발 작업에 적용됩니다.

⚠️ 중요한 규칙 – 따라야 합니다:
1. 메인 브랜치에서 직접 변경하거나 커밋하지 마세요.
2. 모든 변경은 기능 분기로 시작해야 합니다(형식: <유형>/<이름>/<설명>)
3. 완료되면 Pull Request를 통해 병합
4. 이 기술은 파일 수정, 기능 추가, 버그 수정과 관련된 모든 작업에 적용됩니다.

트리거: 모든 프로젝트 파일 작업(추가, 편집, 삭제)은 이 표준을 따라야 합니다.
