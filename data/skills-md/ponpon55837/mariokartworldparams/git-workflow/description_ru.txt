Стандарты именования ветвей Git и рабочих процессов для обеспечения согласованности командной работы и передовых методов контроля версий. Включает формат именования ветвей, соглашения о фиксации, поток запросов на извлечение и механизм безопасности, предотвращающий прямые изменения в основной ветке. Применяется ко всем задачам разработки, включая изменение файлов, добавление функций или исправление ошибок.

⚠️ Критические правила – необходимо соблюдать:
1. Никогда не вносите никаких изменений и не фиксируйте непосредственно в основной ветке.
2. Каждое изменение должно начинаться с ветки функции (формат: <тип>/<имя>/<описание>).
3. По завершении выполните объединение с помощью запроса на включение
4. Этот навык применим ко всем задачам, связанным с изменением файлов, добавлением функций или исправлением ошибок.

Триггер: любая операция с файлом проекта (добавление, редактирование, удаление) должна соответствовать этому стандарту.
