Рекомендации по именованию ветвей Git и рабочим процессам, обеспечивающие согласованность командной работы и лучшие практики контроля версий. Включает форматы именования ветвей, стандарты фиксации, процедуры запроса на включение и меры безопасности для предотвращения прямых изменений в основной ветке. Применимо ко всем задачам разработки, включая изменение файлов, новые функции и исправления ошибок.

⚠️Основные правила - необходимо соблюдать:
1. Никогда не вносите никаких изменений и не фиксируйте непосредственно в основной ветке.
2. Все изменения должны сначала создать ветвь функции (формат: <тип>/<имя>/<описание>).
3. Объединение с помощью запроса на включение после завершения
4. Это руководство применимо ко всем задачам, связанным с изменением файлов, новыми функциями или исправлением ошибок.

Триггер: Любая операция, требующая изменения файлов проекта (включая добавление, редактирование или удаление файлов), должна соответствовать этому правилу.
