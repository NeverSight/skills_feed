Git ブランチの命名とワークフローの標準により、チームのコラボレーションの一貫性とバージョン管理のベスト プラクティスを確保します。ブランチの命名形式、コミット規則、プル リクエスト フロー、メイン ブランチへの直接変更を防止する安全メカニズムが含まれます。ファイルの変更、機能の追加、バグ修正を含むすべての開発タスクに適用されます。

⚠️ 重要なルール – 従う必要があります:
1. メインブランチに直接変更やコミットを行わないでください。
2. すべての変更は機能ブランチで開始する必要があります (形式: <type>/<name>/<description>)
3. 完了したらプルリクエスト経由でマージします
4. このスキルは、ファイルの変更、機能の追加、またはバグ修正を含むすべてのタスクに適用されます。

トリガー: プロジェクト ファイルの操作 (追加、編集、削除) はすべて、この標準に従う必要があります。
