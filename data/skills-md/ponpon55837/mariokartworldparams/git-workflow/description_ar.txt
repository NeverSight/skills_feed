معايير تسمية فرع Git وسير العمل لضمان اتساق تعاون الفريق وأفضل ممارسات التحكم في الإصدار. يتضمن تنسيق تسمية الفرع، واصطلاحات الالتزام، وتدفق طلب السحب، وآلية أمان تمنع التغييرات المباشرة على الفرع الرئيسي. ينطبق على جميع مهام التطوير التي تتضمن تعديل الملف أو إضافة ميزة أو إصلاح الأخطاء.

⚠️ القواعد الحاسمة - يجب اتباعها:
1. لا تقم أبدًا بإجراء أي تغييرات أو التزامات مباشرة على الفرع الرئيسي
2. يجب أن يبدأ كل تغيير بفرع ميزة (التنسيق: <type>/<name>/<description>)
3. الدمج عبر طلب السحب عند الانتهاء
4. تنطبق هذه المهارة على جميع المهام التي تتضمن تعديل الملف أو إضافة ميزات أو إصلاح الأخطاء

المشغل: أي عملية لملف المشروع (إضافة، تحرير، حذف) يجب أن تتبع هذا المعيار.
