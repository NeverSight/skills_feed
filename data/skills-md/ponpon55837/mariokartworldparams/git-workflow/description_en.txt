Git branch naming and workflow guidelines to ensure team collaboration consistency and version control best practices. Includes branch naming formats, commit standards, Pull Request procedures, and safeguards to prevent direct changes to the main branch. Applicable to all development tasks involving file modifications, new features, and bug fixes.

⚠️ Key rules - must be followed:
1. Never make any changes or commits directly on the main branch
2. All changes must first create a feature branch (format: <type>/<name>/<description>)
3. Merge via Pull Request after completion
4. This guideline applies to all tasks involving file modifications, new features, or bug fixes

Trigger: Any operation requiring project file changes (including adding, editing, or deleting files) must follow this guideline.
