Git branch naming and workflow standards to ensure team collaboration consistency and version-control best practices. Includes branch naming format, commit conventions, Pull Request flow, and a safety mechanism preventing direct changes to the main branch. Applies to all development tasks involving file modification, feature addition, or bug fixing.

⚠️ Critical rules – must be followed:
1. Never make any changes or commits directly on the main branch
2. Every change must start with a feature branch (format: <type>/<name>/<description>)
3. Merge via Pull Request when finished
4. This skill applies to all tasks involving file modification, feature addition, or bug fixing

Trigger: any project-file operation (add, edit, delete) must follow this standard.
