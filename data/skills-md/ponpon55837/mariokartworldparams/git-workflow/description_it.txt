Denominazione dei rami Git e linee guida sul flusso di lavoro per garantire la coerenza della collaborazione del team e le migliori pratiche di controllo della versione. Include formati di denominazione dei rami, standard di commit, procedure di pull request e misure di salvaguardia per impedire modifiche dirette al ramo principale. Applicabile a tutte le attività di sviluppo che comportano modifiche di file, nuove funzionalità e correzioni di bug.

⚠️ Regole chiave - devono essere seguite:
1. Non apportare mai modifiche o effettuare commit direttamente sul ramo principale
2. Tutte le modifiche devono prima creare un ramo di funzionalità (formato: <tipo>/<nome>/<descrizione>)
3. Unisci tramite Pull Request dopo il completamento
4. Questa linea guida si applica a tutte le attività che comportano modifiche di file, nuove funzionalità o correzioni di bug

Trigger: qualsiasi operazione che richieda modifiche ai file di progetto (inclusa l'aggiunta, la modifica o l'eliminazione di file) deve seguire queste linee guida.
