Va bene分支命名與工作流程規範,確保團隊協作一致性 e 版本控制最佳實踐。包含分支命名格式、提交規範、Pull Richiesta di informazioni sul bug修復的開發任務.

⚠️ 關鍵規則 - 必須遵守：
1. 絕對不可以直接在 principale 分支上進行任何修改或提交
2. 任何變更都必須先建立功能分支（格式：<tipo>/<nome>/<descrizione>）
3. 完成後透過 Pull Request 合併
4. Come risolvere il problema dei bug?

觸發時機：任何需要修改專案檔案的操作（包括新增、編輯、刪除檔案）都必須參考此規範.
