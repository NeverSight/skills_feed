Standard di denominazione dei rami e flusso di lavoro Git per garantire la coerenza della collaborazione del team e le migliori pratiche di controllo della versione. Include formato di denominazione dei rami, convenzioni di commit, flusso di richiesta pull e un meccanismo di sicurezza che impedisce modifiche dirette al ramo principale. Si applica a tutte le attività di sviluppo che comportano la modifica di file, l'aggiunta di funzionalità o la correzione di bug.

⚠️ Regole critiche – devono essere seguite:
1. Non apportare mai modifiche o effettuare commit direttamente sul ramo principale
2. Ogni modifica deve iniziare con un ramo di funzionalità (formato: <tipo>/<nome>/<descrizione>)
3. Al termine, unisci tramite Pull Request
4. Questa competenza si applica a tutte le attività che implicano la modifica di file, l'aggiunta di funzionalità o la correzione di bug

Trigger: qualsiasi operazione sul file di progetto (aggiunta, modifica, eliminazione) deve seguire questo standard.
