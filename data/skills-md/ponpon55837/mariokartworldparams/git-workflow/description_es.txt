Pautas de flujo de trabajo y nombres de sucursales de Git para garantizar la coherencia de la colaboración en equipo y las mejores prácticas de control de versiones. Incluye formatos de nombres de sucursales, estándares de confirmación, procedimientos de solicitud de extracción y salvaguardas para evitar cambios directos en la sucursal principal. Aplicable a todas las tareas de desarrollo que implican modificaciones de archivos, nuevas funciones y corrección de errores.

⚠️ Reglas clave: se deben seguir:
1. Nunca realice cambios ni se comprometa directamente en la rama principal.
2. Todos los cambios deben crear primero una rama de función (formato: <tipo>/<nombre>/<descripción>)
3. Fusionar mediante Pull Request una vez completado
4. Esta directriz se aplica a todas las tareas que implican modificaciones de archivos, nuevas funciones o corrección de errores.

Desencadenante: cualquier operación que requiera cambios en el archivo del proyecto (incluida la adición, edición o eliminación de archivos) debe seguir esta guía.
