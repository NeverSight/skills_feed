Анализирует изменения git и итеративно генерирует семантические коммиты со структурой [Why]/[What]/[Impact]. Группирует связанные файлы, позволяет редактировать их вручную и выполняет команду git add + commit. Используйте, когда у вас есть незафиксированные изменения и вам нужны структурированные сообщения после обычных коммитов (подвиг, исправление, документация, рефакторинг и т. д.).
