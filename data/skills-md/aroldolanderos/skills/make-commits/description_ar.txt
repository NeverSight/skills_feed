يحلل تغييرات git ويولد الالتزامات الدلالية بشكل متكرر باستخدام بنية [لماذا]/[ماذا]/[التأثير]. مجموعات الملفات ذات الصلة، وتسمح بالتحرير اليدوي، وتنفذ git add + Commit. يُستخدم عندما يكون لديك تغييرات غير ملتزم بها وتحتاج إلى رسائل منظمة تتبع عمليات التنفيذ التقليدية (عمل، إصلاح، مستندات، إعادة بناء، وما إلى ذلك).
