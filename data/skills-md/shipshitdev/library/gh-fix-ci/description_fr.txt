Inspectez les vérifications GitHub PR avec gh, extrayez les journaux d'actions GitHub défaillants, résumez le contexte d'échec, puis créez un plan de correctif et implémentez-le après l'approbation de l'utilisateur. À utiliser lorsqu'un utilisateur demande de déboguer ou de corriger les échecs des vérifications PR CI/CD sur les actions GitHub et souhaite un plan ainsi que des modifications de code ; pour les vérifications externes (par exemple, Buildkite), signalez uniquement l'URL des détails et marquez-les hors de portée.
