Проверьте PR-проверки GitHub с помощью gh, извлеките журналы действий GitHub с ошибками, суммируйте контекст сбоя, затем создайте план исправления и реализуйте его после одобрения пользователя. Используйте, когда пользователь запрашивает отладку или исправление неудачных проверок PR CI/CD в действиях GitHub и хочет получить план и внести изменения в код; для внешних проверок (например, Buildkite) сообщайте только URL-адрес подробной информации и отмечайте их вне области действия.
