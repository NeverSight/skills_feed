Erstellen Sie robuste Datenaufnahmepipelines aus APIs. Verwenden Sie diese Option, wenn Sie Skripts erstellen, die paginierte Daten von externen APIs (Twitter, Börsen, beliebige REST-APIs) abrufen und den Fortschritt verfolgen, Duplikate vermeiden, Ratenbeschränkungen verwalten und sowohl inkrementelle Aktualisierungen als auch historische Backfills unterstützen müssen. Auslöser: „Daten von API aufnehmen“, „Tweets abrufen“, „Historische Daten abrufen“, „Von X synchronisieren“, „Datenpipeline erstellen“, „Ohne erneutes Herunterladen abrufen“, „Download fortsetzen“, „Ältere Daten auffüllen“. NICHT für: einfache One-Shot-API-Aufrufe, WebSocket-/Streaming-Verbindungen, Datei-Downloads oder APIs ohne Paginierung.
