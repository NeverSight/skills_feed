Cree canales de ingesta de datos resistentes a partir de API. Úselo al crear secuencias de comandos que obtienen datos paginados de API externas (Twitter, intercambios, cualquier API REST) ​​y necesitan realizar un seguimiento del progreso, evitar duplicados, manejar límites de velocidad y admitir actualizaciones incrementales y reabastecimientos históricos. Activadores: 'ingerir datos de API', 'extraer tweets', 'obtener datos históricos', 'sincronizar desde X', 'crear una canalización de datos', 'obtener sin volver a descargar', 'reanudar la descarga', 'reponer datos antiguos'. NO para: llamadas API simples de una sola vez, conexiones websocket/streaming, descargas de archivos o API sin paginación.
