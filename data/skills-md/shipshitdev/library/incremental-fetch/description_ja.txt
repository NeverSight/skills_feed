API から復元力のあるデータ取り込みパイプラインを構築します。外部 API (Twitter、取引所、任意の REST API) からページ分割されたデータをフェッチし、進行状況を追跡し、重複を回避し、レート制限を処理し、増分更新と履歴バックフィルの両方をサポートする必要があるスクリプトを作成する場合に使用します。トリガー: 「API からのデータの取り込み」、「ツイートのプル」、「履歴データの取得」、「X からの同期」、「データ パイプラインの構築」、「再ダウンロードせずに取得」、「ダウンロードの再開」、「古いデータのバックフィル」。対象外: 単純なワンショット API 呼び出し、WebSocket/ストリーミング接続、ファイルのダウンロード、ページネーションのない API。
