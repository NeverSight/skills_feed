# 报告生成技能

将收集的信息整合为结构化专业报告的工作流程。

## 适用场景

- 项目总结报告
- 调研分析报告
- 会议纪要
- 技术文档

## 报告类型

### 1. 调研报告
适用于主题研究、市场分析、竞品调研

### 2. 技术报告
适用于技术方案、架构设计、故障分析

### 3. 项目报告
适用于项目进度、里程碑总结、复盘

### 4. 会议纪要
适用于会议记录、决议跟踪

## 生成流程

### Phase 1: 信息收集
1. 明确报告目的和受众
2. 检索相关知识库文档
3. 整理对话历史中的关键信息
4. 识别信息缺口

### Phase 2: 结构设计
1. 确定报告类型和模板
2. 设计章节结构
3. 分配内容到各章节
4. 确定图表和附件

### Phase 3: 内容撰写
1. 按章节撰写内容
2. 保持风格一致性
3. 添加必要的图表和数据
4. 标注引用来源

### Phase 4: 润色完善
1. 检查逻辑连贯性
2. 校对格式和排版
3. 补充摘要和结论
4. 添加目录和索引

## 写作原则

1. **读者导向**: 始终考虑目标读者的需求
2. **结构清晰**: 使用层次分明的标题和编号
3. **数据支撑**: 关键结论要有数据佐证
4. **行动导向**: 报告要指向具体行动
5. **易于浏览**: 善用表格、列表、高亮
