Проверьте качество тестирования и проверьте покрытие тестирования для любого модуля. Этот навык следует использовать при просмотре существующих тестов, проверке пробелов в тестах, написании новых тестов или когда вас просят оценить работоспособность тестов. Он внедряет стандарты тестирования в рабочий процесс аудита для создания приоритетного отчета о пробелах. Результатом является отчет, а не код — не пишите тестовые реализации, пока отчет не будет проверен.
