Examinez la qualité des tests et la couverture des tests d’audit pour n’importe quel module. Cette compétence doit être utilisée lors de l'examen des tests existants, de l'audit des lacunes des tests, de la rédaction de nouveaux tests ou lorsqu'il est demandé d'évaluer l'état des tests. Il intègre les normes de test dans le flux de travail d'audit pour produire un rapport sur les lacunes hiérarchisé. Le résultat est un rapport, pas du code : n'écrivez pas d'implémentations de tests tant que le rapport n'a pas été examiné.
