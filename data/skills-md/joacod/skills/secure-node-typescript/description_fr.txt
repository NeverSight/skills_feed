Écrivez des applications Node.js et TypeScript sécurisées par défaut en suivant les meilleures pratiques de sécurité. À utiliser lorsque : (1) l'écriture d'un nouveau code Node.js/TypeScript, (2) la création de points de terminaison ou de middleware d'API, (3) la gestion des entrées utilisateur ou des données de formulaire, (4) la mise en œuvre de l'authentification ou de l'autorisation, (5) l'utilisation de secrets ou de variables d'environnement, (6) la configuration de configurations de projet (tsconfig, eslint), (7) l'utilisateur mentionne des problèmes de sécurité, (8) l'examen du code pour détecter les vulnérabilités, (9) l'utilisation de chemins de fichiers ou de processus enfants, (10) la configuration En-têtes HTTP ou CORS.
