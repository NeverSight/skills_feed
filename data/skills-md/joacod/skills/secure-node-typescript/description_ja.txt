セキュリティのベスト プラクティスに従って、デフォルトで安全な Node.js および TypeScript アプリケーションを作成します。次の場合に使用します: (1) 新しい Node.js/TypeScript コードを作成する、(2) API エンドポイントまたはミドルウェアを作成する、(3) ユーザー入力またはフォーム データを処理する、(4) 認証または認可を実装する、(5) シークレットまたは環境変数を操作する、(6) プロジェクト構成 (tsconfig、eslint) をセットアップする、(7) ユーザーがセキュリティ上の懸念について言及する、(8) コードの脆弱性をレビューする、(9) ファイル パスまたは子を操作するプロセス、(10) HTTP ヘッダーまたは CORS のセットアップ。
