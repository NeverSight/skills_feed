قم بكتابة تطبيقات Node.js وTypeScript الآمنة افتراضيًا باتباع أفضل ممارسات الأمان. يُستخدم عندما: (1) كتابة كود Node.js/TypeScript جديد، (2) إنشاء نقاط نهاية واجهة برمجة التطبيقات أو البرامج الوسيطة، (3) التعامل مع إدخال المستخدم أو بيانات النموذج، (4) تنفيذ المصادقة أو التفويض، (5) العمل مع الأسرار أو متغيرات البيئة، (6) إعداد تكوينات المشروع (tsconfig، eslint)، (7) ذكر المستخدم المخاوف الأمنية، (8) مراجعة التعليمات البرمجية بحثًا عن نقاط الضعف، (9) العمل مع مسارات الملفات أو العناصر الفرعية العمليات، (10) إعداد رؤوس HTTP أو CORS.
