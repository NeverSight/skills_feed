Escriba aplicaciones Node.js y TypeScript seguras por defecto siguiendo las mejores prácticas de seguridad. Úselo cuando: (1) Escribir nuevo código Node.js/TypeScript, (2) Crear endpoints API o middleware, (3) Manejar entradas de usuario o datos de formulario, (4) Implementar autenticación o autorización, (5) Trabajar con secretos o variables de entorno, (6) Configurar configuraciones de proyecto (tsconfig, eslint), (7) El usuario menciona preocupaciones de seguridad, (8) Revisar el código en busca de vulnerabilidades, (9) Trabajar con rutas de archivos o procesos secundarios, (10) Configurar Encabezados HTTP o CORS.
