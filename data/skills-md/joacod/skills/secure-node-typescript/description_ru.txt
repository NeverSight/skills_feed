Создавайте безопасные по умолчанию приложения Node.js и TypeScript, следуя рекомендациям по обеспечению безопасности. Используйте, когда: (1) Написание нового кода Node.js/TypeScript, (2) Создание конечных точек или промежуточного программного обеспечения API, (3) Обработка пользовательского ввода или данных формы, (4) Реализация аутентификации или авторизации, (5) Работа с секретами или переменными среды, (6) Настройка конфигураций проекта (tsconfig, eslint), (7) Пользователь упоминает проблемы безопасности, (8) Проверка кода на наличие уязвимостей, (9) Работа с путями к файлам или дочерними процессами, (10) Настройка заголовков HTTP или CORS.
