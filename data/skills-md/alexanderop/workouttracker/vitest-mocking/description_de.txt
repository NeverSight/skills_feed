Testen Sie Spottmuster und testen Sie Doppelstrategien. Verwenden Sie diese Option, wenn Sie Tests schreiben, die Mocks, Spione, Stubs oder Fakes benötigen. Zu den Auslösern gehören: Verspotten von Modulen, Stubbing von Globals (Abruf, Datum, Timer), Testen von Composables, Verspotten von Importen (Standard/benannt), Erstellen von Testdoppeln, Ersetzen von Abhängigkeiten, Überprüfen von Mock-Aufrufen, Bereinigen von Mocks oder jede Frage zum Thema „Wie verspotte ich X in Vitest“. Verwenden Sie es auch, wenn Sie Tests auf ordnungsgemäße Spottpraktiken überprüfen.
