Guide pour créer des pages de contenu markdown (blogs, juridiques, journaux des modifications) dans Next.js à l'aide de fumadocs-mdx, rehype-pretty-code et shiki. Couvre les collections MDX, la typographie en prose autonome, les blocs de code mis en évidence par la syntaxe et les mappages de composants qui maintiennent la cohérence visuelle des composants de code React rendus et personnalisés. Aucun cadre d'interface utilisateur de documentation requis.
