Analyser la qualité du code sur la base des principes du « Clean Code ». Identifiez les problèmes de dénomination, de taille de fonction, de duplication, de sur-ingénierie et de nombre magique grâce à des évaluations de gravité et des suggestions de refactorisation. À utiliser lorsque l'utilisateur demande une révision du code, un contrôle qualité, des conseils de refactorisation, une analyse Clean Code, une détection d'odeur de code ou mentionne des termes tels que 代码体检, 代码质量, 重构检查.
