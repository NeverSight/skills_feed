تحليل جودة الكود بناءً على مبادئ "الكود النظيف". حدد مشكلات التسمية وحجم الوظيفة والتكرار والهندسة الزائدة والأرقام السحرية من خلال تقييمات الخطورة واقتراحات إعادة البناء. يُستخدم عندما يطلب المستخدم مراجعة التعليمات البرمجية، أو فحص الجودة، أو نصيحة إعادة البناء، أو تحليل التعليمات البرمجية النظيفة، أو اكتشاف رائحة التعليمات البرمجية، أو عندما يذكر مصطلحات مثل 代码体检، 代码质量، 重构检查.
