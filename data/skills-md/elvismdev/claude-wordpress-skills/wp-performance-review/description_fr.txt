Examen du code de performance WordPress et analyse d’optimisation. À utiliser lors de l'examen du code WordPress PHP pour des problèmes de performances, de l'audit des thèmes/plugins pour l'évolutivité, de l'optimisation de WP_Query, de l'analyse des stratégies de mise en cache, de la vérification du code avant le lancement ou de la détection d'anti-modèles, ou lorsque l'utilisateur mentionne « examen des performances », « audit d'optimisation », « WordPress lent », « requêtes lentes », « trafic élevé », « mise à l'échelle de WordPress », « révision du code », « délai d'attente », « erreur 500 », « mémoire insuffisante » ou « le site ne se chargera pas ». Détecte les anti-modèles dans les requêtes de base de données, les hooks, la mise en cache d'objets, AJAX et le chargement de modèles.
