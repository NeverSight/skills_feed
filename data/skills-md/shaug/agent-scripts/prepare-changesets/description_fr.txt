Workflow spécifique à GitHub pour recomposer une grande branche source prête à être révisée en une chaîne ordonnée d'ensembles de modifications plus petits (branches empilées + PR GitHub) qui préservent le comportement et améliorent la possibilité de révision. À utiliser lorsque vous travaillez avec des piles de PR GitHub via la CLI gh pour diviser des branches monolithiques, créer des PR empilés, fusionner des ensembles de modifications examinés et propager/mettre à jour des bases de PR en aval après les fusions.
