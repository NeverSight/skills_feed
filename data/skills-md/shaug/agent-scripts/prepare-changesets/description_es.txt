Flujo de trabajo específico de GitHub para recomponer una rama fuente grande y lista para revisión en una cadena ordenada de conjuntos de cambios más pequeños (ramas apiladas + PR de GitHub) que preservan el comportamiento y mejoran la revisabilidad. Úselo cuando trabaje con pilas de relaciones públicas de GitHub a través de la CLI de gh para dividir ramas monolíticas, crear relaciones públicas apiladas, fusionar conjuntos de cambios revisados ​​y propagar/actualizar bases de relaciones públicas posteriores después de las fusiones.
