Flusso di lavoro specifico di GitHub per ricomporre un ramo di origine ampio e pronto per la revisione in una catena ordinata di set di modifiche più piccoli (rami in pila + PR GitHub) che preservano il comportamento e migliorano la revisionabilità. Da utilizzare quando si lavora con gli stack PR di GitHub tramite la CLI gh per dividere rami monolitici, creare PR in stack, unire i changeset revisionati e propagare/aggiornare le basi PR downstream dopo le unioni.
