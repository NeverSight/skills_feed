Génère des messages de validation git conformément à la spécification Conventional Commits 1.0.0 avec des types sémantiques (exploit, correctif, etc.), une portée facultative et des annotations de modification avec rupture. À utiliser lors de la validation de modifications de code ou de la création de messages de validation. Se déclenche sur des expressions telles que « commit », « commit and push », « make a commit », « git commit », ou lorsque la validation fait partie d'une action composée (par exemple, « Bump version and commit »).
