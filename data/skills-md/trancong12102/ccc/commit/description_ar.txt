ينشئ رسائل التزام git وفقًا لمواصفات الالتزامات التقليدية 1.0.0 مع الأنواع الدلالية (الفذ، والإصلاح، وما إلى ذلك)، والنطاق الاختياري، والتعليقات التوضيحية للتغيير. يُستخدم عند تنفيذ تغييرات التعليمات البرمجية أو إنشاء رسائل الالتزام. يتم تشغيل عبارات مثل "الالتزام"، أو "الالتزام والدفع"، أو "إجراء التزام"، أو "الالتزام بـ git"، أو عندما يكون الالتزام جزءًا من إجراء مركب (على سبيل المثال، "إصدار الارتطام والالتزام").
