Leitet eine strikte testgetriebene Entwicklung (TDD) unter Verwendung des Rot-Grün-Refaktor-Zyklus. Stellt sicher, dass kein Produktionscode ohne einen vorherigen fehlgeschlagenen Test geschrieben wird. Nutzen Sie diese Fähigkeit, wenn Sie neue Funktionen implementieren, Fehler beheben oder Code umgestalten, um eine hohe Testabdeckung und Designqualität sicherzustellen. Wird durch Ausdrücke wie „TDD“, „Tests zuerst schreiben“, „testgesteuert“, „Rot-Grün-Refaktor“, „Watch it fail“, „test first“ oder „behaviordriven“ ausgelöst.
