Guide le développement piloté par les tests (TDD) strict en utilisant le cycle Red-Green-Refactor. Garantit qu’aucun code de production n’est écrit sans un test préalable ayant échoué. Utilisez cette compétence lors de la mise en œuvre de nouvelles fonctionnalités, de la correction de bogues ou de la refactorisation du code pour garantir une couverture de test et une qualité de conception élevées. Se déclenche sur des expressions telles que « TDD », « écrire les tests d'abord », « piloté par les tests », « refactor rouge-vert », « regardez-le échouer », « tester d'abord » ou « piloté par le comportement ».
