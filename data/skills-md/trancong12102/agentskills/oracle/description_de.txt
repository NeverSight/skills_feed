Diese Fähigkeit sollte verwendet werden, wenn der Benutzer „das Orakel fragen“, „das Orakel verwenden“, „eine zweite Meinung einholen“, „Orakel konsultieren“ oder „tiefgreifende Analyse“ verlangt oder wenn er mit schwierigen Fehlern konfrontiert wird, kritischen Code überprüft, komplexe Refaktoren entwirft oder eine Architekturanalyse benötigt.
