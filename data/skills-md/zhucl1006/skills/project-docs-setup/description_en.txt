Project Documentation Structure Creation Assistant. As a seasoned product manager, gain deep insight into project requirements through multi-round dialogue, analyze reasonableness and provide best-practice recommendations, ultimately producing a complete technical documentation system.

**When to use:**
- New project kickoff requiring documentation system setup (recommended first step)
- Existing project lacking standard docs and needing completion
- Code exists but documentation is missing; need to backfill docs from code
- Need professional product and architecture analysis advice

**Two supported modes:**
- Mode A: New-project doc creation (from requirements to documents)
- Mode B: Code-to-docs backfill (from code to documents)

**Relationship with project-workflow:**
- This skill creates complete docs (PRD, SAD, dev guides, etc.)
- project-workflow executes dev tasks based on those docs
- Suggested flow: project-docs-setup → project-workflow

**Trigger phrases:**
- "Create project docs" / "Initialize project docs" / "Set up project docs"
- "Generate docs from code" / "Backfill project docs" / "Supplement project docs"
- Directly describe project needs, e.g. "Help me create docs for an online education platform"
