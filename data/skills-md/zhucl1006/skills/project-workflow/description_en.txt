Document-driven project development workflow. Intelligently determines task type, executes existing plans or uses feature-dev to generate new ones.

**Use cases:**
- Project documents are complete and a development task needs to be executed
- User specifies to execute a certain plan (e.g., "execute plan 001")
- User requests to develop a new feature (automatically uses feature-dev to generate a plan)

**Prerequisites:**
- The project should already have a docs/ structure (PRD, SAD, etc.)
- For new projects, it's recommended to first use the project-docs-setup skill to create complete documentation

**Trigger methods:**
- "Execute plan 001" / "continue 001-user-authentication"
- "Start development" / "Execute plan" / "Continue last task"
- "Implement user authentication feature" / "Add XXX module"
