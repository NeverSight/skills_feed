문서 기반 프로젝트 개발 워크플로. 작업 유형을 지능적으로 결정하고, 기존 계획을 실행하거나 기능 개발을 사용하여 새로운 계획을 생성합니다.

**사용 사례:**
- 프로젝트 문서가 완료되었으며 개발 작업을 실행해야 합니다.
- 사용자가 특정 계획을 실행하도록 지정합니다(예: "계획 001 실행").
- 사용자가 새로운 기능을 개발하도록 요청합니다. (feature-dev를 자동으로 사용하여 계획을 생성합니다.)

**전제조건:**
- 프로젝트에는 이미 문서/구조(PRD, SAD 등)가 있어야 합니다.
- 새 프로젝트의 경우 먼저 project-docs-setup 기술을 사용하여 완전한 문서를 작성하는 것이 좋습니다.

**트리거 방법:**
- "계획 001 실행" / "001-사용자 인증 계속"
- "개발 시작" / "계획 실행" / "마지막 작업 계속"
- "사용자 인증 기능 구현" / "XXX 모듈 추가"
