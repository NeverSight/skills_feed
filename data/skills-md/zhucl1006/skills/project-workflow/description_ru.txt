Документированный рабочий процесс разработки проекта. Интеллектуально определяет тип задачи, выполняет существующие планы или использует Feature-dev для создания новых.

**Случаи использования:**
- Проектная документация готова и необходимо выполнить задачу разработки.
- Пользователь указывает выполнение определенного плана (например, «выполнить план 001»).
- Пользовательские запросы на разработку новой функции (автоматически использует Feature-dev для создания плана)

**Предварительные требования:**
- Проект уже должен иметь документацию/структуру (PRD, SAD и т.д.)
- Для новых проектов рекомендуется сначала использовать навык настройки проектной документации для создания полной документации.

**Методы триггера:**
- «Выполнить план 001» / «продолжить 001-аутентификацию пользователя»
- «Начать разработку» / «Выполнить план» / «Продолжить последнее задание»
- «Реализовать функцию аутентификации пользователя» / «Добавить модуль XXX»
