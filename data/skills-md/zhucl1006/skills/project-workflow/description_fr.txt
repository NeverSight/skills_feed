Flux de travail de développement de projet basé sur des documents. Détermine intelligemment le type de tâche, exécute les plans existants ou utilise le développement de fonctionnalités pour en générer de nouveaux.

**Cas d'utilisation :**
- Les documents du projet sont complets et une tâche de développement doit être exécutée
- L'utilisateur précise qu'il doit exécuter un certain plan (par exemple, "exécuter le plan 001")
- Les utilisateurs demandent à développer une nouvelle fonctionnalité (utilise automatiquement feature-dev pour générer un plan)

**Prérequis :**
- Le projet doit déjà avoir une documentation/structure (PRD, SAD, etc.)
- Pour les nouveaux projets, il est recommandé d'utiliser d'abord la compétence project-docs-setup pour créer une documentation complète

**Méthodes de déclenchement :**
- "Exécuter le plan 001" / "continuer 001-authentification-utilisateur"
- "Démarrer le développement" / "Exécuter le plan" / "Continuer la dernière tâche"
- "Implémenter la fonctionnalité d'authentification des utilisateurs" / "Ajouter un module XXX"
