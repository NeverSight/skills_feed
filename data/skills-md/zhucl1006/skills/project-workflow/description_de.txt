Dokumentgesteuerter Projektentwicklungsworkflow. Bestimmt intelligent den Aufgabentyp, führt bestehende Pläne aus oder nutzt Feature-Dev, um neue zu generieren.

**Anwendungsfälle:**
- Die Projektunterlagen sind vollständig und eine Entwicklungsaufgabe muss ausgeführt werden
– Der Benutzer gibt an, einen bestimmten Plan auszuführen (z. B. „Plan 001 ausführen“).
- Benutzeranfragen zur Entwicklung einer neuen Funktion (verwendet automatisch Feature-Dev, um einen Plan zu erstellen)

**Voraussetzungen:**
- Das Projekt sollte bereits über eine Dokumentation/Struktur verfügen (PRD, SAD usw.)
- Bei neuen Projekten wird empfohlen, zunächst den Skill „project-docs-setup“ zu verwenden, um eine vollständige Dokumentation zu erstellen

**Trigger-Methoden:**
- „Plan 001 ausführen“ / „001-Benutzerauthentifizierung fortsetzen“
- „Entwicklung starten“ / „Plan ausführen“ / „Letzte Aufgabe fortsetzen“
- „Benutzerauthentifizierungsfunktion implementieren“ / „XXX-Modul hinzufügen“
