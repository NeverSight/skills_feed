Úselo al diseñar esquemas de bases de datos, necesita modelar entidades y relaciones de dominio con claridad, crear ontologías u gráficos de conocimiento, crear modelos de datos API, definir límites e invariantes del sistema, migrar entre modelos de datos, establecer taxonomías o jerarquías, el usuario menciona "esquema", "modelo de datos", "entidades", "relaciones", "ontología", "gráfico de conocimiento" o cuando las estructuras de datos dispersas/inconsistentes necesitan formalización.
