Систематическая количественная оценка технического долга и управление им с использованием методологии SQALE с приоритизацией затрат и усилий, поэтапными дорожными картами погашения и стратегиями предотвращения. Используйте, когда технический долг неизмерен или субъективен, требуется объективная расстановка приоритетов, планирование работы по рефакторингу, внедрение методов предотвращения долга или отслеживание тенденций долга с течением времени. Содержит 6 компонентов методологии (измерение с помощью индекса SQALE, категоризация с помощью таксономии запаха кода, определение приоритетов с помощью матрицы «ценность-усилия», дорожная карта поэтапной окупаемости, система отслеживания тенденций, рекомендации по предотвращению), 3 шаблона (количественная оценка на основе SQALE, картирование таксономии запаха кода, приоритизация затрат и усилий), 3 принципа (сначала высокая ценность с низкими усилиями, SQALE обеспечивает объективный базовый уровень, сложность порождает долг по ремонтопригодности). Подтверждено ускорение в 4,5 раза по сравнению с ручным подходом, 85 % переносимость между языками (Go, Python, JavaScript, Java, Rust), стандартная методология SQALE.
