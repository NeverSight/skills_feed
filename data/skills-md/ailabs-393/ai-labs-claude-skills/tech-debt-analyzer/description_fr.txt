Cette compétence doit être utilisée lors de l'analyse de la dette technique dans une base de code, de la documentation des problèmes de qualité du code, de la création de registres de dette technique ou de l'évaluation de la maintenabilité du code. Utilisez-le pour identifier les odeurs de code, les problèmes d'architecture, les problèmes de dépendance, la documentation manquante, les vulnérabilités de sécurité et pour créer une documentation technique complète sur la dette.
