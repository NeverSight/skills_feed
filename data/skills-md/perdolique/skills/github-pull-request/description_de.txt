Erstellen Sie GitHub-Pull-Requests direkt über die API aus Codeänderungen. Analysiert Branch-Commits, generiert PR-Titel und -Beschreibungen und erstellt standardmäßig PRs auf GitHub. Verwenden Sie diese Option, wenn der Benutzer PR erstellen, PR öffnen, PR erstellen, PR übermitteln, PR aktualisieren oder Pull-Anfrage, PR, Zusammenführungsanfrage, Codeüberprüfung, PR-Vorbereitung und GitHub-Workflow erwähnen möchte. Wird auch verwendet, wenn der Benutzer PR-Inhalte zum Kopieren/Ansehen generieren möchte (Fallback-Modus).
