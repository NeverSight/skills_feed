Intégrez MercadoPago Checkout Pro (basé sur la redirection) dans les applications Next.js avec n'importe quelle base de données PostgreSQL (Supabase, AWS RDS, Neon, PlanetScale, auto-hébergée, Prisma, Drizzle ou raw pg). À utiliser lorsque l'utilisateur a besoin de : (1) ajouter le traitement des paiements MercadoPago à une application Next.js, (2) créer un flux de paiement avec MercadoPago, (3) configurer des webhooks de paiement pour MercadoPago, (4) créer des pages de réussite/échec de paiement, (5) créer un panier avec intégration de paiement, (6) résoudre les problèmes d'intégration de MercadoPago (erreurs de retour automatique, échecs de webhook, incompatibilités d'hydratation, doubles soumissions). Se déclenche sur les demandes mentionnant MercadoPago, Mercado Pago, l'intégration des paiements avec MP, le traitement des paiements argentins/latino-américains ou le paiement avec MercadoPago. Prend en charge tous les pays MercadoPago : Argentine (ARS), Brésil (BRL), Mexique (MXN), Colombie (COP), Chili (CLP), Pérou (PEN), Uruguay (UYU).
