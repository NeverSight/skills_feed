Experte für Code-Design-Standards, einschließlich SOLID-Prinzipien, Clean-Code-Muster (KISS, YAGNI, DRY, TDA) und pragmatisches Software-Design. **Immer verwenden, wenn Sie Klassen/Module entwerfen, Funktionen implementieren, Fehler beheben, Code umgestalten oder Funktionen schreiben.** Verwenden Sie es proaktiv, um ordnungsgemäßes Design, Trennung von Bedenken, Einfachheit und Wartbarkeit sicherzustellen.
