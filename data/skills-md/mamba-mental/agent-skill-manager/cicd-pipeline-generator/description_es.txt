Esta habilidad se debe utilizar al crear o configurar archivos de canalización de CI/CD para pruebas, creación e implementación automatizadas. Úselo para generar flujos de trabajo de GitHub Actions, configuraciones de GitLab CI, configuraciones de CircleCI u otras configuraciones de plataforma CI/CD. Ideal para configurar canalizaciones automatizadas para aplicaciones Node.js/Next.js, incluido linting, pruebas, construcción e implementación en plataformas como Vercel, Netlify o AWS.
