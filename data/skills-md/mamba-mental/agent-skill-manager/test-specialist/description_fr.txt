Cette compétence doit être utilisée lors de la rédaction de cas de test, de la correction de bogues, de l'analyse du code pour détecter des problèmes potentiels ou de l'amélioration de la couverture des tests pour les applications JavaScript/TypeScript. Utilisez-le pour les tests unitaires, les tests d'intégration, les tests de bout en bout, le débogage des erreurs d'exécution, les bogues logiques, les problèmes de performances, les vulnérabilités de sécurité et l'analyse systématique du code.
