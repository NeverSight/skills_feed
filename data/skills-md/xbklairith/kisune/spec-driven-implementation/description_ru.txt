Используйте, когда готовы реализовать разработанные функции — разбивает проект на задачи TDD (Red-Green-Refactor), отслеживает прогресс с помощью флажков в Tasks.md, обеспечивает строгую дисциплину тестирования. Активируется, когда пользователь говорит «реализовать это», «напишем код», «начнём выполнение», упоминает «задачи», «TDD» или использует команды /dev-workflow:spec (задачи, выполнение).
