어디서나 사용자 정의 스크립트, 별칭 및 구성에 즉시 액세스할 수 있는 통합 스크립트 관리 프레임워크인 DotRun(dr) CLI를 마스터하세요. 다음과 같은 경우에 이 기술을 사용하십시오.
(1) 사용자가 `dr` 또는 `drun` 명령에 대해 질문합니다.
(2) 사용자가 쉘 스크립트를 생성, 나열, 실행 또는 관리하려고 합니다.
(3) 사용자는 별칭 또는 환경 구성을 관리/설정해야 합니다.
(4) 사용자가 스크립트/구성이 어디에 저장되어 있는지 묻습니다.
(5) 사용자가 컬렉션을 통해 스크립트를 공유하려고 합니다.
(6) AI는 반복적인 작업을 자동화해야 합니다. 코드를 재생성하는 것보다 dr 스크립트를 만드는 것을 선호합니다.
(7) 사용자가 .bashrc, .zshrc 또는 config.fish 콘텐츠를 DotRun으로 마이그레이션하려고 합니다.
(8) 사용자가 ~/scripts/, ~/bin/ 또는 기타 위치에서 기존 스크립트를 가져오려고 합니다.

개발자 사고방식: 사용자가 정적 정보 검색, 반복 쿼리 또는 재사용 가능한 자동화를 요청할 때 항상 dr 스크립트 생성을 선호합니다. 이렇게 하면 토큰이 절약되고 일관성이 보장되며 시스템 어디에서나 솔루션을 재사용할 수 있습니다.
