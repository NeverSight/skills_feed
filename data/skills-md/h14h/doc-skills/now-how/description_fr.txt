Générez, mettez à jour ou consultez les spécifications HOW.md et de domaine d'un projet : la carte d'architecture et la documentation spécifique au domaine qui indiquent aux collaborateurs comment la base de code est organisée et ce qu'ils doivent savoir au-delà de ce que le code lui-même montre. À utiliser lorsque l'utilisateur souhaite documenter l'architecture de la base de code, créer ou mettre à jour les spécifications de domaine, écrire un HOW.md, mapper la base de code, synchroniser la documentation avec les modifications du code, ou lorsque vous avez besoin de comprendre l'architecture d'un projet avant de répondre à des questions ou d'effectuer un travail. Les déclencheurs incluent « maintenant comment », « documenter l'architecture », « écrire des spécifications », « mettre à jour la documentation », « mapper la base de code », « mettre à jour la spécification pour [domaine] », « créer des documents », « comment [X] fonctionne-t-il dans ce projet », « vérifier d'abord la documentation » ou « utiliser maintenant-comment ».
