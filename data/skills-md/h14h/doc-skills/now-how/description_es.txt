Genere, actualice o consulte el HOW.md de un proyecto y las especificaciones de dominio: el mapa de arquitectura y la documentación específica del dominio que les indica a los colaboradores cómo está organizada la base del código y qué necesitan saber más allá de lo que muestra el código en sí. Úselo cuando el usuario desee documentar la arquitectura de la base de código, crear o actualizar especificaciones de dominio, escribir un HOW.md, mapear la base de código, mantener la documentación sincronizada con los cambios de código o cuando necesite comprender la arquitectura de un proyecto antes de responder preguntas o realizar un trabajo. Los activadores incluyen "ahora cómo", "documentar la arquitectura", "escribir especificaciones", "actualizar los documentos", "mapear el código base", "actualizar las especificaciones para [dominio]", "crear documentos", "cómo funciona [X] en este proyecto", "verificar los documentos primero" o "usar ahora cómo".
