Используйте метод анализа текста DRI (Data-Rule-Interaction), чтобы разбить описания требований на естественном языке по словам и выполнить моделирование предметной области. Сократите неструктурированный текст бизнес-требований до структурированной архитектурной абстракции по трем измерениям — данные (D), правило (R) и взаимодействие (I) — и напрямую создавайте концептуальные таблицы, готовые для проектирования системы. Подходит для анализа требований, извлечения предметного языка, анализа текста перед архитектурой и преобразования длинных документов с требованиями в четкую разбивку задач разработки.
