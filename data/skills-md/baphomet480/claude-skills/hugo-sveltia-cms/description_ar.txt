قم بتشغيل مواقع Hugo الجديدة باستخدام Sveltia CMS وBasecoat UI، أو قم بتحويل المواقع الحالية (أي SSG أو CMS) إلى Hugo + Sveltia CMS. استخدم هذه المهارة عندما يذكر المستخدم ترحيل Hugo، أو Sveltia CMS، أو Decap CMS، أو ترحيل TinaCMS، أو إعداد CMS للموقع الثابت، أو CMS بدون رأس لـ Hugo، أو يريد إضافة واجهة إدارة محتوى إلى موقع Hugo. يتم تشغيله أيضًا عند تحويل WordPress، أو Jekyll، أو Eleventy، أو TinaCMS، أو مواقع أخرى إلى Hugo، أو عند إعداد إدارة المحتوى المستندة إلى Git. يغطي سير العمل الكامل بدءًا من السقالات وحتى نشر صفحات Cloudflare باستخدام مصادقة GitHub OAuth.
