Progetta flussi di lavoro git che coprono strategie di ramificazione, sviluppo basato su trunk, modifiche in pila, commit convenzionali, pipeline CI/CD e igiene del repository. Da utilizzare durante l'impostazione di modelli di ramificazione, la scrittura di messaggi di commit, la configurazione di azioni GitHub, la gestione di PR in pila, la pulizia di rami obsoleti, la creazione di modelli di problemi o il recupero di commit persi.
