Réglage et optimisation des performances PostgreSQL 17/18+. Couvre la configuration des E/S asynchrones, l'investigation du plan de requête, les stratégies d'indexation, le réglage du vide automatique, l'optimisation de la recherche vectorielle, le regroupement de connexions, le partitionnement déclaratif et les modèles de requête pratiques. À utiliser pour diagnostiquer des requêtes lentes, configurer les E/S asynchrones, régler l'autovacuum, optimiser les index vectoriels, analyser les plans d'exécution avec EXPLAIN BUFFERS, configurer PgBouncer ou le pool de connexions, configurer le partitionnement de table, implémenter la pagination du curseur ou optimiser le traitement de la file d'attente.
