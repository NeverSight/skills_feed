E2E-Testarchitektur und -muster mit Playwright. Verwenden Sie es beim Entwerfen von Testsuiten, beim Strukturieren von Seitenobjektmodellen, beim Planen von CI-Sharding-Strategien, beim Einrichten von Authentifizierungsabläufen oder beim Organisieren von Tests mit Tags und Anmerkungen. Verwendung für Testarchitektur, Barrierefreiheitsprüfung mit Axe-Core, Netzwerk-Mocking-Strategien, visuelle Regressions-Workflows, HAR-Wiedergabe und StorageState-Authentifizierungsmuster. Für Details zur Playwright-API, Browser-Automatisierung oder Web-Scraping verwenden Sie stattdessen den Skill „Playwright“.
