모든 Atomemo 플러그인 개발 작업에 이 기술을 사용하세요. 여기에는 일반적인 지식과 다를 수 있는 권위 있는 CLI 명령, SDK 패턴 및 워크플로가 포함되어 있으므로 답변하기 전에 항상 문의하세요. 사용자가 Atomemo 플러그인 프로젝트 생성(atomemo 플러그인 init, bun 명령, 프로젝트 구조), 도구 플러그인 구현(ToolDefinition, 외부 API 래퍼), 모델 플러그인 정의(ModelDefinition, LLM 통합), 자격 증명 플러그인 정의(CredentialDefinition, API 키, OAuth 토큰), 선언적 매개변수 구성, 플러그인 허브 연결 디버깅 또는 마켓플레이스에 게시에 관해 질문할 때마다 호출합니다. 또한 개발자가 "플러그인"이라고 말하지 않더라도 Atomemo 사용자에게 코드나 서비스를 노출하려는 경우에도 호출합니다. VS Code, Chrome, Obsidian, Figma 또는 Raycast 플러그인을 호출하지 마십시오. npm 패키지 게시; 또는 플러그인 구축에 관한 것이 아닌 일반적인 Atomemo 워크플로 구성(스케줄링, 트리거, 노드 설정).
