Nutzen Sie diesen Skill für alle Atomemo-Plugin-Entwicklungsarbeiten – konsultieren Sie ihn immer, bevor Sie antworten, da er maßgebliche CLI-Befehle, SDK-Muster und Workflows enthält, die von Ihrem Allgemeinwissen abweichen können. Wird immer dann aufgerufen, wenn der Benutzer Folgendes fragt: Erstellen von Atomemo-Plugin-Projekten (Atomemo-Plugin-Init, Bun-Befehle, Projektstruktur), Implementieren von Tool-Plugins (ToolDefinition, externe API-Wrapper), Definieren von Modell-Plugins (ModelDefinition, LLM-Integrationen), Definieren von Anmeldeinformations-Plugins (CredentialDefinition, API-Schlüssel, OAuth-Tokens), Konfigurieren deklarativer Parameter, Debuggen von Plugin-Hub-Verbindungen oder Veröffentlichen auf dem Marktplatz. Rufen Sie es auch auf, wenn Entwickler ihren Code oder ihre Dienste Atomemo-Benutzern zugänglich machen möchten, auch wenn diese nicht „Plugin“ sagen. NICHT aufrufen für: VS Code-, Chrome-, Obsidian-, Figma- oder Raycast-Plugins; NPM-Paketveröffentlichung; oder allgemeine Atomemo-Workflow-Konfiguration (Planung, Trigger, Knoteneinrichtung), bei der es nicht um die Erstellung von Plugins geht.
