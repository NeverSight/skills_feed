Utilisez cette compétence pour tous les travaux de développement de plugins Atomemo – consultez-la toujours avant de répondre, car elle contient des commandes CLI, des modèles SDK et des flux de travail faisant autorité qui peuvent différer de vos connaissances générales. Invoquez chaque fois que l'utilisateur demande : la création de projets de plugin Atomemo (init du plugin atomemo, commandes bun, structure du projet), la mise en œuvre de plugins d'outils (ToolDefinition, wrappers d'API externes), la définition de plugins de modèle (ModelDefinition, intégrations LLM), la définition de plugins d'informations d'identification (CredentialDefinition, clés API, jetons OAuth), la configuration des paramètres déclaratifs, le débogage des connexions du hub de plugin ou la publication sur le marché. Invoquez également lorsque les développeurs souhaitent exposer leur code ou leurs services aux utilisateurs d'Atomemo, même s'ils ne disent pas « plugin ». N'invoquez PAS pour : les plugins VS Code, Chrome, Obsidian, Figma ou Raycast ; publication de packages npm ; ou une configuration générale du workflow Atomemo (planification, déclencheurs, configuration des nœuds) qui ne concerne pas la création de plugins.
