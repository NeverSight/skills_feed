Gestisci la documentazione nel sottomodulo docs/git con struttura e sincronizzazione adeguate. Da utilizzare quando (1) l'utente menziona esplicitamente "aggiungi documenti", "specifiche" o creazione di documentazione, (2) crea o legge file nella directory docs/, (3) lavora su funzionalità che necessitano di documenti di specifiche, piani di implementazione o monitoraggio dei progressi, (4) documenta soluzioni, PRD o specifiche tecniche. Gestisce l'inizializzazione del sottomodulo, la corretta struttura della directory (soluzioni/specifiche/funzionalità), la sincronizzazione dal master remoto e il commit delle modifiche.
