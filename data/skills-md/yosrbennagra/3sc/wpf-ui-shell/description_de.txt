Erstellen oder überarbeiten Sie eine .NET 8 WPF-Shell für einen Desktop-Widget-Host mithilfe von CommunityToolkit.Mvvm und einem benutzerdefinierten Viewmodel-First-Navigationsdienst. Verwenden Sie diese Option, wenn Sie an App.xaml, MainWindow/ShellWindow, freigegebenen ResourceDictionaries, hellen/dunklen Laufzeitdesigns oder dem Shell-Layout für Apps im Widget-Stil arbeiten. Vermeiden Sie Prism, Frame-Navigation und umfangreiche Regionsmanager.
