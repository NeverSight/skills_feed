استخدم هذه المهارة عندما يشعر تطبيق المستخدم بالبطء، أو أن قاعدة التعليمات البرمجية متضخمة، أو بعد أعمال تطوير كبيرة. يتم التحسين عبر أربعة أبعاد: السرعة (تحميل الصفحة، استجابة واجهة برمجة التطبيقات)، التعليمات البرمجية (الملفات غير المستخدمة، التعليمات البرمجية الميتة)، قاعدة البيانات (البيانات المعزولة، نظافة المخطط)، والتبعيات (تضخم الحزمة، حجم الحزمة).
