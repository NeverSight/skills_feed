Разработать автоматизированный инструмент для создания и ведения документации по разработке. После завершения разработки требований он автоматически создает документы требований (PRD) и документы интерфейса API; когда код обновляется, он автоматически поддерживает CHANGELOG и API CHANGELOG. Условия срабатывания: автоматически активируется, когда пользователь говорит «создать документы», «записать документы», «обновить документы» или упоминает PRD, документацию по API, журнал изменений или документы с требованиями.
