Créez des commits git de haute qualité : examinez/stadez les modifications prévues, divisez-les en commits logiques et rédigez des messages de commit clairs (en suivant les commits conventionnels, avec la prise en charge d'Emoji). Utilisez cette compétence lorsque l'utilisateur demande de valider du code, d'écrire des messages de validation, d'effectuer des modifications ou de diviser le travail en plusieurs validations. Lorsque l'utilisateur mentionne le mot-clé 【breaking change】, le message de validation doit suivre la spécification BREAKING CHANGE de cette compétence en utilisant le format de point d'exclamation. Donnez la priorité à la validation des fichiers dans la zone de préparation git ; ne considérez l’intégralité de l’arborescence de travail que si la zone de transit est vide. Lorsque l'utilisateur mentionne le mot-clé 【categorize】, suivez les spécifications de division multi-commit de cette compétence et divisez soigneusement en plusieurs validations à partir des quatre dimensions du type de fichier, du module métier, du type de modification et de la portée de la modification.
