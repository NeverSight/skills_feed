Use this skill when the user asks to commit code, write commit messages, stage changes, or split work into multiple commits. When the user mentions the keyword 【breaking change】, the commit message must follow the BREAKING CHANGE spec of this skill using the exclamation mark format. Prioritize committing files in the git staging area; only consider the entire working tree if the staging area is empty. When the user mentions the keyword 【categorize】, follow this skill’s multi-commit splitting spec and carefully split into multiple commits from the four dimensions of file type, business module, change type, and change scope.
