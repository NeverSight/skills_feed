Cree confirmaciones de git de alta calidad: revise/establezca los cambios previstos, divídalos en confirmaciones lógicas y escriba mensajes de confirmación claros (siguiendo las confirmaciones convencionales, con compatibilidad con Emoji). Utilice esta habilidad cuando el usuario solicite confirmar código, escribir mensajes de confirmación, realizar cambios en etapas o dividir el trabajo en varias confirmaciones. Cuando el usuario menciona la palabra clave 【cambio importante】, el mensaje de confirmación debe seguir la especificación de CAMBIO ÚLTIMO de esta habilidad usando el formato de signo de exclamación. Priorizar la confirmación de archivos en el área de preparación de git; Solo considere todo el árbol de trabajo si el área de preparación está vacía. Cuando el usuario mencione la palabra clave 【categorizar】, siga la especificación de división de confirmaciones múltiples de esta habilidad y divida cuidadosamente en múltiples confirmaciones de las cuatro dimensiones de tipo de archivo, módulo comercial, tipo de cambio y alcance del cambio.
