고품질 git 커밋 생성: 의도한 변경 사항을 검토/스테이징하고, 논리적 커밋으로 분할하고, 명확한 커밋 메시지를 작성합니다(기존 커밋에 따라 Emoji 지원 포함). 사용자가 코드 커밋, 커밋 메시지 작성, 변경 사항 단계 지정 또는 작업을 여러 커밋으로 분할하도록 요청할 때 이 기술을 사용하세요. 사용자가 【breaking Change】라는 키워드를 언급할 때 커밋 메시지는 느낌표 형식을 사용하여 이 스킬의 breaking change 사양을 따라야 합니다. git 준비 영역에서 파일 커밋의 우선순위를 지정하세요. 준비 영역이 비어 있는 경우에만 전체 작업 트리를 고려하십시오. 사용자가 【분류】라는 키워드를 언급하면 ​​이 스킬의 다중 커밋 분할 사양을 따르고 파일 유형, 비즈니스 모듈, 변경 유형 및 변경 범위의 4가지 차원에서 여러 커밋으로 조심스럽게 분할합니다.
