إنشاء التزامات git عالية الجودة: مراجعة/تنظيم التغييرات المقصودة، وتقسيمها إلى التزامات منطقية، وكتابة رسائل التزام واضحة (بعد الالتزامات التقليدية، مع دعم الرموز التعبيرية). استخدم هذه المهارة عندما يطلب المستخدم تنفيذ تعليمات برمجية، أو كتابة رسائل التزام، أو تغيير المرحلة، أو تقسيم العمل إلى التزامات متعددة. عندما يذكر المستخدم الكلمة الأساسية [破坏性变更]، يجب أن تستخدم رسالة الالتزام تنسيق علامة التعجب وفقًا لمواصفات التغيير العاجل لهذه المهارة. إعطاء الأولوية لملفات الالتزام في منطقة التدريج git؛ ضع في اعتبارك شجرة العمل بأكملها فقط إذا كانت منطقة التدريج فارغة. عندما يذكر المستخدم الكلمة الأساسية [分门别类]، قم بتقسيم العمل إلى التزامات متعددة بدقة وفقًا للأبعاد الأربعة لنوع الملف، ووحدة الأعمال، ونوع التغيير، وتغيير النطاق.
