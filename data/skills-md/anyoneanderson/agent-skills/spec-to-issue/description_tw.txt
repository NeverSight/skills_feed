從規格文件建立 GitHub 問題 — 根據規格自動產生結構化功能問題。

分析 .specs/{feature}/ 中的規範文檔（requirement.md、design.md、tasks.md）和
透過 gh issues create 產生結構化特徵問題。
最好與規範生成器創建的規範一起使用。

英語觸發器：
- “從規範創建問題”，“將規範註冊為問題”
- “將規範轉換為 GitHub 問題”、“將規範發佈到問題”
- 在規範生成器之後：“把這變成一個問題”

日本語：
- 「仕様書を問題にして」「問題に登入して」「規格から問題作成」
- 「仕様書から問題生成」「specを問題に変換」
- 規格產生器結束後に「これを問題にして」「問題にして」
