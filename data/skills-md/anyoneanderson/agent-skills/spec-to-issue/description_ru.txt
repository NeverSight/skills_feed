Создание проблемы GitHub на основе документов спецификации — автоматическое создание структурированных проблем с функциями на основе спецификаций.

Анализирует документы спецификаций (requirement.md, design.md, Tasks.md) в .specs/{feature}/ и
генерирует структурированную проблему с функцией с помощью gh Issue create.
Лучше всего использовать со спецификациями, созданными spec-generator.

Английские триггеры:
- «Создать задачу из спецификации», «Зарегистрировать спецификацию как проблему»
- «Преобразовать спецификацию в выпуск GitHub», «Опубликовать спецификацию в выпуск».
- После генератора спецификаций: «Превратите это в проблему»

日本語トリガー:
- 「仕様書をПроблемаにして」「Проблемаに登録して」「спецификацияからПроблема作成」
- 「仕様書からIssue生成」「specをIssueに変換」
- генератор спецификаций完了後に「これをIssueにして」「Issueにして」
