从规范文档创建 GitHub 问题 — 根据规范自动生成结构化功能问题。

分析 .specs/{feature}/ 中的规范文档（requirement.md、design.md、tasks.md）和
通过 gh issues create 生成结构化特征问题。
最好与规范生成器创建的规范一起使用。

英语触发器：
- “从规范创建问题”，“将规范注册为问题”
- “将规范转换为 GitHub 问题”、“将规范发布到问题”
- 在规范生成器之后：“把这变成一个问题”

日本语：
- 「仕様书を问题にして」「问题に登录して」「规格から问题作成」
- 「仕様书から问题生成」「specを问题に変换」
- 规格生成器结束后に「これを问题にして」「问题にして」
