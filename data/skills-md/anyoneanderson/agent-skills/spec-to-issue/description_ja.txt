仕様ドキュメントから GitHub 問題を作成 — 仕様から構造化された機能の問題を自動生成します。

.specs/{feature}/ 内の仕様ドキュメント (requirement.md、design.md、tasks.md) を分析し、
gh issue create を介して構造化された機能課題を生成します。
spec-generator によって作成されたスペックで使用するのが最適です。

英語のトリガー:
・「スペックから課題を作成」「スペックを課題として登録」
- 「仕様を GitHub 課題に変換」、「仕様を課題に公開」
- spec-generator 後: 「これを問題にします」

日本語トリガー:
- 「仕様書を発行して」「発行に登録して」「仕様から発行作成」
- 「仕様書から問題生成」「仕様を問題に変換」
- spec-generator完了後に「これを発行して」「発行して」
