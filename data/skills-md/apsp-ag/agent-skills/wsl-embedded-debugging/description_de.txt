Führen Sie eingebettete Build-/Flash-/Debug-Workflows von WSL aus, indem Sie ausführbare Windows-Dateien aufrufen, die über USB-/JTAG-Zugriff verfügen (z. B. „cargo.exe“, „probe-rs.exe“ oder „openocd.exe“). Verwenden Sie diese Option, wenn ein Agent das Firmware-Verhalten auf Hardware reproduzieren, eine MCU flashen, begrenzte Laufzeitprotokolle sammeln oder Probe-Konnektivität von einem WSL-Arbeitsbereich aus debuggen muss.
