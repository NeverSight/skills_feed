Aplicación del flujo de trabajo de desarrollo basado en pruebas para proyectos Python y React. Úselo cuando el usuario solicite TDD, desarrollo de prueba primero o metodología de refactor rojo-verde. Aplica un ciclo estricto: escriba UNA prueba fallida -> implemente el código mínimo para aprobar -> refactorice mientras esté verde -> repita. Se aplica tanto al backend (pytest) como al frontend (Biblioteca de pruebas). Cambia el comportamiento del agente para escribir pruebas antes del código. NO proporciona patrones de prueba (use pytest-patterns o react-testing-patterns para saber cómo escribir pruebas).
