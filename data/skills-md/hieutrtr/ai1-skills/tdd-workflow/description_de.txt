Testgesteuerte Durchsetzung des Entwicklungsworkflows für Python- und React-Projekte. Verwenden Sie diese Option, wenn der Benutzer TDD, Test-First-Entwicklung oder Rot-Grün-Refaktor-Methodik anfordert. Erzwingt einen strikten Zyklus: EINEN fehlgeschlagenen Test schreiben -> minimal zu bestehenden Code implementieren -> bei grünem Zustand umgestalten -> wiederholen. Gilt sowohl für Backend (pytest) als auch für Frontend (Testing Library). Ändert das Agentenverhalten, um Tests vor dem Code zu schreiben. Stellt KEINE Testmuster bereit (verwenden Sie „Pytest-Patterns“ oder „React-Testing-Patterns“, um Tests zu schreiben).
