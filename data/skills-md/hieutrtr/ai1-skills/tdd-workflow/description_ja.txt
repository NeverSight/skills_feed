Python および React プロジェクトに対するテスト駆動開発ワークフローの適用。ユーザーが TDD、テストファースト開発、またはレッド-グリーン-リファクタリング手法を要求する場合に使用します。厳密なサイクルを強制します: 失敗したテストを 1 つ作成する -> 合格する最小限のコードを実装する -> 緑色の間にリファクタリングする -> 繰り返す。バックエンド (pytest) とフロントエンド (テスト ライブラリ) の両方に適用されます。コードの前にテストを記述するようにエージェントの動作を変更します。テスト パターンは提供しません (テストの作成方法については、pytest-patterns または React-testing-patterns を使用します)。
