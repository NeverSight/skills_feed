Python 및 React 프로젝트에 대한 테스트 중심 개발 워크플로 시행. 사용자가 TDD, 테스트 우선 개발 또는 적록 리팩터링 방법론을 요청할 때 사용합니다. 엄격한 주기를 적용합니다. 하나의 실패한 테스트 작성 -> 통과할 최소 코드 구현 -> 녹색 동안 리팩터링 -> 반복. 백엔드(pytest)와 프런트엔드(테스트 라이브러리) 모두에 적용됩니다. 코드보다 먼저 테스트를 작성하도록 에이전트 동작을 변경합니다. 테스트 패턴을 제공하지 않습니다(테스트 작성 방법에 대해 pytest-patterns 또는 React-testing-patterns 사용).
