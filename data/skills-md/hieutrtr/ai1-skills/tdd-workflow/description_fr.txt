Application du workflow de développement piloté par les tests pour les projets Python et React. À utiliser lorsque l'utilisateur demande une méthodologie TDD, test-first ou refactor rouge-vert. Applique un cycle strict : écrivez UN test échoué -> implémentez le code minimum à réussir -> refactorisez en vert -> répétez. S'applique à la fois au backend (pytest) et au frontend (Testing Library). Modifie le comportement de l'agent pour écrire des tests avant le code. Ne fournit PAS de modèles de test (utilisez des modèles pytest ou des modèles de test de réaction pour savoir comment écrire des tests).
