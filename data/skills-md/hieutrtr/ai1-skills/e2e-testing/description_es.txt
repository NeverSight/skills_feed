Patrones de prueba de un extremo a otro con Playwright para aplicaciones Python/React de pila completa. Úselo al escribir pruebas E2E para flujos de trabajo de usuario completos (inicio de sesión, CRUD, navegación), pruebas de regresión de ruta crítica o validación entre navegadores. Cubre la estructura de prueba, el modelo de objetos de página, la estrategia de selección (data-testid > role > label), estrategias de espera, reutilización del estado de autenticación, gestión de datos de prueba e integración de CI. NO cubre pruebas unitarias ni pruebas de componentes (use patrones de pytest o patrones de prueba de reacción).
