Réagissez aux modèles de test de composants et de hooks avec Testing Library et Vitest. À utiliser lors de l'écriture de tests pour les composants React, les hooks personnalisés ou la logique de récupération de données. Couvre les tests de rendu des composants, la simulation d'interaction utilisateur, les tests d'état asynchrone, MSW pour la simulation d'API, les tests de hook avec renderHook, les assertions d'accessibilité et les directives de test d'instantanés. Ne couvre PAS les tests E2E (utilisez e2e-testing) ou l'application du workflow TDD (utilisez tdd-workflow).
