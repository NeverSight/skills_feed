Guide les décisions et les modèles d’architecture de code Rails 8 modernes. À utiliser pour décider où placer le code, choisir entre des modèles (objets de service, préoccupations ou objets de requête), concevoir une architecture de fonctionnalités, refactoriser pour une meilleure organisation ou lorsque l'utilisateur mentionne l'architecture, l'organisation du code, les modèles de conception ou la conception en couches.
