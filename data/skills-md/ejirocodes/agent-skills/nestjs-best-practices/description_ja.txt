TypeScript を使用したエンタープライズ Node.js アプリケーションの NestJS 11 以降のベスト プラクティス。 NestJS コントローラ、サービス、モジュール、または API を作成、レビュー、またはリファクタリングするときに使用します。トリガー対象: NestJS モジュール、コントローラー、プロバイダー、依存関係注入、@Injectable、@Controller、@Module、ミドルウェア、ガード、インターセプター、パイプ、例外フィルター、ValidationPipe、クラスバリデーター、クラストランスフォーマー、DTO、JWT 認証、パスポート戦略、@nestjs/passport、TypeORM エンティティ、Prisma クライアント、Drizzle ORM、リポジトリ パターン、循環依存関係、forwardRef、@nestjs/swagger、OpenAPI デコレーター、GraphQL リゾルバー、@nestjs/graphql、マイクロサービス、TCP トランスポート、Redis トランスポート、NATS、Kafka、NestJS 11 の重大な変更、Express v5 移行、カスタム デコレーター、ConfigService、@nestjs/config、ヘルス チェック、または NestJS テスト パターン。
