Générez une documentation vivante à partir de git diffs : analysez les comparaisons de branches ou les N derniers commits pour créer ou mettre à jour automatiquement les documents de composants, les journaux de modifications, les ADR et les Runbooks dans Markdown avec le frontmatter YAML compatible avec Obsidian. À utiliser lorsqu'on vous demande de : (1) documenter les modifications à partir d'une différence de branche, (2) générer des notes de version, (3) mettre à jour la documentation du service, (4) analyser les validations et produire des documents, (5) créer des ADR à partir de modifications architecturales. Déclencheurs : « documenter la différence », « générer des documents à partir des commits », « mettre à jour les documents pour [service] », « notes de version », « ce qui a changé et le documenter », « documents évolutifs », « analyser le diff et générer de la documentation ».
