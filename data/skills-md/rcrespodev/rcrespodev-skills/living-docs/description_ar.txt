أنشئ وثائق حية من اختلافات git - قم بتحليل مقارنات الفروع أو التزامات N الأخيرة لإنشاء أو تحديث مستندات المكونات وسجلات التغيير وADRs وكتب التشغيل في Markdown باستخدام المادة الأمامية YAML المتوافقة مع Obsidian. يُستخدم عندما يُطلب منك: (1) توثيق التغييرات من فرع فرعي، (2) إنشاء ملاحظات الإصدار، (3) تحديث وثائق الخدمة، (4) تحليل الالتزامات وإنتاج المستندات، (5) إنشاء ADRs من التغييرات الهيكلية. المشغلات: "توثيق الاختلاف"، و"إنشاء مستندات من الالتزامات"، و"تحديث المستندات لـ [الخدمة]"، و"ملاحظات الإصدار"، و"ما تغير وتوثيقه"، و"المستندات الحية"، و"تحليل الاختلاف وأنواع الوثائق".
