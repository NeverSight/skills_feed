git diff から生きているドキュメントを生成します。ブランチ比較または最後の N コミットを分析して、Obsidian 互換の YAML フロントマターを使用して Markdown でコンポーネント ドキュメント、変更ログ、ADR、Runbook を自動的に作成または更新します。 (1) ブランチの差分からの変更を文書化する、(2) リリース ノートを生成する、(3) サービス ドキュメントを更新する、(4) コミットを分析してドキュメントを生成する、(5) アーキテクチャの変更から ADR を作成する、という要求に応じて使用します。トリガー: 「差分を文書化」、「コミットから文書を生成」、「[サービス] の文書を更新」、「リリースノート」、「変更内容と文書化」、「生きた文書」、「文書化の分析」。
