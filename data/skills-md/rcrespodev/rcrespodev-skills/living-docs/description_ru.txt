Создавайте живую документацию из различий git — анализируйте сравнения ветвей или последние N коммитов, чтобы автоматически создавать или обновлять документы компонентов, журналы изменений, ADR и Runbook в Markdown с помощью совместимого с Obsidian фронтмена YAML. Используйте, когда вас просят: (1) документировать изменения из различий в ветках, (2) создавать примечания к выпуску, (3) обновлять сервисную документацию, (4) анализировать коммиты и создавать документы, (5) создавать ADR на основе архитектурных изменений. Триггеры: «задокументировать различия», «создать документы на основе коммитов», «обновить документы для [сервиса]», «примечания к выпуску», «что изменилось и задокументировать это», «действующие документы», «анализ различий и родовую документацию».
