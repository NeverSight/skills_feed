LangGraph アプリケーションの状態スキーマの設計、リデューサーの実装、永続性の構成、状態の問題のデバッグを行います。ユーザーが、(1) LangGraph グラフの状態スキーマを設計または定義する、(2) 状態蓄積のためのリデューサー関数を実装する、(3) チェックポイント作成ツール (InMemorySaver/MemorySaver、SqliteSaver、PostgresSaver) による永続性を構成する、(4) 状態更新の問題または予期しない状態の動作をデバッグする、(5) バージョン間で状態スキーマを移行する、(6) 状態スキーマ構造を検証する、(7) 次のいずれかを選択する場合に使用します。 TypedDict パターンと MessagesState パターン、(8) リスト、辞書、またはセットのカスタム リデューサーを実装する、(9) Overwrite 型を使用してリデューサーをバイパスする、(10) マルチターン会話用にスレッドベースの永続性を設定する、または (11) デバッグ用のチェックポイントを検査します。
