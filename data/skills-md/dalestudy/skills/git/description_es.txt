Guía de mejores prácticas y flujo de trabajo para el control de versiones de Git. Úselo en las siguientes situaciones: (1) Al escribir mensajes de confirmación de Git (aplicar reglas de confirmación convencionales), (2) Al crear y administrar ramas (basadas en GitHub Flow), (3) Al elegir una estrategia de creación y fusión de relaciones públicas, (4) Al limpiar el historial de Git (rebase, squash, cherry-pick), (5) Al resolver conflictos de fusión, (6) Al trabajar con palabras clave 'git', '.git', 'commit', 'branch', 'fusionar', 'rebase'
