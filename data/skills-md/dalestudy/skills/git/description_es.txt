Guía de buenas prácticas y flujo de trabajo con Git. Usar al: (1) redactar mensajes de commit (Conventional Commits), (2) crear y gestionar ramas (GitHub Flow), (3) crear PR y elegir estrategia de merge, (4) limpiar historial (rebase, squash, cherry-pick), (5) resolver conflictos de merge, (6) tareas con «git», «commit», «branch», «merge», «rebase».
