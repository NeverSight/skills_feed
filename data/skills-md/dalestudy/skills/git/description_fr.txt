Meilleures pratiques et guide de workflow pour le contrôle de version Git. À utiliser dans les situations suivantes : (1) Lors de l'écriture de messages de validation Git (appliquer les règles de validation conventionnelles), (2) Lors de la création et de la gestion de branches (basées sur GitHub Flow), (3) Lors du choix d'une stratégie de création et de fusion de relations publiques, (4) Lors du nettoyage de l'historique Git (rebase, squash, Cherry-pick), (5) Lors de la résolution de conflits de fusion, (6) Lorsque vous travaillez avec les mots-clés 'git', '.git', 'commit', 'brancher', 'fusionner', 'rebase'
