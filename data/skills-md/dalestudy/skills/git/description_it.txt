Best practice e guida al flusso di lavoro per il controllo della versione Git. Utilizzare nelle seguenti situazioni: (1) Quando si scrivono messaggi di commit Git (applicare le regole dei Commit convenzionali), (2) Quando si creano e gestiscono rami (basati su GitHub Flow), (3) Quando si sceglie una strategia di creazione e unione PR, (4) Quando si pulisce la cronologia Git (rebase, squash, cherry-pick), (5) Quando si risolvono conflitti di unione, (6) Quando si lavora con le parole chiave 'git', '.git', 'commit', 'branch', 'unire', 'ribasare'
