Git è stato creato per essere utilizzato per gestire i problemi. 다음 상황에서 사용: (1) Git 커밋 메시지 작성 시 (Commit convenzionali 규칙 적용), (2) 브랜치 생성 및 관리 시 (GitHub Flow 기반), (3) PR 생성 및 병합 전략 선택 시, (4) Git 히스토리 정리 작업 시 (rebase, squash, cherry-pick), (5) Conflitto di unione 해결 시, (6) 'git', '.git', 'commit', 'branch', 'merge', 'rebase' 키워드가 포함된 작업 시
