Best practices and workflow guide for Git version control. Use in the following situations: (1) When writing Git commit messages (apply Conventional Commits rules), (2) When creating and managing branches (based on GitHub Flow), (3) When choosing a PR creation and merge strategy, (4) When cleaning up Git history (rebase, squash, cherry-pick), (5) When resolving merge conflicts, (6) When working with keywords 'git', '.git', 'commit', 'branch', 'merge', 'rebase'
