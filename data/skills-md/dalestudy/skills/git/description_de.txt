Best Practices und Workflow-Leitfaden für die Git-Versionskontrolle. In den folgenden Situationen verwenden: (1) Beim Schreiben von Git-Commit-Nachrichten (konventionelle Commits-Regeln anwenden), (2) Beim Erstellen und Verwalten von Zweigen (basierend auf GitHub Flow), (3) Bei der Auswahl einer PR-Erstellungs- und Zusammenführungsstrategie, (4) Beim Bereinigen des Git-Verlaufs (Rebase, Squash, Cherry-Pick), (5) Beim Lösen von Zusammenführungskonflikten, (6) Beim Arbeiten mit Schlüsselwörtern „git“, „.git“, „commit“, „branch“, „merge“, 'rebase'
