Competenze per l'utilizzo del runtime Bun invece di Node.js. Utilizzare nelle seguenti situazioni: (1) quando si crea un nuovo progetto JavaScript/TypeScript, (2) quando si lavora con package.json o dipendenze, (3) quando si eseguono script o si avvia un server di sviluppo, (4) quando si esegue la migrazione di un progetto Node.js su Bun, (5) quando si lavora con attività che coinvolgono le parole chiave "bun", "bunx", "bun.lockb"
