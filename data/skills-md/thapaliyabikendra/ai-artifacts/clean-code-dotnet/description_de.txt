Für C#/.NET angepasste Clean-Code-Prinzipien, einschließlich Benennung, Variablen, Funktionen, SOLID, Fehlerbehandlung und asynchrone Muster. Wird verwendet, wenn: (1) C#-Code überprüft wird, (2) zur Verbesserung der Übersichtlichkeit umgestaltet wird, (3) neuer Code geschrieben wird, (4) Feedback zur Codeüberprüfung erfolgt.
