Guía de desarrollo de API de FHIR para crear puntos finales de atención médica. Úselo cuando: (1) cree puntos finales FHIR REST (paciente, observación, encuentro, condición, solicitud de medicación), (2) valide recursos FHIR y devuelva códigos de estado HTTP y respuestas de error adecuados, (3) implemente SMART en autorización FHIR y alcances OAuth, (4) trabaje con paquetes, transacciones, operaciones por lotes o paginación de búsqueda. Cubre las estructuras de recursos de FHIR R4, campos obligatorios, conjuntos de valores (códigos de estado, género, intención), sistemas de codificación (LOINC, SNOMED, ​​RxNorm, ICD-10) y manejo de errores de OperationOutcome.
