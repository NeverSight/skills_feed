Руководство по разработке API FHIR для создания конечных точек здравоохранения. Используйте при: (1) создании конечных точек FHIR REST (пациент, наблюдение, встреча, состояние, запрос лекарства), (2) проверке ресурсов FHIR и возврате правильных кодов состояния HTTP и ответов об ошибках, (3) реализации SMART в авторизации FHIR и областях OAuth, (4) работе с пакетами, транзакциями, пакетными операциями или поиском по страницам. Охватывает структуры ресурсов FHIR R4, обязательные поля, наборы значений (коды статуса, пол, намерение), системы кодирования (LOINC, SNOMED, ​​RxNorm, ICD-10) и обработку ошибок OperationOutcome.
