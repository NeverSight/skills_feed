ヘルスケア エンドポイントを構築するための FHIR API 開発ガイド。次の場合に使用します: (1) FHIR REST エンドポイント (Patient、Observation、Encounter、Condition、MedicationRequest) を作成する、(2) FHIR リソースを検証し、適切な HTTP ステータス コードとエラー応答を返す、(3) FHIR 認証および OAuth スコープに SMART を実装する、(4) バンドル、トランザクション、バッチ操作、または検索ページネーションを操作する。 FHIR R4 リソース構造、必須フィールド、値セット (ステータス コード、性別、意図)、コーディング システム (LOINC、SNOMED、RxNorm、ICD-10)、および OperationOutcome エラー処理について説明します。
