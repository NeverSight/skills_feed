دليل تطوير FHIR API لبناء نقاط النهاية للرعاية الصحية. يُستخدم عندما: (1) إنشاء نقاط نهاية FHIR REST (المريض، والملاحظة، واللقاء، والحالة، وطلب الدواء)، (2) التحقق من صحة موارد FHIR وإرجاع رموز حالة HTTP المناسبة واستجابات الأخطاء، (3) تنفيذ SMART على ترخيص FHIR ونطاقات OAuth، (4) العمل مع الحزم أو المعاملات أو العمليات المجمعة أو ترقيم صفحات البحث. يغطي هياكل موارد FHIR R4، والحقول المطلوبة، ومجموعات القيم (رموز الحالة، والجنس، والقصد)، وأنظمة الترميز (LOINC، وSNOMED، وRxNorm، وICD-10)، ومعالجة أخطاء OperationOutcome.
