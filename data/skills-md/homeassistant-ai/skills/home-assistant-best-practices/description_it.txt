Best practice per le automazioni, gli assistenti, gli script e i controlli dei dispositivi di Home Assistant.
ATTIVARE QUESTA ABILITÀ QUANDO: - Creare o modificare automazioni, script o scene HA - Scrivere condizioni o trigger di automazione - Creare sensori modello o sensori binari - Utilizzare wait_template o wait_for_trigger - Scegliere tra trigger del dispositivo e trigger di stato - Selezionare le modalità di automazione (singolo/riavvio/in coda/parallelo) - Configurare il pulsante Zigbee/automazioni remote (ZHA o Zigbee2MQTT) - Decidere se utilizzare un built-in sensore helper vs modello
SINTOMI CHE ATTIVANO QUESTA ABILITÀ: - L'agente utilizza per impostazione predefinita i modelli Jinja2 in cui esistono costrutti nativi - L'agente crea sensori modello per un'aggregazione semplice (somma/min/max/media) - L'agente utilizza condizioni modello anziché condizioni native state/numeric_state - L'agente utilizza wait_template anziché wait_for_trigger - L'agente utilizza device_id anziché entità_id nei trigger/azioni - L'agente sceglie la modalità di automazione errata (ad esempio, singola per le luci di movimento)
