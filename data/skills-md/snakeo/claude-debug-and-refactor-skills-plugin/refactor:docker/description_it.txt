Refactoring delle configurazioni Docker per migliorare la sicurezza, le prestazioni e la manutenibilità. Trasforma i file Dockerfile non sicuri e i file docker-compose in contenitori pronti per la produzione seguendo le best practice del 2025. Implementa build a più fasi, utenti non root, versioni di immagini bloccate, controlli di integrità, gestione dei segreti e segmentazione della rete. Risolve anti-pattern comuni come l'esecuzione come root, credenziali hardcoded, .dockerignore mancante e immagini gonfie.
