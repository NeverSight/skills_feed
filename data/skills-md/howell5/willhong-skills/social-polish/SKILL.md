# 社媒长文润色生成器

从核心观点生成可直接发布的优质社媒长文。

## 概述

这个 skill 帮助你将一个核心观点或选题转化为高质量的社媒长文。通过 4D 框架深度分析、去 AI 味处理、6 维度评审，确保输出的内容观点鲜明、情感共鸣、语言自然。

适用平台：X 长推、小红书图文、微信公众号、即刻等。

## 使用方法

```
/social-polish [你的核心观点或选题]
```

**示例：**
- `/social-polish 为什么大多数人学不会编程`
- `/social-polish 远程工作三年后我发现的真相`
- `/social-polish 年轻人为什么越来越不想结婚`

## 工作流程

### Phase 1: 4D 选题分析

收到用户的核心观点后，从四个维度深度拆解：

**分析模板：**

```
## 4D 分析结果

### 哲学维度（增加深度）
- 触及的存在性问题：[分析]
- 可用的哲学视角：[分析]

### 心理学维度（增加共鸣）
- 触发的核心情绪：[分析]
- 可激活的认知偏误：[分析]

### 传播学维度（增加可读性）
- 最佳信息组织框架：[分析]
- 注意力捕获策略：[分析]

### 社会学维度（增加传播性）
- 涉及的身份认同：[分析]
- 社会比较角度：[分析]
```

详细方法论见 @4d-framework.md

### Phase 2: 内容生成

基于 4D 分析结果生成长文：

**结构要求：**
- 开头：直接抓住注意力，不要背景铺垫
- 中间：核心论述，每段有料
- 结尾：有力收束，不说教

**风格要求：**
- 观点鲜明，敢于下判断
- 有「我」的视角和经历
- 语言口语化但不轻浮
- 长度 3000-10000 字

### Phase 3: 去 AI 味处理

对生成的内容进行去 AI 味处理：

1. **删除空洞过渡词** - 「在当今社会」「让我们共同努力」等
2. **重写模板化开头** - 不要「随着 XX 的发展」
3. **重写说教式结尾** - 不要「综上所述」「相信你一定可以」
4. **减少连接词密度** - 每 100 字不超过 2 个「然而、此外、因此」
5. **去除爹味** - 不要居高临下的说教语气
6. **控制符号使用** - 破折号全文不超过 3 处

详细规则见 @deai-patterns.md

### Phase 4: 6 维度评审

对处理后的内容进行自动评分：

| 维度 | 评分 | 评分理由 | 改进建议 |
|------|------|----------|----------|
| 观点独特性 | /10 | | |
| 情感共鸣度 | /10 | | |
| 结构清晰度 | /10 | | |
| 语言自然度 | /10 | | |
| 信息密度 | /10 | | |
| 传播潜力 | /10 | | |
| **总分** | /60 | | |

**评审规则：**
- 总分低于 54 分（平均 9 分）必须触发修订
- 修订后重新评审，直到达标
- 最多 3 轮修订

详细评分标准见 @review-dimensions.md

### Phase 5: 输出定稿

**输出格式：**

```
## 最终稿

[完整文章内容]

---

## 标题建议

1. [主标题]
2. [备选标题 1]
3. [备选标题 2]

## 评审通过

总分：XX/60
各维度得分：[列表]
```

## 关键指令

### 必须遵守

1. **先分析后写作** - 4D 分析是必须步骤，不能跳过
2. **去 AI 味是硬性要求** - 每次生成后都要检查和处理
3. **评审必须量化** - 每个维度都要给出具体分数和理由
4. **低于 9 分必须修改** - 平均分不到 9 分的稿子不能输出
5. **输出要可直接发布** - 不需要用户再做任何修改

### 禁止事项

1. **不要用模板句** - 见 @deai-patterns.md 的黑名单
2. **不要说教** - 不要「你应该」「年轻人要」
3. **不要假装客观** - 敢于表达观点和偏见
4. **不要过度铺垫** - 开头就要有料
5. **不要鸡汤结尾** - 不要「加油」「相信自己」

## 输出示例

**用户输入：**
```
/social-polish 为什么程序员应该学习写作
```

**期望输出流程：**

1. 展示 4D 分析结果
2. 生成初稿（3000-5000 字）
3. 展示去 AI 味处理的修改点
4. 展示 6 维度评审表格
5. 如不达标，展示修订内容
6. 输出最终稿和标题建议
