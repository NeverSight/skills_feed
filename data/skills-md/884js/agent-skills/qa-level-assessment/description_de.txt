Fähigkeit zur Bestimmung des QA-Testniveaus. Kombiniert Story-Beschreibungen, Code-Unterschiede und Dialoge, um das Risiko umfassend zu bewerten und die geeignete Qualitätssicherungsstufe vorzuschlagen.
In folgenden Situationen verwenden:
- Wenn Sie das QS-Level für eine PR bestimmen möchten
- Wenn Sie entscheiden müssen, ob Tests oder eine doppelte Kontrolle erforderlich sind
- Beim Festlegen des Niveaus für eine Testperspektivenüberprüfung
Kann auch mit „QA-Beurteilung“, „Teststufe“, „Risikobewertung“ usw. aufgerufen werden.
