Возможность выполнения рабочего процесса разработки через тестирование (TDD). Когда использовать: (1) при начале реализации новой функции, (2) при исправлении ошибки, (3) перед рефакторингом, (4) когда говорят «писать из теста», (5) когда требуется реализация, ориентированная на качество. Примеры триггеров: «реализовать это с помощью TDD», «на основе тестирования», «записать из теста», «КРАСНЫЙ-ЗЕЛЕНЫЙ-РЕФАКТОР», «написать тесты для UserService».
