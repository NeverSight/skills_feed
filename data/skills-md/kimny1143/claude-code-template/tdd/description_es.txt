Habilidad para ejecutar el flujo de trabajo de desarrollo basado en pruebas (TDD). Úselo cuando: (1) comience a implementar una nueva característica, (2) corrija un error, (3) antes de refactorizar, (4) le digan que "escriba la prueba primero", (5) se necesita una implementación centrada en la calidad. Ejemplos de desencadenantes: "implementar con TDD", "basado en pruebas", "escribir la prueba primero", "RED-GREEN-REFACTOR", "escribir pruebas para UserService".
