Die Fähigkeit, einen Test-Driven-Development-Workflow (TDD) auszuführen. Wann zu verwenden: (1) wenn mit der Implementierung einer neuen Funktion begonnen wird, (2) wenn ein Fehler behoben wird, (3) vor dem Refactoring, (4) wenn gesagt wird, dass „aus dem Test geschrieben werden soll“, (5) wenn eine qualitätsorientierte Implementierung erforderlich ist. Trigger-Beispiele: „Implementiere dies mit TDD“, „testgesteuert“, „schreibe aus dem Test“, „RED-GREEN-REFACTOR“, „schreibe die Tests für UserService.“
