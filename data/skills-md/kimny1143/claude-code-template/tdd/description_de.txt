Fähigkeit zur Ausführung des Test-Driven Development (TDD)-Workflows. Verwenden Sie dies, wenn: (1) mit der Implementierung einer neuen Funktion begonnen wird, (2) ein Fehler behoben wird, (3) vor dem Refactoring, (4) Sie aufgefordert werden, „zuerst den Test zu schreiben“, (5) eine qualitätsorientierte Implementierung erforderlich ist. Beispiele für Auslöser: „Mit TDD implementieren“, „Testgesteuert“, „Test zuerst schreiben“, „RED-GREEN-REFACTOR“, „Tests für UserService schreiben“.
