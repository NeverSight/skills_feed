Compétence pour exécuter le flux de travail de développement piloté par les tests (TDD). À utiliser lorsque : (1) vous commencez à implémenter une nouvelle fonctionnalité, (2) corrigez un bug, (3) avant de refactoriser, (4) il vous est demandé « d'écrire le test en premier », (5) une implémentation axée sur la qualité est nécessaire. Exemples de déclencheurs : « implémenter avec TDD », « piloté par les tests », « écrire le test en premier », « RED-GREEN-REFACTOR », « écrire des tests pour UserService ».
