La capacité d'exécuter un workflow de développement piloté par les tests (TDD). Quand l'utiliser : (1) lorsque vous commencez à implémenter une nouvelle fonctionnalité, (2) lors de la correction d'un bug, (3) avant le refactoring, (4) lorsqu'il est demandé « d'écrire à partir du test », (5) lorsqu'une implémentation axée sur la qualité est requise. Exemples de déclencheurs : « implémentez ceci avec TDD », « piloté par les tests », « écrivez à partir du test », « RED-GREEN-REFACTOR », « écrivez les tests pour UserService ».
