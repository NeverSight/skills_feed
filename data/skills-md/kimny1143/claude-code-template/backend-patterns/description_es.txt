Convenciones de código backend y colección de patrones. Patrones de implementación de rutas API, Repositorio, validación, etc.  
Cuándo usarlo: (1) Al crear una nueva ruta API (2) Al implementar un Repositorio (3) Al diseñar la validación (4) Al implementar el manejo de errores (5) Al verificar las mejores prácticas de backend.  
Ejemplos de desencadenadores: "Crear una ruta API" "Patrón en el repositorio" "Patrones de backend" "Validación con Zod" "Patrones de manejo de errores"
