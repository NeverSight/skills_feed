Когда использовать: (1) При создании нового маршрута API (2) При реализации репозитория (3) При разработке проверки (4) При реализации обработки ошибок (5) При проверке лучших практик серверной части.  
Примеры триггеров: «Создать маршрут API», «Шаблон в репозитории», «Шаблоны серверной части», «Проверка с помощью Zod», «Шаблоны обработки ошибок».
