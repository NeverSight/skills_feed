Gestione di più rami contemporaneamente utilizzando Git worktree. Quando utilizzarlo: (1) Quando si desidera sviluppare più funzionalità in parallelo (2) Quando si desidera lavorare su qualcos'altro durante la revisione di un PR (3) Quando è necessario gestire un hotfix di produzione continuando lo sviluppo (4) Quando si desidera imparare come utilizzare Worktree. Esempi di trigger: "con worktree", "simultaneamente su un altro ramo", "voglio sviluppare in parallelo", "sviluppare durante la revisione di PR", "worktree per hotfix"
