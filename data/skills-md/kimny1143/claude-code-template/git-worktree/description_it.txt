Gestione di più rami contemporaneamente utilizzando Git worktree.  
Quando usarlo:  
(1) Quando si desidera sviluppare più funzionalità in parallelo  
(2) Quando vuoi lavorare su qualcos'altro mentre rivedi un PR  
(3) Quando è necessario gestire contemporaneamente un hotfix di produzione e lo sviluppo in corso  
(4) Quando vuoi imparare come usare worktree  

Esempi di trigger:  
"con worktree", "contemporaneamente su un altro ramo", "voglio sviluppare in parallelo", "sviluppare durante la revisione di un PR", "worktree per hotfix"
