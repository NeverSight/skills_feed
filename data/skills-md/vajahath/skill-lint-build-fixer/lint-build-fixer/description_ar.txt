يُستخدم لإصلاح أخطاء بناء TypeScript ومشكلات فحص ESLint بشكل منهجي. يتم تشغيله عندما يُطلب منك "إصلاح البنية"، أو "حل أخطاء الوبر"، أو "إصلاح أخطاء النوع"، أو عند اكتشاف فشل البناء/الوبر. يتبع أولوية صارمة: 1) تحديد البرامج النصية، 2) إصلاح أخطاء TS/Build أولاً، 3) التحقق من إصلاح البناء وتنفيذه، 4) إصلاح Lint (الإصلاح التلقائي أولاً، ثم قاعدة بقاعدة)، 5) التحقق الكامل النهائي.
