Se utiliza para corregir sistemáticamente errores de compilación de TypeScript y problemas de linting de ESLint. Se activa cuando se le solicita "arreglar la compilación", "resolver errores de pelusa", "corregir errores de tipo" o cuando se detectan fallas de compilación/pelusa. Sigue una prioridad estricta: 1) Identificar scripts, 2) Corregir primero los errores de TS/compilación, 3) Verificar y confirmar la corrección de compilación, 4) Corregir Lint (primero la corrección automática, luego regla por regla), 5) Verificación completa final.
