Эксперт по навигации и управлению историей git. Охватывает поиск ошибок с помощью bisect, понимание эволюции кода, восстановление потерянной работы и безопасное переписывание истории. Понимает, что история git — это ваша машина времени. Используйте, когда упоминается «история git, git пополам, поиск момента появления ошибки, восстановление удаленного, перезапись истории, вина git, потеря фиксации».
