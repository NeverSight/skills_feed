Создавайте системы аутентификации для работников TypeScript/Cloudflare с помощью социальной аутентификации, 2FA, ключей доступа, организаций и RBAC. Самостоятельная альтернатива Clerk/Auth.js.

ВАЖНО: для D1 требуется Drizzle ORM или Kysely — прямого адаптера D1 нет. Версия 1.4.0 (ноябрь 2025 г.) добавляет сеансы без отслеживания состояния, только ESM (взлом), ротацию ключей JWT, подготовку SCIM. В версии 1.3 добавлен SSO/SAML, поддержка нескольких команд.

Используйте при: самостоятельной аутентификации на Cloudflare D1, переходе с Clerk, внедрении мультитенантного SaaS или устранении ошибок адаптера D1, сериализации сеанса, потоках OAuth, проблемах с файлами cookie TanStack Start, аннулировании сеанса nanostore.
