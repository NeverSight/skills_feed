Créez des plugins WordPress sécurisés avec des modèles de base pour les hooks, les interactions de base de données, l'API de paramètres, les types de publication personnalisés, l'API REST et AJAX. Couvre trois modèles d'architecture (Simple, OOP, PSR-4) et la Security Trinity.

À utiliser lors de la création de plugins, de l'implémentation de nonces/sanitization/escaping, de l'utilisation d'instructions préparées $wpdb ou du dépannage d'injection SQL, de vulnérabilités XSS, CSRF ou d'erreurs d'activation de plugin.
