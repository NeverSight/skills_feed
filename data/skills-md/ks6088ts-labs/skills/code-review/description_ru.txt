Навыки проведения code review. Используется, когда пользователь запрашивает проверку кода с точки зрения качества кода, безопасности, тестирования, производительности и архитектуры. Вызывается упоминанием задач проверки кода, таких как «проверка кода», «проверка кода», «проверка PR» или «посмотрите на этот код». Расставьте приоритеты и укажите на уязвимости безопасности, логические ошибки, проблемы с качеством тестирования и производительностью, а также предоставьте конкретные предложения по улучшению.
