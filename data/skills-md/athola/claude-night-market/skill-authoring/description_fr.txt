Guide pour la création efficace de compétences Claude Code en utilisant la méthodologie TDD et les principes de persuasion. À utiliser pour créer de nouvelles compétences, améliorer la conformité ou valider la qualité avant le déploiement. Ne pas utiliser pour évaluer les compétences existantes (utiliser les compétences-évaluation) ou pour analyser l'architecture (utiliser les compétences modulaires). Suivez la loi d'airain : passez un test d'échec avant d'écrire une compétence.
