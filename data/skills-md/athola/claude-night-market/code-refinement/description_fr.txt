Analysez et améliorez la qualité du code vivant : duplication, efficacité algorithmique, principes de code propre, ajustement architectural, modèles anti-slop et robustesse de la gestion des erreurs. À utiliser pour améliorer la qualité du code, réduire les erreurs d'IA, refactoriser pour plus de clarté, optimiser les algorithmes et appliquer des principes de code propre. Ne pas utiliser lors de la suppression de code mort/inutilisé (utilisez conserver:bloat-detector). recherche de bogues (utilisez pensive:bug-review). sélectionner des paradigmes d'architecture (utiliser les compétences en archétypes). Cette compétence améliore activement le code vivant, en complétant la détection des ballonnements (suppression du code mort) par un raffinement de la qualité (amélioration du code vivant).
