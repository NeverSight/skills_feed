Genere o actualice pruebas para archivos modificados en la rama actual de git, utilizando la cobertura de la declaración como métrica de evaluación (objetivo: 80%+). Úselo cuando: (1) el usuario solicita "escribir pruebas para mis cambios", "agregar pruebas para la rama actual" o "mejorar la cobertura", (2) después de implementar una función para garantizar una cobertura de prueba adecuada, (3) antes de que se pruebe un PR para verificar el código modificado. Soporta proyectos Vitest y Cargo. Invocado con /test-generator o frases como "generar pruebas", "probar mis cambios", "cubrir la diferencia".
