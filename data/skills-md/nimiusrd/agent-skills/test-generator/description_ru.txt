Создайте или обновите тесты для измененных файлов в текущей ветке git, используя покрытие операторов в качестве метрики оценки (цель: 80%+). Используйте, когда: (1) пользователь просит «написать тесты для моих изменений», «добавить тесты для текущей ветки» или «улучшить покрытие», (2) после реализации функции, чтобы обеспечить адекватное тестовое покрытие, (3) перед тестированием PR для проверки измененного кода. Поддерживает проекты Vitest и Cargo. Вызывается с помощью /test-generator или таких фраз, как «создать тесты», «проверить мои изменения», «покрыть разницу».
