ステートメント カバレッジを評価指標 (目標: 80%+) として使用して、現在の git ブランチで変更されたファイルのテストを生成または更新します。次の場合に使用します: (1) ユーザーが「変更に対するテストの作成」、「現在のブランチのテストの追加」、または「カバレッジの改善」を要求した場合、(2) 適切なテスト カバレッジを確保する機能を実装した後、(3) 変更されたコードを検証するための PR がテストされる前。 Vitest プロジェクトと Cargo プロジェクトをサポートします。 /test-generator または「テストを生成する」、「変更をテストする」、「差分をカバーする」などのフレーズで呼び出します。
