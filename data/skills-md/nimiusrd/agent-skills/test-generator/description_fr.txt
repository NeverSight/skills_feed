Générez ou mettez à jour des tests pour les fichiers modifiés dans la branche git actuelle, en utilisant la couverture des instructions comme métrique d'évaluation (cible : 80 %+). À utiliser lorsque : (1) l'utilisateur demande « d'écrire des tests pour mes modifications », « d'ajouter des tests pour la branche actuelle » ou « d'améliorer la couverture », (2) après avoir implémenté une fonctionnalité pour garantir une couverture de test adéquate, (3) avant qu'un PR pour vérifier le code modifié ne soit testé. Prend en charge les projets Vitest et Cargo. Invoqué avec /test-generator ou des expressions telles que « générer des tests », « tester mes modifications », « couvrir la différence ».
