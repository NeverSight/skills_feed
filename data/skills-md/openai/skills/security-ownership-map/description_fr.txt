Analysez les référentiels Git pour créer une topologie de propriété de sécurité (personnes vers fichier), calculez le facteur de bus et la propriété du code sensible, et exportez CSV/JSON pour les bases de données graphiques et la visualisation. Déclenchez uniquement lorsque l'utilisateur souhaite explicitement une propriété axée sur la sécurité ou une analyse des facteurs de bus fondée sur l'historique de Git (par exemple : code sensible orphelin, responsables de la sécurité, vérifications de la réalité des CODEOWNERS pour les risques, points chauds sensibles ou clusters de propriété). Ne déclenchez pas les listes générales de responsables ou les questions de propriété non liées à la sécurité.
