À utiliser lorsqu'un utilisateur demande de déboguer ou de corriger les vérifications GitHub PR défaillantes qui s'exécutent dans GitHub Actions ; utilisez « gh » pour inspecter les contrôles et les journaux, résumer le contexte de l'échec, rédiger un plan de correctif et le mettre en œuvre uniquement après approbation explicite. Considérez les fournisseurs externes (par exemple Buildkite) comme hors de portée et signalez uniquement l'URL des détails.
