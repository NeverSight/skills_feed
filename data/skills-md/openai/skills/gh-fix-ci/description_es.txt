Úselo cuando un usuario solicita depurar o corregir comprobaciones fallidas de relaciones públicas de GitHub que se ejecutan en GitHub Actions; utilice `gh` para inspeccionar comprobaciones y registros, resumir el contexto de la falla, redactar un plan de reparación e implementarlo solo después de una aprobación explícita. Trate a los proveedores externos (por ejemplo, Buildkite) como fuera de alcance e informe solo la URL de detalles.
