Schreiben Sie minimale, hilfreiche Codekommentare, die erklären, warum und nicht was, und priorisieren Sie selbstdokumentierenden Code mit klarer Struktur und Benennung. Verwenden Sie diese Fähigkeit, wenn Sie Kommentare zum Code hinzufügen, komplexe Logik dokumentieren, nicht offensichtliche Entscheidungen oder Algorithmen erklären, Funktions- oder Klassendokumentationen schreiben, Inline-Erklärungen für große Codeabschnitte erstellen, veraltete oder nicht hilfreiche Kommentare entfernen, Code so umgestalten, dass er selbsterklärender ist, oder sicherstellen, dass Kommentare dauerhaft und langfristig relevant bleiben. Verwenden Sie es in allen Programmiersprachen, wenn Sie entscheiden, ob und wie Code kommentiert werden soll, einschließlich beim Schreiben oder Überprüfen von .js-, .ts-, .py-, .java-, .go-, .rs-, .rb-, .php-, .jsx-, .tsx-, .vue-Dateien oder anderen Codedateien.
