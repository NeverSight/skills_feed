اكتب تعليقات بسيطة ومفيدة على التعليمات البرمجية تشرح السبب وليس ماذا، مع إعطاء الأولوية لتعليمات التوثيق الذاتي ذات البنية والتسمية الواضحة. استخدم هذه المهارة عند إضافة تعليقات إلى التعليمات البرمجية، أو توثيق المنطق المعقد، أو شرح القرارات أو الخوارزميات غير الواضحة، أو كتابة وثائق الوظيفة أو الفصل، أو إنشاء تفسيرات مضمنة لأقسام كبيرة من التعليمات البرمجية، أو إزالة التعليقات القديمة أو غير المفيدة، أو إعادة بناء التعليمات البرمجية لتكون أكثر وضوحًا، أو ضمان بقاء التعليقات دائمة الخضرة وذات صلة على المدى الطويل. استخدمه عبر جميع لغات البرمجة عند تحديد ما إذا كان سيتم التعليق على التعليمات البرمجية وكيفية التعليق عليها، بما في ذلك عند كتابة أو مراجعة ملفات .js أو .ts أو .py أو .java أو .go أو .rs أو .rb أو .php أو .jsx أو .tsx أو .vue أو أي ملفات تعليمات برمجية أخرى.
