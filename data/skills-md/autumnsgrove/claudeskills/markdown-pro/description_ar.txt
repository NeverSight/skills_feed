مهارة توثيق Markdown الاحترافية لإنشاء ملفات README المصقولة وسجلات التغيير وأدلة المساهمة والوثائق الفنية. يُستخدم من أجل: (1) إنشاء ملف README مع الشارات والأقسام، (2) سجل التغيير الآلي من سجل git، (3) إنشاء جدول المحتويات، (4) إرشادات المساهمة، (5) تنسيق الوثائق الفنية، (6) وثائق التعليمات البرمجية مع تمييز بناء الجملة
