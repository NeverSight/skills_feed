Politique de gestion des dépendances et de versioning NuGet pour les projets .NET. À utiliser lorsque : (1) l'ajout ou la mise à jour de packages dans des fichiers .csproj, (2) la configuration de nouvelles dépendances de projet, (3) des questions sur les versions de packages ou les politiques de stabilité, (4) la configuration des packages d'architecture de base (MediatR, FluentValidation, EF Core, Serilog, etc.), (5) l'évaluation si un package doit être inclus, ou (6) le dépannage des conflits de dépendances.
