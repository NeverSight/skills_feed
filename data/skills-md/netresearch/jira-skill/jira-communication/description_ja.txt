Python CLI スクリプトを介した Jira API 操作。ユーザーが次の操作を行うと自動的にトリガーされます。
Jira URL (https://jira.*/browse/*、https://*.atlassian.net/browse/*) について言及しています。
キー (PROJ-123) を発行するか、Jira の問題について質問します。クロードが次のことを行う必要がある場合に使用します。
(1) JQL クエリによる問題の検索、(2) 問題の詳細の取得または更新、
(3) 新しい課題の作成、(4) 課題のステータスの移行 (例: 「To Do」→「完了」)、
(5) コメントを追加する、(6) 作業時間を記録する (ワークログ)、(7) スプリントとスプリントの問題をリストする、
(8) ボードとボードの問題をリストする、(9) 問題のリンクを作成またはリストする、
(10) 利用可能な Jira フィールドを検出する、(11) ユーザー プロファイル情報を取得する、
(12) 課題から添付ファイルをダウンロードします。
認証が失敗した場合は、jira-setup.py を介して対話型の認証情報のセットアップを提供します。
自動認証検出により、Jira Cloud とサーバー/データ センターの両方をサポートします。
