Verwenden Sie FIRST für jede PR-/Code-Review-Arbeit – Überprüfen von Feedback, Lesen von CR-Kommentaren, Antworten auf Reviewer, Beantwortung von Review-Bot- oder menschlichen Kommentaren oder Vorbereiten von Commits für eine PR. Sammelt Feedback aus ALLEN Quellen (Konversation, Inline-Feedback, Bewertungen), um zu verhindern, dass Inline-Feedback häufig ausfällt. Beginnen Sie mit check-pr-feedback.sh, antworten Sie dann bei Bedarf inline und fassen Sie es mit einem Fixbericht zusammen.
