Flux standard de n'importe quelle source de tâche (lien ou description) vers la création d'un PR : résoudre la tâche, créer une branche et TODO.md, attendre les correctifs, créer un PR par rapport à l'origine uniquement. Utiliser les compétences de recherche pour découvrir les méthodes de requête de source de données ; après confirmation, enregistrez dans la configuration globale afin que l'étape de découverte puisse être ignorée la prochaine fois. À utiliser lorsque l'utilisateur fournit un lien/une description de tâche, demande de « suivre le flux de travail GitHub » ou de « créer un PR à partir d'une tâche ».
