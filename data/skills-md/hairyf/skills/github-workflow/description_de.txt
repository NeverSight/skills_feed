Standardablauf von einer beliebigen Aufgabenquelle (Link oder Beschreibung) zum Erstellen eines PR: Aufgabe auflösen, Zweig und TODO.md erstellen, auf Korrekturen warten, PR nur für den Ursprung erstellen. Verwenden Sie Find-Skills, um Abfragemethoden für Datenquellen zu entdecken. Nach der Bestätigung in der globalen Konfiguration speichern, damit der Erkennungsschritt beim nächsten Mal übersprungen werden kann. Verwenden Sie diese Option, wenn der Benutzer einen Aufgabenlink/eine Aufgabenbeschreibung bereitstellt, auffordert, „dem GitHub-Workflow zu folgen“ oder „PR aus der Aufgabe zu erstellen“.
