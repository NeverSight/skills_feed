Flujo estándar desde cualquier origen de tarea (enlace o descripción) para crear un PR: resolver tarea, crear rama y TODO.md, esperar correcciones, crear PR solo contra el origen. Utilice habilidades de búsqueda para descubrir métodos de consulta de fuentes de datos; Después de la confirmación, guárdelo en la configuración global para poder omitir el paso de descubrimiento la próxima vez. Úselo cuando el usuario proporciona un enlace o una descripción de la tarea, solicita "seguir el flujo de trabajo de GitHub" o "crear PR a partir de la tarea".
