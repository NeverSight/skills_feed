Стандартный путь от любого источника задачи (ссылка или описание) к созданию заявки на выполнение: решить задачу, создать ветку и TODO.md, дождаться исправлений, создать заявку только на источник. Используйте навыки поиска для обнаружения методов запроса источника данных; после подтверждения сохраните в глобальную конфигурацию, чтобы в следующий раз шаг обнаружения можно было пропустить. Используйте, когда пользователь предоставляет ссылку/описание задачи, просит «следовать рабочему процессу GitHub» или «создать PR на основе задачи».
