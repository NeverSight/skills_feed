Erstellen oder aktualisieren Sie ein GitHub-Issue mit klärenden Fragen. TRIGGER, wenn: Der Benutzer bittet darum, ein GitHub-Problem zu erstellen, zu aktualisieren, zu bearbeiten oder zu verfeinern. Beispiele: „Ein Problem erstellen für …“, „Problem 42 aktualisieren“, „Beschreibung für Problem 137 verbessern“. NICHT AUSLÖSEN, wenn: Der Benutzer lediglich Probleme anzeigen/auflisten, eine PR kommentieren oder eine Funktion implementieren möchte.
