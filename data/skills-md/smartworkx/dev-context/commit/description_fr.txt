Organisez et validez les modifications avec un message de validation bien conçu. DÉCLENCHEMENT lorsque : l'utilisateur demande de valider, de transférer ou d'enregistrer les modifications dans git. Exemples : « valider ceci », « valider les modifications », « préparer et valider ». NE PAS DÉCLENCHER lorsque : l'utilisateur demande de pousser, de créer un PR ou de modifier un commit précédent.
