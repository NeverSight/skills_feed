Führen Sie eine strukturierte Überprüfung durch, die sich auf Korrektheit, bewährte Sicherheitspraktiken, Fehler und Randfälle konzentriert. Überprüfen Sie entweder eine GitHub-PR (bevorzugt über gh) oder ein lokales Diff vs. Base, wenn keine PR vorhanden ist. Aktualisierungsverfolgung: Markieren Sie PRD als „Überprüft“, wenn LGTM; Wenn der PR zusammengeführt wird, markieren Sie Zusammengeführt, benennen Sie PRD in done-* um und aktualisieren Sie den Status von task/todo.md auf ✅.
