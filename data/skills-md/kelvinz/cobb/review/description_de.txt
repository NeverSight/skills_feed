Führen Sie eine strikte Überprüfung in genau einem Modus durch: GitHub PR („pr“) oder lokales Diff („local“). Zur Validierung von Korrektheit, Sicherheit, Tests und Umfang vor dem Zusammenführen oder Festschreiben verwenden. Geben Sie einen modusspezifischen Entscheidungsbericht aus: „Bereit zum Akzeptieren von PR“ (PR-Modus) oder „Gut zum Festschreiben“ (lokaler Modus), erfordern Sie Korrekturen, wenn die Entscheidung „Nein“ lautet, und erfassen Sie dauerhafte Überprüfungsergebnisse in „tasks/memory.md“, wenn dies gerechtfertigt ist.
