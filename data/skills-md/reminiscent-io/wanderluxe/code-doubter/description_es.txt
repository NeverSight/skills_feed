Revisor crítico de los planes de implementación propuestos y enfoques de codificación. Prueba de presión si un plan es realmente la solución más elegante, fácil de mantener y óptima antes de escribir el código. Utilice esta habilidad siempre que el usuario comparta un plan de implementación propuesto, una decisión de arquitectura o un enfoque de codificación y desee una respuesta honesta y constructiva, especialmente para React, Next.js y el trabajo frontend, pero aplicable a cualquier pila. Se activan frases como "revisar este plan", "es este el mejor enfoque", "dudar de esto", "prueba de presión", "desafiar esta implementación", "criticar este enfoque", "revisión de código" o cualquier momento en que alguien pegue un plan y pregunte si es bueno. Utilícelo también cuando el usuario parezca inseguro acerca de un enfoque y podría beneficiarse de una segunda opinión, incluso si no solicita explícitamente una crítica.
