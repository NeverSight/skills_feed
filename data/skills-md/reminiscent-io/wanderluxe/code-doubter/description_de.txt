Kritischer Gutachter für vorgeschlagene Implementierungspläne und Codierungsansätze. Prüft unter Druck, ob ein Plan wirklich die eleganteste, wartbarste und optimale Lösung ist, bevor Code geschrieben wird. Nutzen Sie diese Fähigkeit immer dann, wenn der Benutzer einen vorgeschlagenen Implementierungsplan, eine Architekturentscheidung oder einen Codierungsansatz mitteilt und eine ehrliche, konstruktive Gegenleistung wünscht – insbesondere für React, Next.js und Frontend-Arbeit, aber anwendbar auf jeden Stack. Wird bei Formulierungen wie „Diesen Plan überprüfen“, „Ist das der beste Ansatz“, „Zweifle daran“, „Drucktest“, „Diese Implementierung herausfordern“, „Diesen Ansatz kritisieren“, „Codeüberprüfung“ oder immer dann ausgelöst, wenn jemand einen Plan einfügt und fragt, ob er gut ist. Verwenden Sie es auch, wenn der Benutzer sich über einen Ansatz unsicher zu sein scheint und von einer zweiten Meinung profitieren könnte, auch wenn er nicht ausdrücklich um Kritik bittet.
