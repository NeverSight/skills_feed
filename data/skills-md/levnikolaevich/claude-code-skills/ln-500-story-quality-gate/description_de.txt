Qualitäts-Orchestrator auf Story-Ebene mit 4-stufigem Gate (PASS/CONCERNS/FAIL/WAIVED) und Qualitätsfaktor. Durchgang 1: Codequalität -> Regression -> manuelles Testen. Durchgang 2: Tests/Abdeckung überprüfen -> NFR-Werte berechnen -> Story als erledigt markieren. Wird verwendet, wenn der Benutzer ein Qualitätsgate für die Story anfordert oder wenn ln-400 eine Qualitätsprüfung delegiert.
