Шаблоны аутентификации и авторизации, включая OAuth2, JWT, RBAC/ABAC, управление сеансами, ключи API, хеширование паролей и MFA.

КОГДА ИСПОЛЬЗУЕТСЯ: реализация потоков входа в систему, контроля доступа, управления идентификацией, токенов, разрешений, обработки сеансов, аутентификации по ключу API или MFA.
НЕ ИСПОЛЬЗУЙТЕ: для сканирования уязвимостей безопасности (используйте /security-scan), для аудита безопасности (используйте /security-audit), для моделирования угроз (используйте /threat-model).

ТРИГГЕРЫ: вход, выход, вход, регистрация, аутентификация, авторизация, пароль, учетные данные, токен, JWT, OAuth, OAuth2, OIDC, SSO, SAML, сеанс, cookie, RBAC, ABAC, разрешения, роли, MFA, 2FA, TOTP, ключ API, PKCE.
