Ajoutez une couverture de test au code existant avec des modèles fictifs corrects. À utiliser lors de l'ajout de tests à des modules non testés, lors de l'écriture de tests de régression pour détecter des bogues ou lorsque l'utilisateur demande à tester un fichier spécifique. Gère mockReset:true, vi.hoisted(), le modèle de transfert et les utilitaires de création d'applications de test.
