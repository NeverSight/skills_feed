Testgetriebene Entwicklungsmethodik und Rot-Grün-Refaktor-Workflow (früher test-tdd). Diese Fähigkeit sollte beim Üben von TDD, beim Schreiben von Tests zuerst, beim Entwerfen von Tests vor der Implementierung oder beim Überprüfen von Test-First-Ansätzen verwendet werden. Auslöser bei „Tests zuerst schreiben“, „Test vor Code“, „Rot-Grün-Refaktor“, „Testgetriebene Entwicklung“. Dieser Skill deckt NICHT die Besonderheiten des Vitest-Frameworks (verwenden Sie den Vitest-Skill) oder das API-Mocking mit MSW (verwenden Sie den MSW-Skill) ab.
