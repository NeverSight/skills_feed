منهجية التطوير المبنية على الاختبار وسير عمل إعادة البناء باللونين الأحمر والأخضر (المعروف سابقًا باسم test-tdd). يجب استخدام هذه المهارة عند ممارسة TDD، أو كتابة الاختبارات أولاً، أو تصميم الاختبارات قبل التنفيذ، أو مراجعة مناهج الاختبار أولاً. مشغلات على "اكتب الاختبارات أولاً"، و"الاختبار قبل التعليمات البرمجية"، و"إعادة البناء باللون الأحمر والأخضر"، و"التطوير القائم على الاختبار". لا تغطي هذه المهارة تفاصيل إطار عمل Vitest (استخدم مهارة vitest) أو محاكاة واجهة برمجة التطبيقات باستخدام MSW (استخدم مهارة msw).
