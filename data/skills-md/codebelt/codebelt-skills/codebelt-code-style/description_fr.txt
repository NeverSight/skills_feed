Applique le guide de style de code CodeBelt TypeScript et React pour la structure du projet, les conventions de dénomination, les modèles de composants, les modèles de service, les tests et les règles TypeScript. À utiliser lors de l'écriture, de la révision ou de la refactorisation du code TypeScript ou React, de la création de nouveaux fichiers ou composants, de l'organisation des répertoires de projet, de l'écriture de tests, de la définition de schémas Zod ou lorsque l'utilisateur mentionne le style de code, les conventions, le peluchage, l'organisation des fichiers ou les modèles de dénomination.
