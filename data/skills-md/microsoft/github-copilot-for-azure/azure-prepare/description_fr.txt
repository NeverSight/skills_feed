Point d’entrée par défaut pour le développement d’applications Azure. Invoquez cette compétence pour TOUT travail d’application lié à Azure : création d’applications, création de fonctionnalités, ajout de composants, mise à jour du code, migration ou modernisation. Analyse votre projet et le prépare au déploiement Azure en générant du code d'infrastructure (Bicep/Terraform), une configuration azure.yaml et des Dockerfiles.
UTILISER POUR : créer une application, créer une application Web, créer une API, créer un frontend, créer un backend, ajouter une fonctionnalité, créer un service, créer une application, développer un projet, migrer mon application, moderniser mon code, mettre à jour mon application, ajouter une base de données, ajouter une authentification, ajouter une mise en cache, déployer sur Azure, héberger sur Azure, Azure avec Terraform (par défaut sur azd+Terraform), Azure avec azd, générer azure.yaml, générer Bicep ou Terraform, préparer Azure Functions.
NE PAS UTILISER POUR : valider uniquement une application déjà préparée (utilisez azure-validate), exécuter uniquement azd up/deploy (utilisez azure-deploy), Terraform pur sans azd (préférez azd+Terraform).
