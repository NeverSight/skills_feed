Identifizieren Sie alle Dateien, zu denen ein bestimmter Autor in einem Zweig beigetragen hat, im Vergleich zum Upstream, und verfolgen Sie den Code durch Umbenennungen. Verwenden Sie diese Option, wenn Sie gefragt werden, wer was bearbeitet hat, welchen Code ein Autor beigesteuert hat, oder um die Urheberschaft vor einer Zusammenführung zu prüfen. Dieser Skill sollte als Subagent ausgeführt werden – er führt viele Git-Operationen aus und gibt eine übersichtliche Tabelle zurück.
