Écrivez des tests unitaires complets pour le code. À utiliser lorsqu'on vous demande (1) d'écrire des tests pour du code nouveau ou existant, (2) d'ajouter des tests unitaires, (3) de tester une fonction/un module/une classe, (4) de vérifier que le code fonctionne toujours après des modifications, (5) de créer une couverture de test, ou lorsque des expressions telles que « écrire des tests », « ajouter des tests », « tester ceci », « s'assurer que cela fonctionne » apparaissent. Prend en charge Python, JavaScript, R, Rust, Go, C++, SQL, Bash, Ansible, Kubernetes/Kustomize, Docker et Docker Compose.
