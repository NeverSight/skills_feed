Schreiben Sie umfassende Unit-Tests für Code. Verwenden Sie diese Option, wenn Sie aufgefordert werden, (1) Tests für neuen oder vorhandenen Code zu schreiben, (2) Komponententests hinzuzufügen, (3) eine Funktion/ein Modul/eine Klasse zu testen, (4) zu überprüfen, ob der Code nach Änderungen noch funktioniert, (5) eine Testabdeckung zu erstellen oder wenn Ausdrücke wie „Tests schreiben“, „Tests hinzufügen“, „dies testen“ und „stellen Sie sicher, dass dies funktioniert“ auftauchen. Unterstützt Python, JavaScript, R, Rust, Go, C++, SQL, Bash, Ansible, Kubernetes/Kustomize, Docker und Docker Compose.
