مهارة اختبار الوحدة الكاملة لمشاريع TypeScript/NestJS باستخدام Jest وgolevelup/ts-jest وقواعد البيانات الموجودة في الذاكرة.

استخدم هذه المهارة دائمًا عندما يحتاج المستخدم إلى:

**الإعداد** - تهيئة اختبار الوحدة أو تكوينه:
- إعداد Jest لمشروع جديد
- تكوين البنية التحتية للاختبار (jest.config.ts)
- تثبيت تبعيات الاختبار (@nestjs/testing، @golevelup/ts-jest)
- إنشاء مساعدين وهميين أو أدوات اختبار
- إعداد تكوين التغطية

**الكتابة** - إنشاء أو إضافة اختبارات الوحدة:
- كتابة أو إنشاء أو إضافة أو إنشاء اختبارات الوحدة
- اختبار الخدمة أو حالة الاستخدام أو وحدة التحكم أو الحارس أو المعترض أو الأنبوب أو المرشح
- إضافة اختبارات للكود أو الميزات الجديدة
- تحسين تغطية الاختبار أو إضافة الاختبارات المفقودة
- محاكاة التبعيات أو إعداد تركيبات الاختبار
- العمل على أي ملف ينتهي بـ .spec.ts

**المراجعة** - تدقيق أو تقييم اختبارات الوحدة:
- مراجعة الاختبارات الموجودة للتأكد من الجودة
- التحقق من تغطية الاختبار والثغرات
- تدقيق أنماط الاختبار والاتفاقيات
- تقييم جودة التأكيد

**RUN** - تنفيذ أو تحليل نتائج الاختبار:
- تشغيل اختبارات الوحدة
- تحليل نتائج الاختبار أو تقارير التغطية
- فهم حالات الفشل أو النجاح في الاختبار
- التحقق من الاختبارات التي نجحت/رسبت

**DEBUG** - إصلاح الاختبارات الفاشلة أو المعطلة:
- إصلاح اختبارات الوحدة الفاشلة
- تصحيح أخطاء الاختبار أو الاستثناءات
- حل المشكلات الوهمية أو مشكلات الإعداد
- استكشاف أخطاء مهلة الاختبار أو الاختبارات غير المستقرة
- تشخيص النتائج "غير المحددة" أو غير المتوقعة

**التحسين** - تحسين أداء الاختبار وقابلية الصيانة:
- تسريع الاختبارات البطيئة
- إصلاح المقابض المفتوحة التي تمنع الخروج النظيف
- تحسين تنظيم الاختبار
- تقليل وقت تنفيذ الاختبار
- تنظيف رمز الاختبار

الكلمات الرئيسية: اختبار الوحدة، المواصفات، jest، الآلة الكاتبة، Nestjs، mock، DeepMocked، createMock، AAA، تغطية الاختبار، TDD، .spec.ts، الاختبار، اختبار الكتابة، إضافة اختبار، إنشاء اختبار، إصلاح الاختبار، اختبار تصحيح الأخطاء، تشغيل الاختبار، مراجعة الاختبار، تحسين الاختبار، إعداد الاختبار، تكوين jest
