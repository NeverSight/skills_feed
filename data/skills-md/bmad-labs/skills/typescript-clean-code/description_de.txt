Clean Code-Prinzipien, professionelle Praktiken und Arbeitsabläufe für TypeScript-Entwickler. Basierend auf den Büchern „Clean Code“ und „The Clean Coder“ von Robert C. Martin.

Verwenden Sie diese Fähigkeit, wenn:
- Schreiben neuer TypeScript/JavaScript-Funktionen, Klassen oder Module
- Überprüfen von Code- oder Pull-Anfragen
- Refactoring vorhandener Codes
- Nach testgetriebener Entwicklung (TDD)
- Behebung von Fehlern mit angemessener Testabdeckung
- Planung der Teststrategie für Features
- Aufgaben genau einschätzen
- Professioneller Umgang mit Fristen und Verpflichtungen
- Effektives Arbeiten mit Teams
