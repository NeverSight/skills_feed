中国チームの慣習に基づいた構造化された Git コミット ワークフローに関する専門知識。このスキルは、ユーザーが「commit」、「提交」、「stage」、「暂存」、「show status」、「git status」、または git の変更の管理に関連するバリアントを要求するときに使用します。ユーザーは「git-commit-workflow」を明示的に指定する必要はありません。変更をコミットしたり git 状態を管理したりするリクエストはすべて、自動的にこのスキルをトリガーします。厳格な 80 文字行制限に従って、PMS および GitHub 問題追跡を使用した自動コミット メッセージ生成を処理します。
