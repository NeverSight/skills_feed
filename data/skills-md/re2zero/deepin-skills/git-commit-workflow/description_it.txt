Esperienza nel flusso di lavoro strutturato di commit Git con convenzioni del team cinese. Utilizza questa abilità quando l'utente chiede "commit", "提交", "stage", "暂存", "show status", "git status" o qualsiasi variante relativa alla gestione delle modifiche git. NON è necessario che l'utente menzioni esplicitamente "git-commit-workflow": QUALSIASI richiesta di confermare modifiche o gestire lo stato git dovrebbe attivare automaticamente questa abilità. Gestisce la generazione automatica dei messaggi di commit con il monitoraggio dei problemi PMS e GitHub, seguendo rigorosi limiti di riga di 80 caratteri.
