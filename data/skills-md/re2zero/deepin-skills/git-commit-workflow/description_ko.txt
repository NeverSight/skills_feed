중국 팀 규칙을 사용한 구조화된 Git 커밋 워크플로에 대한 전문 지식. 사용자가 "commit", "提交", "stage", "暂存", "show status", "git status" 또는 git 변경 관리와 관련된 변형을 요청할 때 이 기술을 사용하세요. 사용자는 "git-commit-workflow"를 명시적으로 언급할 필요가 없습니다. 변경 사항을 커밋하거나 git 상태를 관리하라는 요청이 있으면 자동으로 이 기술이 트리거되어야 합니다. 엄격한 80자 줄 제한에 따라 PMS 및 GitHub 문제 추적을 통해 자동 커밋 메시지 생성을 처리합니다.
