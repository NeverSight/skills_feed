Применяйте OpenSpec OPSX в строгом цикле «по одной задаче за раз». Используйте, когда пользователь запрашивает выполнение работы как изменение одной задачи, хочет реализовать сначала спецификацию для каждой задачи или говорит использовать метод OpenSpec для каждой задачи из списка задач. Поддерживает как собственные командные среды /opsx, так и резервный режим вручную путем непосредственного создания файлов артефактов OpenSpec.
