Swift のベスト プラクティス、パターン、実装に関する専門家のガイダンス。開発者が以下に言及する場合に使用します: (1) Swift 構成変数または環境変数、(2) Swift-log またはロギング パターン、(3) OpenTelemetry または swift-otel、(4) Swift Testing フレームワークまたは @Test マクロ、(5) Foundation 回避またはクロスプラットフォーム Swift、(6) プラットフォーム固有のコード編成、(7) スパンまたはメモリ安全パターン、(8) コピー不可能なタイプ (~Copyable)、(9) API 設計パターンまたはアクセス修飾子。
