Referenzhandbuch für Best Practices der OpenAPI-Spezifikation, Namenskonventionen und das Ausdrücken komplexer REST-API-Muster wie Polymorphismus, Aufzählungen, Datei-Uploads und vom Server gesendete Ereignisse. Verwenden Sie es beim Schreiben oder Verbessern von OpenAPI-Spezifikationen, um sicherzustellen, dass sie etablierten Konventionen entsprechen und hochwertige SDKs generieren.
