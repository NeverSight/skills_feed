Создавайте реализации маршрутизатора Chi в соответствии с соглашениями о модульной архитектуре Go (маршрутизатор Chi из пакета блоков, Fx DI с интерфейсом chi.Route, конечные точки REST). Всегда используйте этот навык при создании или изменении регистрации HTTP-маршрута во внутреннем/модуле/<модуле>/http/chi/router/, включая любой новый файл маршрутизатора, маршруты CRUD (GET, POST, PUT, DELETE), конечные точки настраиваемых действий, API с поддержкой версий, группы маршрутов с промежуточным программным обеспечением или подключение маршрутизаторов к fx.go модуля.
