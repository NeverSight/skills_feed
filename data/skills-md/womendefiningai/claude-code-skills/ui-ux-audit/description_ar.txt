سير عمل التدقيق الإلزامي لتغييرات واجهة المستخدم/تجربة المستخدم التي تقرأ الحالة الحالية أولاً، وتتحقق من التكرار، وتحترم فلسفة التصميم النظيف، وتحدد الثغرات الحقيقية قبل التنفيذ. يتم الاستدعاء تلقائيًا عندما يذكر المستخدم واجهة المستخدم، أو تجربة المستخدم، أو التصميم، أو التخطيط، أو الصفحة الرئيسية، أو تحسينات الصفحة، أو التغييرات المرئية، أو تعديلات الواجهة.
