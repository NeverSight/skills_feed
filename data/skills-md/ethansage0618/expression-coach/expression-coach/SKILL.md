# 语言表达组织训练技能

本技能旨在通过生成高度定制化的即兴表达训练场景，帮助用户系统性地提升在学术、工作及生活中的沟通能力。它遵循一个模块化的流程，确保每个训练都贴合用户的个人背景和特定需求。

## 核心工作流程

生成一个完整的训练场景包含以下步骤：

1.  **读取用户画像**: 首先，读取 `references/user-profile.md` 文件，以确保生成的场景与用户的身份、专业和挑战紧密相关。

2.  **随机选择场景**: 从 `references/scene-pool/` 目录中随机选择一个场景类别（例如 `academic.md` 或 `social.md`）。这确保了训练的多样性，覆盖从学术交锋到家庭社交的各种情境。

3.  **构建详细情境**: 根据所选的场景类别，结合用户画像，构建一个具体、生动且富有挑战性的情境。此步骤需要：
    *   定义时间、地点、人物关系和对方的关注点。
    *   根据场景类型（工作/生活）融入特定的专业关键词或生活目标。
    *   设置明确的限制条件和核心挑战。

4.  **定义表达任务**: 明确用户的表达任务，包括：
    *   一个略带刁钻的“对方原话”作为触发性问题。
    *   两个有梯度的追问，以考验用户的临场反应和逻辑深度。
    *   明确的表达目标（如：说服、解释、设定边界）和时间限制（3分钟完整版 + 30秒电梯版）。

5.  **提供辅助工具**: 为用户提供完成挑战所需的脚手架，包括：
    *   从 `references/structure-frameworks.md` 中选择一个最合适的表达结构（如 PREP, STAR）。
    *   提供一个包含“路标句”和“稳场句”的语言工具包。
    *   根据场景和受众，从 `references/style-guides/` 目录中推荐一种合适的语言风格。

6.  **启动训练与复盘**: 以明确的指令启动训练。在用户完成表达后，严格按照 `references/feedback-protocol.md` 中的规则进行复盘，提供建设性的反馈。

## 技能模块结构

-   `SKILL.md`: (本文件) 核心工作流程。
-   `references/user-profile.md`: 用户的个人背景和沟通挑战。
-   `references/scene-pool/*.md`: 细分的场景库，每个文件代表一类情境。
-   `references/style-guides/*.md`: 不同沟通风格的指南。
-   `references/structure-frameworks.md`: 结构化表达的经典框架。
-   `references/feedback-protocol.md`: 训练后复盘的规则和流程。
