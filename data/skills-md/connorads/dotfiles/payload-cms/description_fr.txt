À utiliser lorsque vous travaillez avec des projets Payload CMS (payload.config.ts, collections, champs, hooks, contrôle d'accès, API Payload). Déclencheurs sur des tâches impliquant : les définitions de collection, les configurations de champs, les hooks, le contrôle d'accès, les requêtes de base de données, les points de terminaison personnalisés, l'authentification, les téléchargements de fichiers, les brouillons/versions, l'aperçu en direct ou le développement de plugins. À utiliser également lors du débogage des erreurs de validation, des problèmes de sécurité, des requêtes relationnelles, des transactions ou du comportement du hook.
