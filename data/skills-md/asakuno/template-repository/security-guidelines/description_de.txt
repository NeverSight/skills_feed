IPA-Richtlinien-konforme Sicherheitsdiagnose- und Überprüfungsfähigkeiten für Laravel/React-Anwendungen. Wird zur Durchführung von Sicherheitsüberprüfungen während der Codeimplementierung oder -überprüfung verwendet. Diagnostiziert 11 Schwachstellentypen (SQL-Injection, XSS, CSRF usw.), stellt sichere Codebeispiele bereit und führt eine checklistenbasierte Validierung durch. Funktioniert in Verbindung mit .claude/rules/security/, um konkrete Diagnose-Workflows während der Implementierungsphasen bereitzustellen. Wird ausgelöst, wenn: (1) Code auf Sicherheitslücken überprüft wird, (2) Authentifizierung/Autorisierung implementiert wird, (3) Benutzereingaben/-ausgaben verarbeitet werden, (4) mit Sitzungen/Cookies gearbeitet wird, (5) Dateien verarbeitet oder Befehle ausgeführt werden, (6) Formulare oder APIs erstellt werden, (7) Sicherheitsüberprüfungen durchgeführt werden.
