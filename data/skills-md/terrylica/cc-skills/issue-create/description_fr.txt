Créez des tickets GitHub bien formatés avec des suggestions d'étiquettes intelligentes basées sur l'IA et une détection du type de contenu. À utiliser chaque fois que l'utilisateur souhaite créer un rapport de bogue, une demande de fonctionnalité, une question ou un problème de documentation sur GitHub, ou dit « déposer un problème », « créer un problème » ou « créer un problème gh ». Ne PAS utiliser pour gérer les problèmes existants, organiser les hiérarchies de problèmes (utilisez plutôt le workflow des problèmes) ou pour la création de relations publiques.
