Erstellen Sie einen GitHub-Pull-Request gemäß den Projektkonventionen. Verwenden Sie diese Option, wenn der Benutzer eine PR erstellen, Änderungen zur Überprüfung einreichen oder eine Pull-Anfrage öffnen möchte. Übernimmt die Commit-Analyse, die Zweigstellenverwaltung, die Verwendung von PR-Vorlagen und die PR-Erstellung mithilfe des gh-CLI-Tools.
