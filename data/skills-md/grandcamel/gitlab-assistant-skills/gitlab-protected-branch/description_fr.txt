GitLab a protégé les opérations de branche via l'API. Utilisez TOUJOURS cette compétence lorsque l'utilisateur souhaite : (1) afficher les règles de protection des branches, (2) protéger/déprotéger les branches, (3) configurer les niveaux d'accès push/fusion, (4) configurer les exigences d'approbation du propriétaire de code.
