Wenn eine Sitzung einen erheblichen Fehler, ein Missverständnis oder eine Wiederholung enthält
Beheben Sie den Fehler, analysieren Sie die Grundursache und entwerfen Sie eine neue Fähigkeit, die verhindert hätte
es. Die Fertigkeitsbibliothek sollte aus echten und nicht hypothetischen Misserfolgen entstehen
Verbesserungen. Jede schmerzhafte Debugging-Sitzung ist eine Fähigkeit, die darauf wartet, geschrieben zu werden.
