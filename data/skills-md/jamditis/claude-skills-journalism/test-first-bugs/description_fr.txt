Cette compétence doit être utilisée lorsque l'utilisateur signale un bug, décrit un comportement inattendu, dit que quelque chose est « cassé », « ne fonctionne pas », « en échec », mentionne une « erreur », un « problème » ou un « problème » dans le code, ou demande de « réparer » quelque chose. Applique un flux de travail de correction de bogues piloté par les tests.
