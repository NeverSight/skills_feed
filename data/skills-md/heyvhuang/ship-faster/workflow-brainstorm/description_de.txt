Verwenden Sie diese Option, wenn Sie eine vage Idee vor der Implementierung (neue Funktion/Komponente/Verhaltensänderung) in eine bestätigte Designspezifikation umwandeln müssen. Überprüfen Sie zunächst den Projektkontext, stellen Sie dann jeweils eine Frage, bieten Sie zwei bis drei Optionen mit Kompromissen an und geben Sie schließlich den Entwurf in Segmenten (jeweils etwa 200 bis 300 Wörter) mit jeweils einer Bestätigung aus. Auslöser: Brainstorming, Idee klären, Designspezifikation, Konzept verfeinern, Anforderungsklärung.
