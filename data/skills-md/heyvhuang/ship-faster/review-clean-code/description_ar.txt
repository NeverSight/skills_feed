تحليل جودة الكود بناءً على مبادئ "الكود النظيف". حدد مشكلات التسمية وحجم الوظيفة والتكرار والهندسة الزائدة والأرقام السحرية من خلال تقييمات الخطورة واقتراحات إعادة البناء. يُستخدم عندما يطلب المستخدم فحوصات جودة التعليمات البرمجية، أو نصيحة إعادة البناء، أو تحليل التعليمات البرمجية النظيفة، أو اكتشاف رائحة التعليمات البرمجية، أو عندما يذكر مصطلحات مثل مراجعة التعليمات البرمجية، أو جودة التعليمات البرمجية، أو فحص إعادة البناء.
