Analyser la qualité du code sur la base des principes du « Clean Code ». Identifiez les problèmes de dénomination, de taille de fonction, de duplication, de sur-ingénierie et de nombre magique grâce à des évaluations de gravité et des suggestions de refactorisation. À utiliser lorsque l'utilisateur demande des contrôles de qualité du code, des conseils de refactorisation, une analyse Clean Code, une détection d'odeur de code ou mentionne des termes tels que révision de code, qualité du code, vérification de refactoring.
