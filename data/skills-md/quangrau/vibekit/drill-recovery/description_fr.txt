Exercices de reprise après sinistre et listes de contrôle de sécurité pour les projets d'applications Web (SPA, SSR, applications Web full-stack). Axé sur les développeurs solo/indépendants utilisant une infrastructure de niveau libre (Vercel, Supabase, Cloudflare, Netlify, Railway, etc.). Relie les meilleures pratiques des grandes technologies (NIST, Google SRE DiRT, ISO 22301) à l'échelle indépendante. À utiliser lorsque l'utilisateur mentionne des exercices, une reprise après sinistre, un audit de sécurité, une simulation d'incident, une vérification de l'état du projet, des tests de résilience, des stratégies de sauvegarde, une rotation des secrets ou une réponse aux incidents pour les projets Web. Pas pour les applications mobiles, les logiciels de bureau, les outils CLI ou les jeux.
