# 小红书信息图系列生成器

将复杂内容拆分为适合小红书的吸睛信息图系列,支持多种样式选项。

## 使用方法

```bash
# 根据内容自动选择样式和布局
/xhs-images posts/ai-future/article.md

# 指定样式
/xhs-images posts/ai-future/article.md --style notion

# 指定布局
/xhs-images posts/ai-future/article.md --layout dense

# 同时指定样式和布局
/xhs-images posts/ai-future/article.md --style tech --layout list

# 直接输入内容
/xhs-images
[粘贴内容]

# 直接输入并指定选项
/xhs-images --style bold --layout comparison
[粘贴内容]
```

## 选项

| 选项 | 描述 |
|------|------|
| `--style <名称>` | 视觉样式(见样式画廊) |
| `--layout <名称>` | 信息布局(见布局画廊) |

## 两个维度

| 维度 | 控制内容 | 可选值 |
|------|----------|--------|
| **样式(Style)** | 视觉美学:颜色、线条、装饰 | cute, fresh, tech, warm, bold, minimal, retro, pop, notion |
| **布局(Layout)** | 信息结构:密度、排列 | sparse, balanced, dense, list, comparison, flow |

样式 × 布局 可以自由组合。例如:`--style notion --layout dense` 创建一个具有知识分子气质、信息密度高的知识卡片。

## 样式画廊

### 1. `cute` (默认)
甜美、可爱、少女风 - 经典小红书美学
- **颜色**: 粉色、桃色、薄荷色、淡紫色、奶油色背景
- **元素**: 可爱贴纸、表情图标、爱心、星星、闪光
- **最适合**: 生活方式、美妆、时尚、日常小贴士

### 2. `fresh`
清新、自然、干净
- **颜色**: 薄荷绿、天蓝色、白色、浅黄色
- **元素**: 植物图标、云朵、简单形状、留白空间
- **最适合**: 健康、养生、极简生活方式、自我关怀

### 3. `tech`
现代、智能、数字化
- **颜色**: 深蓝色、紫色、青色、深色背景加霓虹强调
- **元素**: 几何图形、数据图标、电路图案、发光效果
- **最适合**: 科技教程、AI内容、数字工具、生产力

### 4. `warm`
温馨、友好、亲切
- **颜色**: 暖橙色、金黄色、棕色、奶油色
- **元素**: 太阳图案、咖啡杯、温馨插画、暖色灯光
- **最适合**: 个人故事、人生感悟、情感内容

### 5. `bold`
高冲击力、吸引眼球
- **颜色**: 红色、橙色、黑色、黄色强调
- **元素**: 强烈的字体排版、感叹号、箭头、对比
- **最适合**: 重要提示、警告、必知内容

### 6. `minimal`
极简、精致
- **颜色**: 黑色、白色、单一强调色
- **元素**: 最大留白、简单图标、干净线条
- **最适合**: 专业内容、严肃话题、优雅呈现

### 7. `retro`
复古、怀旧、时尚
- **颜色**: 柔和粉彩、棕褐色、褪色调
- **元素**: 复古徽章、半调网点、经典字体
- **最适合**: 怀旧内容、经典技巧、永恒建议

### 8. `pop`
活力、充满能量、醒目
- **颜色**: 明亮的主色、霓虹强调、白色
- **元素**: 大胆图形、漫画风格元素、动态构图
- **最适合**: 精彩公告、趣味事实、吸引人的教程

### 9. `notion`
极简手绘线条艺术,知识分子气质
- **颜色**: 黑色轮廓、白色背景、1-2种粉彩强调色
- **元素**: 简单线条涂鸦、几何图形、手绘摇晃效果、最大留白
- **最适合**: 知识分享、概念解释、SaaS内容、生产力技巧

## 布局画廊

### 1. `sparse` (默认)
最少信息、最大冲击力
- **密度**: 每张图1-2个要点
- **留白**: 画布的60-70%
- **结构**: 单一焦点、一个核心信息
- **最适合**: 封面、引语、有力陈述、情感内容

### 2. `balanced`
标准内容布局
- **密度**: 每张图3-4个要点
- **留白**: 画布的40-50%
- **结构**: 标题 + 3-4个要点或章节
- **最适合**: 常规内容页、教程、解释说明

### 3. `dense`
高信息密度,知识卡片风格
- **密度**: 每张图5-8个要点
- **留白**: 画布的20-30%
- **结构**: 多个部分、结构化网格、更多文字
- **最适合**: 总结卡片、备忘单、综合指南、干货总结

### 4. `list`
枚举和排名格式
- **密度**: 4-7项
- **留白**: 画布的30-40%
- **结构**: 编号或符号垂直列表、一致的项目格式
- **最适合**: Top N列表、检查清单、分步指南、排名

### 5. `comparison`
并排对比布局
- **密度**: 2个主要部分,每个2-4个要点
- **留白**: 画布的30-40%
- **结构**: 左右对比、前后对比、优缺点
- **最适合**: 比较、转变、决策助手、对比图

### 6. `flow`
流程和时间线布局
- **密度**: 3-6个步骤/阶段
- **留白**: 画布的30-40%
- **结构**: 带箭头的连接节点、顺序流程
- **最适合**: 流程、时间线、因果链、工作流

## 自动样式选择

当未指定 `--style` 时,根据内容分析选择:

| 内容特征 | 选择的样式 |
|---------|-----------|
| 美妆、时尚、可爱、女孩、粉色 | `cute` |
| 健康、自然、干净、清新、有机 | `fresh` |
| 科技、AI、代码、数字、应用、工具 | `tech` |
| 生活、故事、情感、感受、温暖 | `warm` |
| 警告、重要、必须、关键 | `bold` |
| 专业、商务、优雅、简洁 | `minimal` |
| 经典、复古、古老、传统 | `retro` |
| 有趣、令人兴奋、哇、惊人 | `pop` |
| 知识、概念、生产力、SaaS、notion | `notion` |

## 自动布局选择

当未指定 `--layout` 时,根据内容结构分析选择:

| 内容特征 | 选择的布局 |
|---------|-----------|
| 单一引语、一个要点、封面 | `sparse` |
| 3-4个要点、解释、教程 | `balanced` |
| 5+个要点、总结、备忘单、干货 | `dense` |
| 编号项、top N、检查清单、步骤 | `list` |
| vs、比较、前后、优缺点 | `comparison` |
| 流程、流、时间线、有顺序的步骤 | `flow` |

**按位置选择布局**:
| 位置 | 推荐布局 |
|------|---------|
| 封面 | `sparse` |
| 内容 | `balanced` 或根据内容选择 |
| 结尾 | `sparse` 或 `balanced` |

## 文件管理

### 带文章路径

保存到与文章同一文件夹下的 `xhs-images/` 子目录:

```
posts/ai-future/
├── article.md
└── xhs-images/
    ├── outline.md
    ├── prompts/
    │   ├── 01-cover.md
    │   ├── 02-content-1.md
    │   └── ...
    ├── 01-cover.png
    ├── 02-content-1.png
    └── 03-ending.png
```

### 不带文章路径

保存到 `xhs-outputs/YYYY-MM-DD/[主题简称]/`:

```
xhs-outputs/
└── 2026-01-13/
    └── ai-agent-guide/
        ├── outline.md
        ├── prompts/
        │   ├── 01-cover.md
        │   └── ...
        ├── 01-cover.png
        └── 02-ending.png
```

## 工作流程

### 步骤1: 分析内容并选择样式/布局

1. 阅读内容
2. 如果指定了 `--style`,使用该样式;否则自动选择
3. 如果指定了 `--layout`,使用该布局;否则为每张图自动选择
4. 根据内容复杂度确定图片数量:

| 内容类型 | 图片数量 |
|---------|---------|
| 简单观点/单一主题 | 2-3张 |
| 中等复杂度/教程 | 4-6张 |
| 深度分析/多维度 | 7-10张 |

**注意**: 布局可以在系列中的每张图片上变化。封面通常使用 `sparse`,内容页根据需要使用 `balanced`/`dense`/`list`。

### 步骤2: 生成大纲

为每张图片规划样式和布局规范:

```markdown
# 小红书信息图系列大纲

**主题**: [主题描述]
**样式**: [选择的样式]
**默认布局**: [选择的布局或"不同"]
**图片数量**: N
**生成时间**: YYYY-MM-DD HH:mm

---

## 图片 1 / N

**位置**: 封面
**布局**: sparse
**核心信息**: [一句话概括]
**文件名**: 01-cover.png

**文本内容**:
- 标题: xxx
- 副标题: xxx

**视觉概念**: [符合样式+布局的描述]

---

## 图片 2 / N

**位置**: 内容
**布局**: [balanced/dense/list/comparison/flow]
**核心信息**: [一句话概括]
**文件名**: 02-xxx.png

**文本内容**:
- 标题: xxx
- 要点: [根据布局密度列出]

**视觉概念**: [匹配样式+布局的描述]

---
...
```

### 步骤3: 保存大纲

将大纲保存为 `outline.md`。

### 步骤4: 逐一生成图片

为每张图片创建一个带有样式和布局规范的提示词文件。

**提示词格式**:

```markdown
信息图主题: [主题]
样式: [样式名称]
布局: [布局名称]
位置: [封面/内容/结尾]

视觉构图:
- 主视觉: [符合样式的描述]
- 排列方式: [布局特定结构]
- 装饰元素: [样式特定装饰]

配色方案:
- 主色: [样式主色]
- 背景色: [样式背景色]
- 强调色: [样式强调色]

文本内容:
- 标题: 「xxx」(大号、突出)
- 关键要点: [根据布局密度]

布局说明: [布局特定指导]
样式注意事项: [样式特定特征]
```

**布局特定说明**:

| 布局 | 排列说明 |
|------|---------|
| `sparse` | 单一焦点居中,1-2个文本元素,最大留白 |
| `balanced` | 标题在顶部,3-4个要点清晰分区,中等间距 |
| `dense` | 网格或多部分布局,5-8个要点,紧凑但有序 |
| `list` | 垂直编号/符号列表,一致的项目间距,清晰层次 |
| `comparison` | 两栏分割,清晰分隔线,左右镜像结构 |
| `flow` | 横向或纵向流程带箭头,连接的节点/步骤 |

然后使用以下命令生成:

```bash
/gemini-web --promptfiles [SKILL_ROOT]/skills/xhs-images/prompts/system.md [TARGET_DIR]/prompts/01-cover.md --image [TARGET_DIR]/01-cover.png
```

每张图片生成后:
1. 确认生成成功
2. 报告进度:"已生成 X/N"
3. 继续下一张

### 步骤5: 完成报告

```
小红书信息图系列完成!

主题: [主题]
样式: [样式名称]
布局: [布局名称或"不同"]
位置: [目录路径]
图片: 共N张

- 01-cover.png ✓ 封面 (sparse)
- 02-content-1.png ✓ 内容 (balanced)

```

## 布局参考详情

### sparse
```
信息密度: 非常低 (1-2个要点)
留白: 60-70%
结构: 单一居中焦点
文本元素: 仅标题,或标题 + 一个副标题/标语
视觉平衡: 居中、对称、四周留白充足
最佳搭配: 任何样式,特别适合 bold、minimal、notion
```

### balanced
```
信息密度: 中等 (3-4个要点)
留白: 40-50%
结构: 标题在顶部,内容部分在下方
文本元素: 标题 + 3-4个要点或关键信息
视觉平衡: 标题在上方,内容均匀分布在下方
最佳搭配: 所有样式都适用
```

### dense
```
信息密度: 高 (5-8个要点)
留白: 20-30%
结构: 多部分网格或堆叠块
文本元素: 标题 + 多个带标题的部分 + 众多要点
视觉平衡: 有序的复杂性,清晰的部分边界,紧凑间距
最佳搭配: tech、notion、minimal (干净样式防止视觉过载)
```

### list
```
信息密度: 中高 (4-7项)
留白: 30-40%
结构: 带编号或符号的垂直枚举
文本元素: 标题 + 编号/符号项,每项格式一致
视觉平衡: 左对齐列表,清晰的编号/符号层次
最佳搭配: 所有样式,特别是 cute (清单)、bold (排名)
```

### comparison
```
信息密度: 中等 (2×2-4个要点)
留白: 30-40%
结构: 两栏分割带中央分隔线
文本元素: 标题 + 左标签 + 右标签 + 镜像要点
视觉平衡: 左右对称,清晰的视觉对比
最佳搭配: bold (戏剧性对比)、tech (数据比较)、warm (前后故事)
```

### flow
```
信息密度: 中等 (3-6步)
留白: 30-40%
结构: 带方向箭头的连接节点
文本元素: 标题 + 步骤标签 + 每步可选描述
视觉平衡: 方向流动 (上→下或左→右),清晰进程
最佳搭配: tech (流程图)、notion (简单流程)、fresh (有机流程)
```

## 内容拆分原则

1. **封面(第1张)**: 强烈视觉冲击,核心标题,吸引注意 → `sparse` 布局
2. **内容(中间)**: 每张图核心要点,密度根据内容变化 → `balanced`/`dense`/`list`/`comparison`/`flow`
3. **结尾(最后一张)**: 总结/行动号召/难忘引语 → `sparse` 或 `balanced`

## 注意事项

- 图片生成通常需要每张10-30秒
- 生成失败时自动重试一次
- 对敏感公众人物使用卡通替代形象
- 输出语言与输入内容语言匹配
- 在系列的所有图片中保持所选样式的一致性
