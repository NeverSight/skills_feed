Esta habilidad debe utilizarse para el desarrollo de funciones estructuradas con comprensión de la base del código. Se activa con el comando /do. Proporciona un flujo de trabajo de 7 fases (descubrimiento, exploración, aclaración, arquitectura, implementación, revisión, resumen) utilizando codeagent-wrapper para organizar el explorador de código, el arquitecto de código, el revisor de código y el desarrollo de agentes en paralelo.
