Patrones de arquitectura basados ​​en servidor para Inertia Rails + React. Cargue esto PRIMERO cuando cree cualquier página o característica de Inertia: dirige a la habilidad correcta. Matriz de decisiones para carga de datos, formularios, navegación, gestión de estados. NUNCA useEffect+fetch, NUNCA redirija_a para URL externas (use inertia_location), NUNCA reaccione-hook-form (use el componente Formulario). DEBE invocarse al agregar páginas, modelos con vistas, CRUD o mostrar datos en una aplicación de Inertia Rails. SIEMPRE `renderiza inercia: { clave: valor }` para pasar datos; `@ivars` NO se pasan automáticamente como accesorios.
