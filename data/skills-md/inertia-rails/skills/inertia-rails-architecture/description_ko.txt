Inertia Rails + React를 위한 서버 기반 아키텍처 패턴입니다. Inertia 페이지나 기능을 구축할 때 이 항목을 가장 먼저 로드하세요. 올바른 스킬로 라우팅됩니다. 데이터 로딩, 양식, 탐색, 상태 관리를 위한 결정 매트릭스입니다. 절대 Effect+fetch를 사용하지 마세요. 외부 URL에 절대 리디렉션_to를 사용하지 마세요(inertia_location 사용). 절대 반응 후크 양식(양식 구성 요소 사용)을 사용하지 마세요. 페이지, 뷰가 있는 모델, CRUD를 추가하거나 Inertia Rails 앱에 데이터를 표시할 때 반드시 호출해야 합니다. 데이터를 전달하기 위해 항상 `render inertia: { key: value }`를 사용하세요 — `@ivars`는 props로 자동 전달되지 않습니다.
