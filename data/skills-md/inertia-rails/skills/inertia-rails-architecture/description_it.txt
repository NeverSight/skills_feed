Modelli di architettura basati su server per Inertia Rails + React. Carica questo PRIMA quando crei una pagina o una funzione Inertia: indirizza alla giusta abilità. Matrice decisionale per caricamento dati, modulistica, navigazione, gestione stati. Non utilizzare MAI Effect+fetch, MAI reindirizzare_to per URL esterni (utilizzare inertia_location), MAI react-hook-form (utilizzare il componente Modulo). DEVE essere richiamato quando si aggiungono pagine, modelli con visualizzazioni, CRUD o si visualizzano dati in un'app Inertia Rails. SEMPRE `render inertia: { key: value }` per passare i dati — le `@ivar` NON vengono passate automaticamente come oggetti di scena.
