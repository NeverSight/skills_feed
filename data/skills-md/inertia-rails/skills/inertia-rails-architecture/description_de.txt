Servergesteuerte Architekturmuster für Inertia Rails + React. Laden Sie dies ZUERST, wenn Sie eine Inertia-Seite oder ein Feature erstellen – es leitet an den richtigen Skill weiter. Entscheidungsmatrix für Datenladen, Formulare, Navigation, Statusverwaltung. NIEMALS „Effect+fetch“ verwenden, NIEMALS „direct_to“ für externe URLs verwenden (inertia_location verwenden), NIEMALS „react-hook-form“ verwenden (Formularkomponente verwenden). MUSS beim Hinzufügen von Seiten, Modellen mit Ansichten, CRUD oder beim Anzeigen von Daten in einer Inertia Rails-App aufgerufen werden. IMMER „render inertia: { key: value }“, um Daten zu übergeben – „@ivars“ werden NICHT automatisch als Requisiten übergeben.
