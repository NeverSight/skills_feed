Inertia Rails + React のサーバー駆動型アーキテクチャ パターン。 Inertia ページまたは機能を構築するときは、これを最初にロードします。これは、適切なスキルにルーティングされます。データのロード、フォーム、ナビゲーション、状態管理のための意思決定マトリックス。決して、Effect+fetch を使用しないでください。外部 URL の redirect_to を決して使用しないでください (inertia_location を使用してください)。また、react-hook-form を決して使用しないでください (Form コンポーネントを使用してください)。 Inertia Rails アプリでページ、ビューを含むモデル、CRUD を追加するとき、またはデータを表示するときに呼び出す必要があります。データを渡すには常に `render inertia: { key: value }` を使用します。 `@ivars` は props として自動的に渡されません。
