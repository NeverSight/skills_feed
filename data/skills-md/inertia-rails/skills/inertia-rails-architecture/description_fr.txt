Modèles d'architecture pilotés par serveur pour Inertia Rails + React. Chargez-le en PREMIER lors de la création d'une page ou d'une fonctionnalité Inertia : il mène à la bonne compétence. Matrice de décision pour le chargement des données, les formulaires, la navigation, la gestion des états. NE JAMAIS utiliser Effect+fetch, NE JAMAIS redirect_to pour les URL externes (utilisez inertia_location), NE JAMAIS réagir-hook-form (utiliser le composant Form). DOIT être invoqué lors de l'ajout de pages, de modèles avec des vues, de CRUD ou de l'affichage de données dans une application Inertia Rails. TOUJOURS `render inertia: { key: value }` pour transmettre les données — `@ivars` ne sont PAS transmis automatiquement en tant qu'accessoires.
