TOUJOURS `render inertia: { key: data }` pour transmettre les données en tant qu'accessoires — les variables d'instance ne sont PAS transmises automatiquement (seul alba-inertia le fait). Modèles de contrôleur Rails pour Inertia.js : inertie de rendu, types d'accessoires (différé, facultatif, fusion, défilement), données partagées, flash, redirections PRG, erreurs de validation. À utiliser lors de l'écriture de contrôleurs qui chargent des données, affichent des enregistrements ou servent des réponses d'inertie. CRITIQUE : les URL externes (Stripe/OAuth) DOIVENT utiliser inertia_location, JAMAIS redirect_to.
