SIEMPRE `renderizar inercia: { clave: datos }` para pasar datos como accesorios; las variables de instancia NO se pasan automáticamente (solo alba-inertia hace eso). Patrones de controlador Rails para Inertia.js: inercia de renderizado, tipos de accesorios (aplazar, opcional, fusionar, desplazar), datos compartidos, flash, redirecciones PRG, errores de validación. Úselo al escribir controladores que cargan datos, muestran registros o brindan respuestas de inercia. CRÍTICO: las URL externas (Stripe/OAuth) DEBEN usar inertia_location, NUNCA redirigir_a.
