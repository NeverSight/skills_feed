Comment tester efficacement les modèles de domaine : tests d'objets de valeur (immuabilité, validation), tests d'entités (identité, logique métier), tests d'exceptions de domaine, tests agrégés, modèles de couverture élevée (95 % et plus) et tests d'invariants et de contraintes.

À utiliser dans les cas suivants : tester le code de la couche de domaine, valider des objets de valeur, tester des entités avec une logique métier, garantir les invariants de domaine ou atteindre une couverture de plus de 95 % sur les modèles de domaine.
