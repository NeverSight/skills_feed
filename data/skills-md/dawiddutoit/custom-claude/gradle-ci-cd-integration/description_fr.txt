Configure les versions Gradle pour les plates-formes CI/CD, y compris les actions GitHub,
GitLab CI, Jenkins et CircleCI. À utiliser lors de la configuration de builds automatisés,
configurer des stratégies de mise en cache, exécuter des tests dans des pipelines ou créer
Images Docker dans CI. Déclencheurs sur « configurer Gradle CI », « configurer les actions GitHub »,
"optimiser le cache de build" ou "pipeline CI pour Gradle". Fonctionne avec .github/workflows,
.gitlab-ci.yml, Jenkinsfile et inclut l'optimisation des performances et la gestion des artefacts.
