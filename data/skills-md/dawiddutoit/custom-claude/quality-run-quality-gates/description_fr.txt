Appliquez la définition de Done en exécutant des portes de qualité (vérification de type, peluchage, détection de code mort, tests). À utiliser lorsque la tâche est terminée, avant les validations, lors de la déclaration « terminé » ou lorsque l'utilisateur dit « tout vérifier », « exécuter les portes de qualité », « valider la qualité du code » ou « exécuter des tests et du peluchage ». S'adapte à Python (pyright/ruff/vulture/pytest), JavaScript (tsc/eslint/jest) et à d'autres écosystèmes. OBLIGATOIRE avant de marquer les tâches comme terminées.
