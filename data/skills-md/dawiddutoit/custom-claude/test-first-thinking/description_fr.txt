Applique la discipline de réflexion sur les tests, les fonctionnalités et la maintenabilité AVANT d'écrire le code d'implémentation. À utiliser lors du démarrage de nouvelles classes/méthodes, de la refactorisation du code existant ou lorsqu'on vous demande de « penser d'abord aux tests », « concevoir pour la testabilité », « de quels tests ai-je besoin », « l'approche test d'abord » ou « la réflexion TDD ». Favorise des conceptions simples et maintenables en considérant la testabilité dès le départ. Fonctionne avec n'importe quelle base de code nécessitant une couverture de test et des normes de qualité.
