Effectuez une auto-évaluation systématique des modifications de code avant les validations à l’aide d’une liste de contrôle structurée. Valide les limites de l'architecture, la qualité du code, la couverture des tests, la documentation et les anti-modèles spécifiques au projet. À utiliser avant de s'engager, de créer des PR ou lorsque l'utilisateur dit « vérifier mes modifications », « auto-révision », « vérifier mon code ». S'adapte aux projets Python, JavaScript, TypeScript, Go, Rust.
