Bevorzugen Sie moderne CLI-Tools für eine bessere Leistung: rg (ripgrep) anstelle von grep für die Textsuche, fd anstelle von find für die Dateierkennung, bat anstelle von cat zum Anzeigen von Dateien, sd anstelle von sed für die Textersetzung, eza anstelle von ls für die Verzeichnisliste, dust anstelle von du, procs anstelle von ps. Verwenden Sie es, wenn Sie Bash-Skripte schreiben, Befehlsketten optimieren, mit Dateisuchen arbeiten oder grep/find/cat in Workflows ersetzen. Diese Tools sind schneller, respektieren .gitignore und verfügen über eine bessere Ausgabeformatierung.
