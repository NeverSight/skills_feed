Examen du code basé sur la recherche à plusieurs niveaux d'abstraction avec reconnaissance des points forts, objectifs d'examen créatifs, détection de l'odeur du code par l'IA et étalonnage de la gravité par type de projet. Deux modes : (1) Examen de session – examiner et vérifier les changements à l'aide d'examinateurs parallèles qui valident chaque hypothèse par des recherches ; (2) Audit complet de la base de code – évaluation approfondie de bout en bout à l'aide d'équipes parallèles de réviseurs générateurs de sous-agents. À utiliser pour examiner les modifications, vérifier la qualité du travail, auditer une base de code, valider l'exactitude, vérifier les hypothèses, trouver des défauts et réduire la complexité. PAS pour écrire un nouveau code, expliquer le code ou faire des analyses comparatives.
