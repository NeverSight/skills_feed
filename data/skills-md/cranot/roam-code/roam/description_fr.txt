Compréhension de la base de code via la CLI itinérante. À utiliser lors de l'exploration des bases de code, de la planification de modifications, du débogage des échecs, de l'évaluation des risques de relations publiques ou de la vérification de l'état de l'architecture. Déclencheurs : compréhension de la structure du projet, contrôles de sécurité avant modification, recherche de symboles/fichiers, analyse du rayon d'explosion, tests concernés, notation de l'état de santé, conseils de refactorisation, révision du code. Nécessite l'installation du code itinérant (`pip install roam-code`) et un projet indexé (`roam init`).
