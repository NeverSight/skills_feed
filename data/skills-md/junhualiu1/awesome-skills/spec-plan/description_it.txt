Quando l'utente desidera condurre uno sviluppo basato sulle specifiche (sviluppo Spec-first / Spec-driven), DEVE utilizzare questa abilità: convergere innanzitutto i requisiti e i vincoli in una specifica implementabile e rivedibile prima di procedere all'implementazione. Questa abilità alla fine produce solo un file Spec scritto in /docs/specs/ ed esegue un git commit (commettendo solo quella Spec). Questa competenza non scrive codice di implementazione.
