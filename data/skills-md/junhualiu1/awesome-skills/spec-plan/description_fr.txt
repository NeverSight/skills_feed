Lorsque l'utilisateur souhaite effectuer un développement basé sur les spécifications (Spec-first / Spec-driven development), vous DEVEZ utiliser cette compétence : faire d'abord converger les exigences et les contraintes dans une spécification implémentable et révisable avant de procéder à la mise en œuvre. Cette compétence ne produit finalement qu'un seul fichier de spécifications écrit dans /docs/specs/ et effectue une validation git (en validant uniquement cette spécification). Cette compétence n'écrit pas de code d'implémentation.
