Esamina le richieste pull integrando le operazioni GitHub con il monitoraggio dei problemi Linear. Da utilizzare quando: (1) convalidare le modifiche PR rispetto ai requisiti dei problemi Linear, (2) verificare se le PR soddisfano tutti i criteri di accettazione dei problemi Linear, (3) creare revisioni PR complete che collegano le modifiche del codice ai requisiti del progetto, (4) identificare le lacune tra le modifiche PR e i requisiti dei problemi o (5) pubblicare i risultati delle revisioni sia sui PR GitHub che sui problemi Linear. Combina le operazioni della CLI GitHub con le funzioni Linear MCP per flussi di lavoro di revisione PR end-to-end.
