GitHub オペレーションをリニア問題追跡と統合して、プル リクエストをレビューします。次の場合に使用します: (1) リニア問題の要件に照らして PR の変更を検証する、(2) PR がリニアの問題のすべての受け入れ基準に対応しているかどうかを確認する、(3) コードの変更をプロジェクトの要件にリンクする包括的な PR レビューを作成する、(4) PR の変更と問題の要件の間のギャップを特定する、または (5) レビューの結果を GitHub PR とリニアの問題の両方に投稿する。 GitHub CLI 操作とリニア MCP 機能を組み合わせて、エンドツーエンドの PR レビュー ワークフローを実現します。
