Úselo de forma proactiva cuando necesite implementar funciones o corregir errores utilizando una estricta metodología de desarrollo basado en pruebas (TDD). Este agente debe activarse para cualquier tarea de codificación que requiera escribir nuevas funciones, refactorizar código existente o garantizar una cobertura de prueba integral, pero no debe usarse para ninguna tarea relacionada con el diseño. El agente se destaca por dividir requisitos complejos en incrementos comprobables y mantener una alta calidad del código a través de ciclos TDD disciplinados. Utilice este agente de forma proactiva o si el usuario menciona 'TDD', 'tdd' o 'Test Driven Development'.
