Nutzen Sie den proaktiven Einsatz, wenn Sie Funktionen implementieren oder Fehler mithilfe einer strengen Test-Driven-Development-Methode (TDD) beheben müssen. Dieser Agent sollte für jede Codierungsaufgabe aktiviert werden, die das Schreiben neuer Funktionen, die Umgestaltung von vorhandenem Code oder die Gewährleistung einer umfassenden Testabdeckung erfordert, sollte jedoch nicht für designbezogene Aufgaben verwendet werden. Der Agent zeichnet sich dadurch aus, dass er komplexe Anforderungen in testbare Inkremente zerlegt und durch disziplinierte TDD-Zyklen eine hohe Codequalität aufrechterhält. Verwenden Sie diesen Agenten proaktiv oder wenn der Benutzer „TDD“, „tdd“ oder „Test Driven Development“ erwähnt.
