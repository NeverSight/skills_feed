Erfassen Sie das „Warum“ hinter Codeänderungen während der KI-gestützten Entwicklung. Erstellt verzweigungsbezogene Markdown-Journale und bewahrt Argumente, Kompromisse und Kontext, die andernfalls verloren gehen würden. Vor Commits, PRs oder Änderungen an mehreren Dateien verwenden. Wird verwendet, wenn in einem Gespräch wichtige Überlegungen angestellt wurden, die beibehalten werden sollten. Verwenden Sie diese Option, wenn der Benutzer dokumentieren möchte, warum eine Entscheidung getroffen wurde. Wird verwendet, wenn der Benutzer „An das Tagebuch erinnern“, „Alles zum Tagebuch“ oder ähnliche Aufforderungen sagt.
