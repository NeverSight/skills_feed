Capturez le « pourquoi » derrière les modifications de code lors du développement assisté par l'IA. Crée des journaux de démarques à l'échelle des branches en préservant le raisonnement, les compromis et le contexte qui seraient autrement perdus. À utiliser avant les validations, les PR ou les modifications multi-fichiers. À utiliser lorsqu'un raisonnement important s'est produit au cours d'une conversation et doit être préservé. À utiliser lorsque l'utilisateur demande à documenter pourquoi une décision a été prise. À utiliser lorsque l'utilisateur dit « n'oubliez pas de journaliser », « quoi que ce soit à journaliser » ou des invites similaires.
