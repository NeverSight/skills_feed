EARS 요구사항 → 설계 → 작업 → 코드 → 테스트 전반에 걸쳐 완전한 요구사항 추적성을 검증합니다.

트리거 용어: 추적성, 요구 사항 적용 범위, 적용 범위 매트릭스, 추적성 매트릭스,
요구 사항 매핑, 테스트 적용 범위, EARS 적용 범위, 요구 사항 추적, 추적성 감사,
격차 감지, 고아 요구 사항, 테스트되지 않은 코드, 적용 범위 검증, 추적성 분석.

포괄적인 검증을 통해 헌법 제5조(추적성 의무)를 시행합니다.
- 요구사항 → 설계 매핑(100% 적용)
- 디자인 → 업무 매핑
- 태스크 → 코드 구현 매핑
- 코드 → 테스트 매핑 (100% 커버리지)
- 갭 감지(고아 요구사항, 테스트되지 않은 코드)
- 적용 비율 보고
- 추적성 매트릭스 생성

사용 시기: 사용자에게 추적성 검증, 적용 범위 분석, 격차 감지가 필요합니다.
또는 전체 개발 라이프사이클에 걸쳐 요구사항을 추적합니다.
