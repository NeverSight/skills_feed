Développement piloté par les tests avec le flux de travail canonique en 5 étapes de Kent Beck : liste de tests, écriture du test, réussite, refactorisation, répétition. Cycles de tranches verticales stricts avec détection automatique du projet. À utiliser lorsque l'utilisateur demande « d'écrire des tests en premier », « d'utiliser TDD », « red-green-refactor », « développement piloté par les tests », « créer une fonctionnalité avec TDD », « corriger un bug avec TDD », souhaite tester d'abord le développement pour n'importe quel langage ou framework, ou mentionne « tracer bullet » ou « slice verticale ».
