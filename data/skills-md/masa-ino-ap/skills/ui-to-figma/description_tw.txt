一種工作流程技能，可分析 CSS、Tailwind 設定和現有專案的元件，以了解設計標記、直觀地改進 UI，並透過 Figma MCP 伺服器將其傳送到 Figma。它不僅僅是增強 UI，還確保改進與專案的設計系統保持一致。它提出了新的顏色和間距，無需硬編碼，保持在現有的標記和命名約定範圍內。在以下情況下使用它：“查看 UI 並將其發送到 Figma”、“改進此頁面的設計並將其反映在 Figma 中”、“分析 UI 並在 Figma 中創建設計方案”，或者每當需要改進 Web 應用程序 UI 並將設計移交給 Figma 時，尤其是當用戶說“ui-to-figma”、“改進或將 UI”或發送到 Figma”先決條件：Playwright（網頁應用程式測試技能）和 Figma MCP 伺服器必須可用。
