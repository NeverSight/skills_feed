CSS, Tailwind 설정, 기존 프로젝트의 구성 요소를 분석하여 디자인 토큰을 이해하고 UI를 시각적으로 개선한 후 Figma MCP 서버를 통해 Figma로 보내는 워크플로 기술입니다. 단순히 UI를 향상시키는 것이 아니라 프로젝트의 디자인 시스템에 맞춰 개선이 이루어지도록 보장합니다. 하드코딩 없이 기존 토큰과 명명 규칙을 유지하면서 새로운 색상과 간격을 제안합니다. "UI를 보고 Figma로 보내기", "이 페이지 디자인을 개선하여 Figma에 반영하기", "UI를 분석하여 Figma에서 디자인 제안 만들기"와 같은 상황에서 사용하거나, 웹앱 UI 개선 및 디자인을 Figma로 전달해야 할 때마다, 특히 사용자가 "ui-to-figma", "UI를 Figma로 보내기" 또는 "디자인을 개선하여 Figma로 보내기"라고 말할 때 사용합니다. 전제조건: Playwright(웹앱 테스트 기술) 및 Figma MCP 서버를 사용할 수 있어야 합니다.
