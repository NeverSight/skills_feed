مهارة سير العمل التي تحلل CSS وإعدادات Tailwind ومكونات المشروع الحالي لفهم الرموز المميزة للتصميم، وتحسين واجهة المستخدم بشكل مرئي، وإرسالها إلى Figma عبر خادم Figma MCP. بدلاً من مجرد تحسين واجهة المستخدم، فإنه يضمن أن التحسينات تتماشى مع نظام تصميم المشروع. فهو يقترح ألوانًا ومسافات جديدة بدون ترميز ثابت، مع البقاء ضمن الرموز المميزة واصطلاحات التسمية الحالية. استخدمه في مواقف مثل: "انظر إلى واجهة المستخدم وأرسلها إلى Figma"، أو "تحسين تصميم هذه الصفحة وعكسها في Figma"، أو "تحليل واجهة المستخدم وإنشاء اقتراح تصميم في Figma"، أو عندما تكون هناك حاجة إلى تحسينات واجهة مستخدم تطبيق الويب وتسليم التصميم إلى Figma - خاصة عندما يقول المستخدم "ui-to-Figma" أو "أرسل واجهة المستخدم إلى Figma" أو "تحسين التصميم وإرساله إلى Figma". المتطلبات الأساسية: يجب أن يكون الكاتب المسرحي (مهارة اختبار تطبيق الويب) وخادم Figma MCP متاحين.
