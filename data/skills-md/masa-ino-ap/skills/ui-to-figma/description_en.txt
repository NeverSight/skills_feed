Analyze the CSS, Tailwind settings, and components of an existing project to understand its design tokens, then visually improve the UI and send the result to Figma via the Figma MCP server. Rather than just UI tweaks, it guarantees changes aligned with the project’s design system—no hard-coded colors or spacing, only proposals within existing tokens and naming rules. Use it when: – “Look at this UI and send it to Figma” / “Improve this page’s design and sync it to Figma” – “Analyze the UI and create a design proposal in Figma” – Whenever you need to refine a web app’s UI and sync the design to Figma – When the user says “ui-to-figma”, “send UI to Figma”, or “improve design and push to Figma” Prerequisites: Playwright (webapp-testing skill) and Figma MCP server must be available.
