明確なインターフェイス、カプセル化、最小限のプロパティを備えた単一責任原則に従って、再利用可能で構成可能な UI コンポーネントを設計します。このスキルは、フレームワーク (React、Vue、Svelte、Web コンポーネント) またはコンポーネント ファイルでフロントエンド コンポーネントを作成または変更するときに使用します。コンポーネント ディレクトリ (components/、src/components/、ui/、lib/) 内の .jsx、.tsx、.vue、.svelte ファイルを操作する場合、コンポーネントのプロパティとインターフェイスを定義する場合、コンポーネント構成パターンを実装する場合、コンポーネント レベルの状態を管理する場合、再利用可能な UI 要素 (ボタン、フォーム、カード、モーダル) を作成する場合、コンポーネント API を文書化する場合、または再利用性を高めるためにコンポーネントをリファクタリングする場合に適用します。そしてメンテナンス性。
