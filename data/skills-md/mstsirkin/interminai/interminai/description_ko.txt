vim, git rebase -i, git add -i, git add -p, apt, rclone config, sudo, w3m 및 TUI 앱과 같은 대화형 터미널 애플리케이션을 제어합니다. 또한 다른 CLI LLM(커서 에이전트, 코덱스 등)을 감독할 수 있습니다. 확인 프롬프트에서 y/n을 눌러 해당 작업을 승인하거나 거부할 수 있습니다. 키보드 입력이 필요하거나 프롬프트, 메뉴를 표시하거나 전체 화면 인터페이스가 있는 애플리케이션과 상호 작용해야 할 때 사용합니다. 또한 명령이 실패하거나 "입력이 터미널이 아닙니다" 또는 "출력이 터미널이 아닙니다"와 같은 오류가 발생하는 경우에도 사용합니다. GIT_SEQUENCE_EDITOR와 같은 특정 애플리케이션 핵이나 파일 사용을 통한 상호 작용을 우회하는 것보다 낫습니다.
