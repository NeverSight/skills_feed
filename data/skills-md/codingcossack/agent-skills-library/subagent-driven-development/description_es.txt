Ejecución secuencial de subagente con puertas de revisión de dos etapas para planes de implementación. Úselo cuando ejecute planes de tareas múltiples en la sesión actual, cuando las tareas necesiten un nuevo contexto de subagente para evitar la contaminación, cuando se requieran ciclos de revisión formales (cumplimiento de especificaciones y luego calidad del código) entre tareas o cuando necesite una validación basada en diferencias de cada tarea antes de continuar.
