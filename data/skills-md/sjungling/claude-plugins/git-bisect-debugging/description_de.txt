Wird zum Debuggen von Regressionen oder zum Identifizieren des Commits verwendet, der einen Fehler verursacht hat – bietet einen systematischen Workflow für Git Bisect mit automatisierten Testskripten, manueller Überprüfung oder hybriden Ansätzen. Kann über das systematische Debugging als Debugging-Technik aufgerufen oder eigenständig verwendet werden, wenn Sie wissen, dass es sich um ein historisches Problem handelt.
