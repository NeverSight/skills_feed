À utiliser lors du débogage de régressions ou de l'identification du commit qui a introduit un bogue - fournit un flux de travail systématique pour git bisect avec des scripts de test automatisés, une vérification manuelle ou des approches hybrides. Peut être invoqué à partir du débogage systématique en tant que technique de débogage, ou utilisé de manière autonome lorsque vous savez que le problème est historique.
