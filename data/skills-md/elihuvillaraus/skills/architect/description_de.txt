Gibt PRDs aus, die in Prioritätsgruppen organisiert sind, wobei Aufgaben innerhalb jeder Gruppe parallel von unabhängigen Entwickler-Subagenten (Ralph) ausgeführt werden können. Jede User Story enthält Dateieigentum, technische Spezifikationen und Akzeptanzkriterien, die so detailliert sind, dass ein Modell der Sonnet-Klasse ohne Klärung implementiert werden kann.
