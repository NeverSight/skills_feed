Enseigner et appliquer les pratiques Effect-TS (effect, @effect/schema, @effect/platform) pour l'architecture, les erreurs de frappe, les couches, l'injection de dépendances, la sécurité des ressources, les tests et la migration depuis async/await/Promise. À utiliser lorsqu'un utilisateur demande comment utiliser Effect, demande les meilleures pratiques, souhaite refactoriser vers Effect ou a besoin d'un mappage des modules de plate-forme vers les API Node/Bun/Browser.
