Appliquez l’approche de singularité de la base de code : compréhension et modification fiables de la base de code avec un flux de travail reproductible, des garde-fous et des portes de vérification. À utiliser pour le travail de dépôt (fonctionnalité, correction de bugs, refactorisation, migration) lorsque vous avez besoin d'une confiance élevée, de différences minimales et de critères de validation et de sortie explicites.
