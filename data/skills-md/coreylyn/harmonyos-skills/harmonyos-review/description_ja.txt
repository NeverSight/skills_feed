Huawei の公式開発ガイドラインとセキュリティのベスト プラクティスに照らして ArkTS プロジェクトを監査するための HarmonyOS コード レビュー スキル。 HarmonyOS アプリケーションをレビューする際に使用します: (1) セキュリティ コンプライアンス (ハードコードされた資格情報、暗号化、入力検証)、(2) ArkTS 言語標準 (hilog の使用、タイプ セーフティ、マジック ナンバー)、(3) コンポーネントのライフサイクル管理 (リソースのクリーンアップ、イベント サブスクリプションの処理)、(4) 状態管理 (V1/V2 デコレータの一貫性)、(5) データベース操作 (ResultSet の処理、トランザクション管理、暗号化)、(6) 権限管理 (公式権限パターン)、 (7) パフォーマンスの問題 (async forEach、リソース リーク)、(8) API バージョンの互換性、(9) キットの使用に関するベスト プラクティス。優先順位付けされた修正推奨事項を含む包括的な値下げレポートを生成します。
