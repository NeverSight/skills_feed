Этот навык следует использовать при создании профессиональной документации по программному обеспечению (SRS, PRD, OpenAPI,
руководства пользователя, учебные пособия, инструкции) из шаблонов (с нуля) или документации по реверс-инжинирингу.
из существующего кода, такого как Spring Boot или FastAPI (старое поле). Также занимается аудитом/проверкой документации,
преобразование форматов (Markdown, DOCX, PDF) и создание диаграмм (C4, Mermaid, PlantUML, ER, последовательность).
Используйте, когда вас просят «создать документацию», «задокументировать мой код», «написать SRS», «создать PRD» или «специалист по документации».
