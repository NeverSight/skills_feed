PostgreSQL-Entwicklung für Python mit Volltextsuche (tsvector, tsquery, BM25 über pg_search), Vektorähnlichkeit (pgvector mit HNSW/IVFFlat), JSONB- und Array-Indizierung und Produktionsbereitstellung. Verwendung beim Erstellen von Suchfunktionen, Speichern von KI-Einbettungen, Abfragen von Vektorähnlichkeiten, Optimieren von PostgreSQL-Indizes oder Bereitstellen in AWS RDS/Aurora, GCP Cloud SQL/AlloyDB oder Azure. Behandelt psycopg2, psycopg3, asyncpg, SQLAlchemy-Integration, Docker-Entwicklungseinrichtung und Indexauswahlstrategien. Auslöser sind „PostgreSQL-Suche“, „pgvector“, „BM25 postgres“, „JSONB-Index“, „psycopg“, „asyncpg“, „PostgreSQL Docker“, „AlloyDB-Vektor“. Deckt NICHT ab: DBA-Administration (Sicherung, Replikation, Benutzer), MySQL/MongoDB/Redis, Schema-Design-Theorie, gespeicherte Prozeduren.
