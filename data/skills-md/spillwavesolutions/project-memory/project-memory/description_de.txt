Richten Sie in docs/project_notes/ ein strukturiertes Projektspeichersystem ein und pflegen Sie es, das Fehler mit Lösungen, Architekturentscheidungen, wichtigen Projektfakten und dem Arbeitsverlauf verfolgt. Verwenden Sie diese Fähigkeit, wenn Sie aufgefordert werden, „Projektspeicher einzurichten“, „unsere Entscheidungen zu verfolgen“, „einen Bugfix zu protokollieren“, „Projektspeicher zu aktualisieren“ oder „Speichersystem zu initialisieren“. Konfiguriert sowohl CLAUDE.md als auch AGENTS.md, um die Speichererkennung über verschiedene KI-Codierungstools hinweg aufrechtzuerhalten.
