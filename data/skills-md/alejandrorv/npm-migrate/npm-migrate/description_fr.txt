Migration et mise à niveau assistées par l'IA des packages npm. Gère les modifications avec rupture entre les versions majeures, le nettoyage des dépréciations dans les versions mineures, l'adoption de nouvelles API et de nouveaux modèles, les mises à niveau basées sur la sécurité à partir de l'audit npm et le remplacement complet des dépendances (en échangeant un package contre un autre, par exemple moment → dayjs). Analyse les journaux de modifications, les différences git et les documents, analyse votre base de code pour une utilisation réelle, fait des références croisées pour trouver ce qui est affecté, génère des correctifs de code ou des codemods ciblés et vérifie avec votre suite de tests. Utilisez cette compétence chaque fois qu'un utilisateur mentionne la mise à niveau, la migration ou la mise à jour de packages npm, la gestion de modifications importantes, la correction d'avertissements de dépréciation, le remplacement d'une dépendance par une alternative, l'adoption de nouvelles API à partir d'une mise à jour de package, l'exécution d'un correctif d'audit npm avec des modifications de code ou la comparaison de ce qui a changé entre les versions de package. Les phrases de déclenchement incluent : "mettre à niveau axios vers la v2", "migrer vers express 5", "remplacer moment par dayjs", "corriger les avertissements de dépréciation", "npm audit dit vulnérable", "adopter la nouvelle API", "ce qui a changé entre les versions X et Y", "échanger lodash contre es-toolkit", "aidez-moi à mettre à niveau mes dépendances", "nettoyer les appels obsolètes".
