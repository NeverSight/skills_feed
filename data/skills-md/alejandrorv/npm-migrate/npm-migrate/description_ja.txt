AI 支援による npm パッケージの移行とアップグレード。メジャー バージョン間の重大な変更、マイナー バージョン内の非推奨のクリーンアップ、新しい API とパターンの採用、npm Audit からのセキュリティ主導のアップグレード、および完全な依存関係の置き換え (あるパッケージを別のパッケージと交換する (例: moment → dayjs)) を処理します。変更ログ、git diff、およびドキュメントを分析し、実際の使用状況についてコードベースをスキャンし、影響を受けるものを見つけるための相互参照を行い、ターゲットを絞ったコード修正またはコード修正を生成し、テスト スイートで検証します。ユーザーが npm パッケージのアップグレード、移行、更新、重大な変更への対処、非推奨の警告の修正、依存関係の代替への置き換え、パッケージ更新からの新しい API の採用、コード変更を伴う npm Audit fix の実行、またはパッケージ バージョン間での変更点の比較について言及するたびに、このスキルを使用してください。トリガー フレーズには、「axios を v2 にアップグレードする」、「express 5 に移行する」、「moment を dayjs に置き換える」、「非推奨の警告を修正する」、「npm 監査で脆弱性があると報告された」、「新しい API を採用する」、「バージョン X と Y の間での変更点」、「lodash を es-toolkit に交換する」、「依存関係のアップグレードを手伝ってください」、「非推奨の呼び出しをクリーンアップする」などがあります。
