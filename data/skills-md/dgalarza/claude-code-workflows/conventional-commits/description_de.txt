Diese Fähigkeit sollte beim Erstellen von Git-Commits verwendet werden, um sicherzustellen, dass sie der Spezifikation für konventionelle Commits entsprechen. Es bietet Anleitungen zur Struktur, den Typen, dem Umfang und den Best Practices von Commit-Nachrichten zum Schreiben klarer, konsistenter und automatisierungsfreundlicher Commit-Nachrichten. Verwenden Sie diese Option, wenn Sie Codeänderungen festschreiben oder den Festschreibungsverlauf überprüfen.
