Cette compétence doit être utilisée lors de la création de commits Git pour garantir qu'ils respectent la spécification des commits conventionnels. Il fournit des conseils sur la structure, les types, les portées et les meilleures pratiques des messages de validation pour rédiger des messages de validation clairs, cohérents et automatisés. À utiliser lors de la validation des modifications de code ou de la révision de l'historique des validations.
