Diese Fähigkeit sollte bei der Implementierung von Funktionen aus linearen Problemen mit vollständigem TDD-Workflow, automatisierter Planung, parallelen Codeüberprüfungen (Best Practices für Sicherheit und Rails), systematischer Feedback-Implementierung und automatisierter PR-Erstellung mit linearer Integration verwendet werden. Verwenden Sie diese Option, wenn der Benutzer eine lineare Problem-ID angibt (z. B. „TRA-9“, „DEV-123“) und einen vollständigen Implementierungsworkflow vom Problem bis zur PR wünscht.
