遅いクライアント側を変換することで Next.js App Router データの取得を最適化する 
React Server Components (RSC) を使用した高速サーバー側フェッチへのフェッチ。

次の場合に使用します。
- ユーザーから、読み込みスピナーを使用すると初期ページの読み込みが遅いと報告されました
- ページはデータの取得に useEffect + useState を使用します
- StoreContext/useStore パターンによりウォーターフォールフェッチが発生する
- SEOを改善する必要がある（初期HTMLにコンテンツが含まれていない）
- 「クライアント使用」ページをサーバーコンポーネントに変換
