Compétence complète pour la surveillance des erreurs Sentry et le suivi des performances. À utiliser lorsque Claude doit (1) configurer les SDK Sentry pour le suivi des erreurs et la surveillance des performances, (2) gérer les versions, les cartes sources et les symboles de débogage via CLI, (3) interroger les problèmes, les événements et les métriques via l'API, (4) configurer les règles d'alerte et de notification, (5) configurer les stratégies d'échantillonnage et la gestion des quotas, (6) déployer des instances Sentry auto-hébergées, (7) intégrer OpenTelemetry pour le traçage distribué ou toute autre tâche d'automatisation Sentry.
