Umfassende Kenntnisse in GitOps-Methodik und -Prinzipien für Cloud-native Vorgänge. Verwenden Sie, wenn (1) eine GitOps-Architektur für Kubernetes-Bereitstellungen entworfen wird, (2) eine deklarative Infrastruktur mit Git als Single Source of Truth implementiert, (3) Continuous Deployment-Pipelines mit ArgoCD/Flux/Kargo eingerichtet wird, (4) Verzweigungsstrategien und Repository-Strukturen eingerichtet werden, (5) Abweichungen, Synchronisierungsfehler oder Abstimmungsprobleme behoben werden, (6) GitOps-Tooling-Entscheidungen bewertet werden, (7) Lehren oder Erklären GitOps-Konzepte und Best Practices, (8) Bereitstellung von ArgoCD auf Azure Arc-fähigem Kubernetes oder AKS mit Workload Identity. Deckt die 4 Säulen von GitOps (OpenGitOps), Muster, Anti-Patterns, Tooling-Ökosystem, Azure Arc-Integration und Betriebsanleitung ab.
