Создайте новые элементы пользовательского интерфейса для реестра tryelements.dev. Используйте, когда: (1) Добавление новых компонентов пользовательского интерфейса (кнопки, элементы ввода, карточки), (2) Создание компонентов интеграции (Clerk, Stripe, Uploadthing), (3) Создание элементов, связанных с темой, (4) Любой компонент реестра в стиле shadcn. ВАЖНО: Для компонентов логотипа с вариантами (значок/текстовый знак/логотип + темный/светлый) вместо этого используйте навык «Логотип с вариантами». Этот навык включает в себя создание шаблонов, схему реестра и шаблоны компонентов. ВСЕГДА используйте Context7 MCP для получения последних документов о зависимостях перед реализацией.
