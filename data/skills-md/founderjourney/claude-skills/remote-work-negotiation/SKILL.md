# Remote Work Negotiation

Sistema para negociar efectivamente trabajo remoto y compensacion competitiva como Senior Full-Stack Developer.

## Tu Posicion Negociadora

### Perfil de Valor

```
EXPERIENCIA VERIFICABLE:
- 8+ anos desarrollo full-stack
- Multi-tenancy, SaaS, integraciones complejas
- HostelOS: +1000 usuarios, 99.2% uptime
- Digitaliza: 400 clientes, integracion bancaria
- Tech stack moderno: React, Node.js, PostgreSQL

DIFERENCIADORES:
- Arquitectura multi-tenant (pocos developers lo dominan)
- Integracion de pagos (Stripe, pasarelas locales)
- Experiencia real en produccion a escala
- Track record de delivery remoto exitoso
```

```
TU RATE RECOMENDADO:

Salario deseado $140K / 2000 horas = $70/hora
+ 30% overhead (impuestos, beneficios) = $91/hora
Redondear a $85-95/hora
```

## Scripts de Negociacion

### Cuando preguntan tu expectativa salarial

```
FASE TEMPRANA (antes de entrevistas):
"Prefiero entender mejor el scope del rol y lo que buscan
antes de hablar de numeros. Que rango tienen presupuestado
para este nivel?"

SI INSISTEN:
"Basado en mi experiencia con arquitectura SaaS multi-tenant
y el mercado actual para senior developers remotos, estoy
buscando en el rango de $X-Y. Pero estoy abierto a discutir
el paquete completo incluyendo equity y beneficios."

NUNCA DECIR:
- Un numero exacto primero
- Tu salario actual
- "Estoy flexible" sin contexto
```

### Defendiendo tu rate

```
"Entiendo que puede parecer alto comparado con [mercado local].
Mi rate refleja:

1. Experiencia especifica en multi-tenant SaaS, que es dificil
   de encontrar - la mayoria de developers nunca han manejado
   tenant isolation real.

2. Track record verificable: HostelOS con +1000 usuarios y
   99.2% uptime, Digitaliza con integracion bancaria en
   produccion.

3. Menor riesgo para ustedes: puedo contribuir desde el dia 1
   sin ramp-up extenso en arquitectura SaaS.

Dicho esto, estoy abierto a estructuras creativas - podriamos
explorar equity, bonos por entrega, o un periodo de prueba
con rate ajustado."
```

### Negociando trabajo 100% remoto

```
SI PIDEN HIBRIDO/OFICINA:
"Mis ultimos 4 anos he trabajado 100% remoto con equipos en
US y Europa, entregando consistentemente. HostelOS y Digitaliza
fueron desarrollados completamente de forma remota.

Mi setup esta optimizado para productividad remota:
- Timezone flexible (puedo hacer overlap con US/EU)
- Comunicacion asincrona documentada
- Disponibilidad para calls sincronas cuando sea necesario

Estaria dispuesto a viajar [X veces al ano] para kickoffs o
team events. Podemos hacer un trial period remoto para
demostrar que funciona?"

SI ES DEAL-BREAKER:
"Entiendo que el trabajo presencial es importante para ustedes.
En mi caso, el trabajo remoto no es solo preferencia - es parte
de como he estructurado mi vida y productividad. Si es un
requisito firme, probablemente no soy el mejor fit, pero
aprecio la conversacion."
```

## Evaluacion de Ofertas

### Framework de Decision

```
PONDERACION (ajustar segun prioridades):

Base Salary:        30%
Equity/Bonus:       15%
Remote flexibility: 25%
Tech stack/growth:  15%
Team/culture:       10%
Benefits:           5%

CALCULAR TOTAL COMPENSATION:
Base + (Equity/4) + Bonus target + valor de beneficios
```

### Red Flags

```
EVITAR:
[ ] "Startup equity" sin valoracion clara
[ ] "Competitive salary" sin rangos
[ ] Unlimited PTO (usualmente significa menos PTO real)
[ ] "We're like a family" (boundaries borrosos)
[ ] Respuestas vagas sobre runway/financiamiento
[ ] Presion para decidir rapido
[ ] No pueden explicar el tech stack

PREGUNTAR:
- Cual es el runway actual?
- Como es un dia tipico en el equipo?
- Por que salio la ultima persona de este rol?
- Como se mide el exito en los primeros 90 dias?
```

### Comparando Ofertas

```
OFERTA A vs OFERTA B:

                    | Oferta A    | Oferta B    |
--------------------|-------------|-------------|
Base                | $140K       | $120K       |
Equity              | 0.1%        | 0.5%        |
Remoto              | 100%        | Hibrido     |
Tech Stack          | Legacy+New  | Greenfield  |
PTO                 | 15 dias     | Unlimited   |
Growth              | IC track    | Lead path   |

ANALISIS:
- Oferta B: equity mas alto pero hibrido es dealbreaker
- Oferta A: mejor cash, 100% remoto, tech mixto
- Decision: Oferta A, negociar equity adicional
```

## Equity: Guia Rapida

### Valoracion de Stock Options

```
FORMULA SIMPLIFICADA:
Valor = (Shares * Strike Price) * (Exit Multiple - 1) * (1 - Dilution)

REALIDAD:
- 90% de startups no tienen exit significativo
- Dilution puede ser 50-70% para Series A-C
- Vesting 4 anos = solo 25%/ano

REGLA PRACTICA:
Valorar equity en $0 para decision, cualquier exit es bonus.
Negociar base salary como si equity no existiera.
```

### Preguntas sobre Equity

```
PREGUNTAR SIEMPRE:
1. Cual es el numero total de shares outstanding?
   (Para calcular tu % real)

2. Cual fue la valuacion del ultimo round?
   (Para estimar valor actual)

3. Cual es el strike price de mis options?
   (Costo para ejercer)

4. Hay preferential liquidation para investors?
   (Afecta tu payout en exit)

5. Que pasa con mis options si me voy antes de 4 anos?
   (Vesting cliff, acelerador)
```

## Respuestas a Preguntas Dificiles

### "Por que buscas trabajo remoto?"

```
"El trabajo remoto me permite ser mas productivo y entregar
mejor codigo. En mis ultimos 4 anos 100% remoto, he entregado
sistemas complejos como HostelOS (multi-tenant, +1000 usuarios)
y integraciones bancarias para Digitaliza.

Mi productividad aumenta sin commute y con control de mi
ambiente. Ademas, puedo hacer deep work sin interrupciones
de oficina, lo cual es critico para arquitectura compleja."
```

### "Tu rate es mas alto que developers locales"

```
"Es cierto que el rate por hora es mas alto. Pero considerando
el valor total:

1. ROI: Un developer senior que resuelve problemas de
   arquitectura correctamente desde el inicio ahorra meses
   de refactoring despues.

2. Especializacion: Multi-tenancy y SaaS billing son areas
   donde errores tempranos cuestan 10x arreglar despues.

3. Velocidad: Mi experiencia significa menos ramp-up y
   contribucion inmediata.

Si el presupuesto es una preocupacion real, podemos explorar
un scope mas acotado o un engagement part-time inicial."
```

### "Necesitamos alguien que pueda crecer a lead/manager"

```
"Mi trayectoria es como Senior IC (Individual Contributor).
He liderado proyectos tecnicos y mentorado developers junior,
pero mi fortaleza esta en resolver problemas complejos de
arquitectura hands-on.

En mi experiencia, los mejores equipos tienen tracks separados
para IC y management. Estoy feliz creciendo en el track IC,
llegando a Staff o Principal level, sin necesariamente ir
a management.

Dicho esto, estoy abierto a roles de Tech Lead donde
la mayoria del tiempo sigo siendo hands-on."
```

## Checklist Pre-Negociacion

```
ANTES DE LA LLAMADA:
[ ] Investigar rangos salariales (Levels.fyi, Glassdoor)
[ ] Conocer el stage/funding de la empresa
[ ] Preparar tu "walk away" number
[ ] Tener 2-3 ofertas alternativas (o parecer que las tienes)
[ ] Preparar ejemplos concretos de impacto

DURANTE:
[ ] Dejar que ellos digan el numero primero
[ ] Nunca aceptar en la llamada ("necesito revisarlo")
[ ] Preguntar por el paquete completo, no solo salary
[ ] Mantener tono colaborativo, no adversarial

DESPUES:
[ ] Pedir la oferta por escrito
[ ] Tomarse 48-72 horas para responder
[ ] Negociar UN aspecto a la vez
[ ] Tener respuesta lista si dicen "no"
```
