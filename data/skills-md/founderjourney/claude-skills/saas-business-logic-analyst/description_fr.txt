Activez lorsque l'utilisateur a besoin de : (1) Auditer la logique métier dans le code, (2) Détecter les fuites de revenus ou les bugs de facturation,
(3) Examiner la logique d'abonnement/facturation/multi-tenant, (4) Analyser les cas extrêmes avec un impact commercial,
(5) Évaluer le code pour la mise à l'échelle/la préparation au pivot, (6) Diligence raisonnable sur la base de code SaaS,
(7) Identifier les violations invariantes, (8) Évaluer les risques organisationnels/connaissances dans le code.
