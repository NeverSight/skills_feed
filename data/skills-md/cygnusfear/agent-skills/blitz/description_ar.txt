يجب استخدام هذه المهارة عند موازنة سباقات السرعة متعددة المشكلات باستخدام أشجار عمل git والوكلاء المتوازيين. يُستخدم عند معالجة مشكلات GitHub المتعددة في وقت واحد، عندما يذكر المستخدم "الهجوم الخاطف" أو "السباق الموازي" أو "سير عمل شجرة العمل" أو عند التعامل مع أكثر من 3 مشكلات مستقلة يمكن العمل عليها بشكل متزامن. ينسق سير العمل الكامل بدءًا من فرز المشكلات وحتى تفويض الوكيل الموازي وحتى الدمج المتسلسل.
