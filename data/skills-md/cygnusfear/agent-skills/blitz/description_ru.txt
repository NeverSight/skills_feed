Этот навык следует использовать при распараллеливании спринтов с несколькими задачами с использованием рабочих деревьев git и параллельных агентов. Используйте при одновременном решении нескольких проблем GitHub, когда пользователь упоминает «блиц», «параллельный спринт», «рабочий процесс рабочего дерева» или при обработке более 3 независимых проблем, над которыми можно работать одновременно. Организует весь рабочий процесс от сортировки проблем через параллельное делегирование агентов до последовательного слияния.
