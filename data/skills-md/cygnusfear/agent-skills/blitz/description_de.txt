Diese Fähigkeit sollte bei der Parallelisierung von Multi-Issue-Sprints mithilfe von Git-Worktrees und parallelen Agenten verwendet werden. Verwenden Sie diese Option, wenn Sie mehrere GitHub-Probleme gleichzeitig bearbeiten, wenn der Benutzer „Blitz“, „Parallel Sprint“, „Arbeitsbaum-Workflow“ erwähnt oder wenn Sie mehr als 3 unabhängige Probleme bearbeiten, an denen gleichzeitig gearbeitet werden könnte. Orchestriert den gesamten Arbeitsablauf von der Problemtriage über die parallele Agentendelegation bis hin zur sequenziellen Zusammenführung.
