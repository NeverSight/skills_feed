이 기술은 풀 요청 검토, 포괄적인 코드 검토 수행, 병합 전 코드 변경 분석 또는 사용자가 철저하고 매우 중요한 코드 검토를 요청할 때 사용해야 합니다. 런타임 오류, 코드 일관성 문제, 아키텍처 문제, 환경 호환성 위험 및 검증 전략을 식별하는 매우 중요한 6단계 분석을 수행합니다. GitHub PR 댓글로 구조화된 리뷰를 게시합니다. 사용자가 "PR 검토", "이 코드 검토", "변경 사항 검토", "이 PR 확인", "PR 분석", "검토 게시"를 요청하거나 devflow의 3단계에 사용합니다. 합의 기반 검토를 위해 승수(코드 검토-3, 코드 검토 6X)가 포함된 병렬 검토 모드를 지원합니다. 그는 일을 놓치지 않고 완벽함만을 원하는 극도로 비판적인 평론가입니다.
