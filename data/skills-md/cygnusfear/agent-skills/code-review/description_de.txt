Diese Fähigkeit sollte verwendet werden, wenn Pull-Anfragen überprüft, eine umfassende Codeüberprüfung durchgeführt, Codeänderungen vor der Zusammenführung analysiert werden oder wenn der Benutzer eine gründliche/äußerstkritische Codeüberprüfung anfordert. Führt eine EXTREM KRITISCHE 6-Pass-Analyse durch, um Laufzeitfehler, Codekonsistenzprobleme, Architekturprobleme, Umgebungskompatibilitätsrisiken und Verifizierungsstrategien zu identifizieren. Veröffentlicht eine strukturierte Rezension als GitHub-PR-Kommentar. Wird verwendet, wenn der Benutzer „PR überprüfen“, „diesen Code überprüfen“, „Änderungen überprüfen“, „diesen PR überprüfen“, „PR analysieren“, „Überprüfung veröffentlichen“ auffordert oder für Phase 3 von Devflow. Unterstützt den parallelen Überprüfungsmodus mit Multiplikator (Code-Überprüfung-3, Code-Überprüfung 6X) für konsensbasierte Überprüfungen. Dies ist ein äußerst kritischer Rezensent, der nichts durchgehen lässt und nur Perfektion wünscht.
