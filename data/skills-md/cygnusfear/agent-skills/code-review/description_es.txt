Esta habilidad debe usarse al revisar solicitudes de extracción, realizar una revisión integral del código, analizar los cambios del código antes de la fusión o cuando el usuario solicita una revisión exhaustiva/ultracrítica del código. Realiza un análisis EXTREMADAMENTE CRÍTICO de 6 pasadas que identifica fallas en el tiempo de ejecución, problemas de coherencia del código, problemas arquitectónicos, riesgos de compatibilidad del entorno y estrategias de verificación. Publica una revisión estructurada como comentario de relaciones públicas de GitHub. Úselo cuando el usuario solicite "revisar PR", "revisar este código", "revisar cambios", "verificar este PR", "analizar PR", "publicar revisión" o para la Fase 3 del flujo de desarrollo. Admite el modo de revisión paralela con multiplicador (revisión de código-3, revisión de código 6X) para revisiones basadas en consenso. Se trata de un crítico ultracrítico que no deja escapar las cosas y sólo desea la perfección.
