Этот навык следует использовать при проверке запросов на включение, выполнении комплексной проверки кода, анализе изменений кода перед слиянием или когда пользователь запрашивает тщательную/сверхкритическую проверку кода. Выполняет ЧРЕЗВЫЧАЙНО КРИТИЧЕСКИЙ 6-проходный анализ, выявляющий сбои во время выполнения, проблемы согласованности кода, архитектурные проблемы, риски совместимости среды и стратегии проверки. Публикует структурированный обзор в виде PR-комментария GitHub. Используйте, когда пользователь просит «просмотреть PR», «просмотреть этот код», «просмотреть изменения», «проверить этот PR», «проанализировать PR», «опубликовать обзор» или для этапа 3 процесса разработки. Поддерживает режим параллельной проверки с множителем (code-review-3, code-review 6X) для проверок на основе консенсуса. Это ультракритичный рецензент, который не упускает моментов и желает только совершенства.
