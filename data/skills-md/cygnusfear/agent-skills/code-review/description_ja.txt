このスキルは、プル リクエストのレビュー、包括的なコード レビューの実行、マージ前のコード変更の分析、またはユーザーが徹底的な/超重要なコード レビューを要求する場合に使用する必要があります。非常に重要な 6 パス分析を実行して、実行時の障害、コードの一貫性の問題、アーキテクチャ上の問題、環境互換性のリスク、および検証戦略を特定します。構造化レビューを GitHub PR コメントとして投稿します。ユーザーが「PR をレビューする」、「このコードをレビューする」、「変更をレビューする」、「この PR を確認する」、「PR を分析する」、「レビューを投稿する」、または devflow のフェーズ 3 を要求する場合に使用します。コンセンサスベースのレビューのための乗数 (コードレビュー 3、コードレビュー 6X) を備えた並列レビュー モードをサポートします。これは、物事を見逃さず、完璧だけを求める超批判的な評論家です。
