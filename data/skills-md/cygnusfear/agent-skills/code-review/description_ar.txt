يجب استخدام هذه المهارة عند مراجعة طلبات السحب، أو إجراء مراجعة شاملة للتعليمات البرمجية، أو تحليل تغييرات التعليمات البرمجية قبل الدمج، أو عندما يطلب المستخدم مراجعة شاملة/بالغة الأهمية للتعليمات البرمجية. يقوم بإجراء تحليل بالغ الأهمية مكون من 6 تمريرات لتحديد حالات الفشل في وقت التشغيل، ومشكلات تناسق التعليمات البرمجية، والمشكلات المعمارية، ومخاطر التوافق مع البيئة، واستراتيجيات التحقق. نشر مراجعة منظمة كتعليق على GitHub PR. يُستخدم عندما يطلب المستخدم "مراجعة العلاقات العامة" أو "مراجعة هذا الرمز" أو "مراجعة التغييرات" أو "التحقق من العلاقات العامة" أو "تحليل العلاقات العامة" أو "نشر المراجعة" أو للمرحلة 3 من تدفق التطوير. يدعم وضع المراجعة المتوازية مع المضاعف (مراجعة الكود 3، مراجعة الكود 6X) للمراجعات المبنية على الإجماع. هذا مراجع بالغ الأهمية ولا يترك الأشياء تفلت من أيدينا ويرغب في الكمال فقط.
