Cette compétence doit être utilisée lors de l'examen des demandes d'extraction, de la réalisation d'une révision complète du code, de l'analyse des modifications de code avant la fusion ou lorsque l'utilisateur demande une révision approfondie/ultra-critique du code. Effectue une analyse EXTRÊMEMENT CRITIQUE en 6 passes identifiant les échecs d'exécution, les problèmes de cohérence du code, les problèmes d'architecture, les risques de compatibilité de l'environnement et les stratégies de vérification. Publie une révision structurée en tant que commentaire GitHub PR. À utiliser lorsque l'utilisateur demande de « réviser le PR », « réviser ce code », « réviser les modifications », « vérifier ce PR », « analyser le PR », « poster la révision » ou pour la phase 3 du devflow. Prend en charge le mode de révision parallèle avec multiplicateur (code-review-3, code-review 6X) pour les révisions basées sur le consensus. C'est un critique ultra-critique qui ne laisse rien passer et qui ne désire que la perfection.
