Eine umfassende Architekturprüfung konzentrierte sich auf die Suche nach totem Code, doppelter Funktionalität, architektonischen Anti-Patterns, Typverwirrung und Code-Gerüchen. Verwenden Sie es, wenn der Benutzer eine Architekturanalyse anfordert, toten Code findet, Duplikate identifiziert oder den Zustand der Codebasis beurteilt.
