Détectez, expliquez et corrigez les odeurs de code « Commentaires » : cas où des commentaires excessifs ou explicatifs signalent que le code sous-jacent doit être refactorisé au lieu d'être annoté. Utilisez cette compétence chaque fois que l'utilisateur partage du code avec de nombreux commentaires, pose des questions sur les meilleures pratiques en matière de commentaires, souhaite une révision du code axée sur la lisibilité, demande « mes commentaires sont-ils trop nombreux ? », mentionne des odeurs de code ou souhaite améliorer la clarté du code. Déclenchez également lorsque l'utilisateur colle du code et demande comment le rendre plus propre, plus auto-documenté ou plus facile à comprendre, même s'il ne mentionne pas explicitement « l'odeur des commentaires ».
