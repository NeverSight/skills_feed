Git-Workflow-Management mit atomaren Commit-Prinzipien. Fähigkeiten: Commit-Organisation, Verzweigungsstrategien, Zusammenführungs-/Rebase-Workflows, PR-Management, Verlaufsbereinigung, gestaffelte Änderungsanalyse, Commits mit Einzelverantwortung. Aktionen: Commit-, Push-, Pull-, Merge-, Rebase-, Branch-, Stage- und Stash-Git-Operationen. Schlüsselwörter: Git Commit, Git Push, Git Pull, Git Merge, Git Rebase, Git Branch, Git Stash, Atomic Commit, Commit Message, konventionelle Commits, Verzweigungsstrategie, GitFlow, Trunk-basiert, PR, Pull Request, Code Review, Git History, Cherry-Pick, Squash, Amend, Interactive Rebase, Staged Changes. Wird verwendet, wenn: Commits organisieren, Zweige erstellen, Code zusammenführen, Rebasing durchführen, Commit-Nachrichten schreiben, PRs verwalten, Git-Verlauf bereinigen, inszenierte Änderungen analysieren.
