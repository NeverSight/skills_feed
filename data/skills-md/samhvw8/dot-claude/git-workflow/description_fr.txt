Gestion du workflow Git avec principes de validation atomique. Capacités : organisation des validations, stratégies de branchement, flux de travail de fusion/rebase, gestion des relations publiques, nettoyage de l'historique, analyse des changements par étapes, validations à responsabilité unique. Actions : valider, pousser, tirer, fusionner, rebaser, créer une branche, mettre en scène, cacher les opérations git. Mots clés : git commit, git push, git pull, git merge, git rebase, git branch, git stash, atomique commit, message de commit, commits conventionnels, stratégie de branchement, GitFlow, basé sur le tronc, PR, pull request, révision de code, historique git, cerise-pick, squash, amend, rebase interactif, changements par étapes. À utiliser lorsque : organiser des commits, créer des branches, fusionner du code, rebaser, rédiger des messages de commit, gérer des PR, nettoyer l'historique git, analyser les modifications par étapes.
