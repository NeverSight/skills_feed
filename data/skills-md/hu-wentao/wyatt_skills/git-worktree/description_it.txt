Crea un albero di lavoro git per il repository corrente allo stesso livello di directory della root del progetto. Questa competenza automatizza la creazione dei rami, la denominazione delle directory in base al formato progetto-ramo T e la configurazione iniziale del progetto (ad esempio, l'installazione delle dipendenze). Utilizzalo quando l'utente desidera lavorare su una nuova funzionalità o correggerla senza cambiare l'area di lavoro corrente.
