Audit et optimisation systématique des performances JavaScript/TypeScript à l'aide du profilage V8 et des modèles d'exécution. À utiliser lorsque : (1) les utilisateurs disent "optimiser les performances", "auditer les performances", "c'est lent", "réduire les allocations", "améliorer la vitesse", "vérifier les performances", (2) analyser le code pour détecter les anti-modèles de performances (complexité O(n²), allocations excessives, blocage d'E/S, gaspillage littéral de modèle), (3) optimiser les fonctions quel que soit le contexte d'utilisation actuel - les utilitaires, les formateurs et les analyseurs sont souvent appelés à chaud chemins même lorsqu'ils semblent simples, (4) Correction de la désoptimisation V8 (problèmes monomorphes/polymorphes, mise en cache en ligne). Audite TOUS les codes pour détecter les anti-modèles et rapporte les résultats avec les gains attendus. Couvre les boucles, la mise en cache, le traitement par lots, la localité mémoire, les correctifs de complexité algorithmique avec les modèles ❌/✅.
