OWASP Top 10에 따른 JavaScript/TypeScript 애플리케이션에 대한 포괄적인 보안 감사 및 취약성 감지. (1) 사용자가 '보안 감사', '취약성 확인', '보안 검토', '인증 구현', '이 코드 보호'라고 말하는 경우, (2) 인증 추가, API 엔드포인트, 파일 업로드 또는 사용자 입력 처리, (3) 비밀, 자격 증명 또는 민감한 데이터 작업, (4) 결제 기능 또는 블록체인 통합 구현, (5) 수행 배포 전 보안 점검. 감사 대상: 하드코딩된 비밀, 주입 취약점, XSS/CSRF, 손상된 액세스 제어, 안전하지 않은 인증, 속도 제한, 종속성 취약점, 민감한 데이터 노출.
