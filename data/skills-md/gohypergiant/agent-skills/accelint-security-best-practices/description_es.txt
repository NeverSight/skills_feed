Auditoría de seguridad integral y detección de vulnerabilidades para aplicaciones JavaScript/TypeScript según OWASP Top 10. Úselo cuando (1) los usuarios digan "auditar seguridad", "comprobar vulnerabilidades", "revisión de seguridad", "implementar autenticación", "asegurar este código", (2) agregar autenticación, puntos finales API, carga de archivos o manejar la entrada del usuario, (3) trabajar con secretos, credenciales o datos confidenciales, (4) implementar funciones de pago o integraciones de blockchain, (5) Realización de controles de seguridad previos al despliegue. Auditorías para: secretos codificados, vulnerabilidades de inyección, XSS/CSRF, control de acceso roto, autenticación insegura, limitación de velocidad, vulnerabilidades de dependencia, exposición de datos confidenciales.
