When the user explicitly requests “write/polish the NSFC proposal abstract,” “generate both Chinese and English abstracts,” or “translate the Chinese abstract into English,” use this.

Output both Chinese and English versions: the English must be a faithful translation of the Chinese (no added information, no omission of key constraints).
Default length limits: Chinese ≤400 characters (including punctuation), English ≤4,000 characters (including punctuation); final limits follow `skills/nsfc-abstract/config.yaml:limits`.

Output method: write the result to `NSFC-ABSTRACTS.md` in the working directory (filename per `config.yaml:output.filename`), with one `#` heading for Chinese and one `#` heading for English.

⚠️ Not applicable in the following cases:
- User only wants to translate a generic text unrelated to the proposal (translate directly)
- User only wants to write the rationale/research content/research foundation (use corresponding nsfc series skill)
