Analysieren Sie die JSON-Ausgabe des Terraform-Plans für den AzureRM-Anbieter, um zwischen falsch-positiven Unterschieden (nur auftragsbezogene Änderungen in Set-Typ-Attributen) und tatsächlichen Ressourcenänderungen zu unterscheiden. Verwenden Sie diese Option, wenn Sie die Ausgabe des Terraform-Plans für Azure-Ressourcen wie Application Gateway, Load Balancer, Firewall, Front Door, NSG und andere Ressourcen mit Set-Typ-Attributen überprüfen, die aufgrund interner Reihenfolgeänderungen falsche Unterschiede verursachen.
