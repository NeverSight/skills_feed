Систематическая отладка с использованием подхода глубокоэшелонированной защиты (симптомы → непосредственная причина → основная причина → системные проблемы → предотвращение). Автоматически активируется, когда пользователь упоминает «ошибка», «сбой», «сломанный», «отладка», «ошибка», «проблема» или при запуске /ccpm:verify. Использование Наблюдение → Выдвижение гипотезы → Тестирование → Подтверждение → Исправление → Проверка → Рабочий процесс документирования. Обновляет Linear с результатами и автоматически регистрирует блокировщики, требующие внешних исправлений. Отслеживает коренные причины, а не устраняет симптомы. Интегрируется с ccpm-code-review для проверки того, что исправления прошли все этапы, прежде чем пометить их как завершенные. Предлагает бинарный поиск периодически возникающих проблем и анализ «пяти почему» для сложных проблем.
