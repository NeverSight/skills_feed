심층 방어 접근 방식을 통한 체계적 디버깅(증상 → 즉각적인 원인 → 근본 원인 → 체계적 문제 → 예방) 사용자가 "오류", "실패", "깨짐", "디버그", "버그", "문제"를 언급하거나 /ccpm:verify가 실행될 때 자동으로 활성화됩니다. 관찰 → 가설 → 테스트 → 확인 → 수정 → 검증 → 문서화 워크플로를 사용합니다. 결과로 선형을 업데이트하고 외부 수정이 필요한 차단기를 자동으로 기록합니다. 증상을 패치하는 대신 근본 원인을 추적합니다. ccpm-code-review와 통합하여 수정 사항이 완료로 표시되기 전에 모든 게이트를 통과했는지 확인합니다. 간헐적인 문제에 대한 이진 검색과 복잡한 문제에 대한 5가지 이유 분석을 제안합니다.
