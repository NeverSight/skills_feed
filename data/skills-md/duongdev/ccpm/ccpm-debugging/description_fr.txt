Débogage systématique avec approche de défense en profondeur (symptômes → cause immédiate → cause première → problèmes systémiques → prévention). S'active automatiquement lorsque l'utilisateur mentionne « erreur », « échec », « cassé », « débogage », « bug », « problème » ou lorsque /ccpm:verify s'exécute. Utilise Observer → Hypothèse → Test → Confirmer → Corriger → Vérifier → Flux de travail de document. Met à jour Linear avec les résultats et enregistre automatiquement les bloqueurs qui nécessitent des correctifs externes. Trace les causes profondes au lieu de corriger les symptômes. S'intègre à ccpm-code-review pour vérifier que les correctifs franchissent toutes les portes avant de marquer comme terminés. Suggère une recherche binaire pour les problèmes intermittents et une analyse des cinq pourquoi pour les problèmes complexes.
