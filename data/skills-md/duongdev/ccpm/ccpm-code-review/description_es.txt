Aplica puertas de verificación de calidad con validación de cuatro pasos (pruebas aprobadas, compilación exitosa, lista de verificación completa, sin bloqueadores) antes de completar la tarea, crear relaciones públicas o actualizar el estado. Se activa automáticamente cuando el usuario dice "hecho", "completo", "terminado", "listo para fusionar" o ejecuta los comandos /ccpm:verify o /ccpm:done. Proporciona un flujo de trabajo de verificación sistemático que evita reclamaciones de finalización falsas y garantiza la preparación para la producción. Bloquea las escrituras del sistema externo (Jira, Slack) hasta que se recopilen pruebas. Se integra con la seguridad del sistema externo para el flujo de trabajo de confirmación. Cuando la verificación falla, sugiere /ccpm:verify para depurar problemas sistemáticamente.
