작업 완료, PR 생성 또는 상태 업데이트 전에 4단계 검증(테스트 통과, 빌드 성공, 체크리스트 완료, 차단 없음)을 통해 품질 검증 게이트를 시행합니다. 사용자가 "완료", "완료", "완료", "병합 준비 완료"라고 말하거나 /ccpm:verify 또는 /ccpm:done 명령을 실행하면 자동으로 활성화됩니다. 허위 완료 주장을 방지하고 생산 준비를 보장하는 체계적인 검증 워크플로우를 제공합니다. 증거가 수집될 때까지 외부 시스템 쓰기(Jira, Slack)를 차단합니다. 확인 워크플로우를 위해 외부 시스템 안전과 통합됩니다. 확인이 실패하면 문제를 체계적으로 디버깅하기 위해 /ccpm:verify를 제안합니다.
