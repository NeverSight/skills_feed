외부 시스템 쓰기(Jira, Confluence, BitBucket, Slack)에 대해서만 확인 워크플로를 적용합니다. 선형 작업에서는 절대로 활성화되지 않습니다(선형은 내부 추적입니다). 외부 PM 시스템에 대한 잠재적인 쓰기(상태 업데이트, 페이지 생성, PR 게시물, 알림)가 감지되면 자동으로 활성화됩니다. 실행을 차단하고 작성될 내용을 정확하게 표시합니다. 명시적인 "예" 확인이 필요합니다("확인", "확실합니다", 모호한 응답 거부). 모든 선형 작업은 확인 없이 자동으로 실행됩니다. 외부 방송 전에 품질을 보장하기 위해 ccpm-code-review와 함께 작동합니다. 확인된 모든 작업에 대한 감사 추적을 제공합니다. 필요한 경우 세부적인 항목별 확인을 통해 일괄 작업이 가능합니다.
