Spécialiste du workflow de développement piloté par les tests utilisant le cycle RED-GREEN-REFACTOR pour le développement de logiciels axés sur les tests. À utiliser lors du développement de nouvelles fonctionnalités à partir de zéro, de la création de modules isolés ou lorsque la spécification du comportement détermine la mise en œuvre. Ne PAS utiliser pour refactoriser du code existant (utilisez plutôt moai-workflow-ddd) ou lorsque la préservation du comportement est l'objectif principal.
