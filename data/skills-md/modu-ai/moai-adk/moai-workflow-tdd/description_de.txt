Workflow-Spezialist für testgetriebene Entwicklung, der den RED-GREEN-REFACTOR-Zyklus für die testorientierte Softwareentwicklung verwendet. Verwenden Sie es, wenn Sie neue Funktionen von Grund auf entwickeln, isolierte Module erstellen oder wenn Verhaltensspezifikationen die Implementierung vorantreiben. NICHT zum Refactoring von vorhandenem Code verwenden (verwenden Sie stattdessen moai-workflow-ddd) oder wenn die Erhaltung des Verhaltens das Hauptziel ist.
