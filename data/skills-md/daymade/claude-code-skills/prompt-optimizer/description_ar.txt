قم بتحويل المطالبات الغامضة إلى مواصفات دقيقة ومنظمة بشكل جيد باستخدام منهجية EARS (المنهج السهل لبناء جملة المتطلبات). يجب استخدام هذه المهارة عندما يقدم المستخدمون متطلبات فضفاضة، أو أوصاف ميزات غامضة، أو عندما يحتاجون إلى تحسين المطالبات الخاصة بالأكواد أو المنتجات أو المستندات التي يتم إنشاؤها بواسطة الذكاء الاصطناعي. تتضمن المشغلات طلبات "تحسين الموجه الخاص بي"، أو "تحسين هذا المتطلب"، أو "جعل هذا أكثر تحديدًا"، أو عندما تفتقر المتطلبات الأولية إلى التفاصيل والبنية.
