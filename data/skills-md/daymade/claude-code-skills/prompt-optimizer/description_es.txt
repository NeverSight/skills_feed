Transforme indicaciones vagas en especificaciones precisas y bien estructuradas utilizando la metodología EARS (Enfoque fácil de sintaxis de requisitos). Esta habilidad debe usarse cuando los usuarios proporcionan requisitos vagos, descripciones de características ambiguas o necesitan mejorar las indicaciones para códigos, productos o documentos generados por IA. Los desencadenantes incluyen solicitudes para "optimizar mi mensaje", "mejorar este requisito", "hacerlo más específico" o cuando los requisitos sin procesar carecen de detalles y estructura.
