Questa competenza dovrebbe essere utilizzata quando si stabiliscono processi completi di test di qualità per qualsiasi progetto software. Da utilizzare durante la creazione di strategie di test, la scrittura di casi di test seguendo gli standard di test di Google, l'esecuzione di piani di test, il monitoraggio dei bug con classificazione P0-P4, il calcolo delle metriche di qualità o la generazione di report sullo stato di avanzamento. Include funzionalità di esecuzione autonoma tramite prompt principali e modelli di documentazione completi per i trasferimenti del team QA di terze parti. Implementa i test di sicurezza OWASP e raggiunge gli obiettivi di copertura del 90%.
