Diese Fähigkeit sollte beim Debuggen von Frontend-/UI-Fehlern verwendet werden, die einen Laufzeitnachweis erfordern. VERWENDEN SIE DIESEN SKILL (anstatt console.log hinzuzufügen), wenn Sie Folgendes tun möchten: „console.log hinzufügen und Benutzer bitten, dies zu überprüfen“, „DevTools öffnen und mir sagen, was Sie sehen“, „den Fehler reproduzieren und die Ausgabe teilen“, „die Browserkonsole überprüfen“. Auslöser: „Debuggen“, „Diesen Fehler beheben“, „Warum funktioniert das nicht“, „Untersuchen Sie dieses Problem“, „Verfolgen Sie das Problem“, „Finden Sie heraus, warum X passiert“, „Benutzeroberfläche wird nicht aktualisiert“, „Status ist falsch“, „Wert ist null/undefiniert“, „Klick funktioniert nicht“, „Modal wird nicht angezeigt“. Automatisiert die serverseitige Protokollerfassung – Sie lesen Protokolle direkt, kein Kopieren und Einfügen durch den Benutzer erforderlich.
