يوجه عملية إعادة هيكلة التعليمات البرمجية بشكل منهجي لتحسين جودة التعليمات البرمجية وقابلية الصيانة والتصميم. يحدد روائح التعليمات البرمجية، ويطبق أنماط إعادة البناء، ويضمن تغطية الاختبار، ويتبع ممارسات إعادة البناء الآمنة. يُنتج تعليمات برمجية أكثر وضوحًا وقابلية للصيانة مع الحفاظ على الوظائف. يُستخدم عند تحسين جودة التعليمات البرمجية، أو التخلص من الديون التقنية، أو التحضير لإضافات الميزات، أو معالجة روائح التعليمات البرمجية، أو عندما يذكر المستخدمون إعادة البناء، أو تنظيف التعليمات البرمجية، أو تحسين التصميم، أو تقليل التعقيد، أو تحديث التعليمات البرمجية القديمة.
