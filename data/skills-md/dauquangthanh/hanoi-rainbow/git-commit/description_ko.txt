기존 커밋 표준 및 모범 사례에 따라 잘 구조화된 git 커밋 메시지를 생성합니다. 적절한 유형 접두사(feat, fix, docs, refactor 등), 간결한 주제 및 필요한 경우 자세한 본문을 사용하여 명확하고 설명이 포함된 커밋을 생성합니다. 코드 변경 사항 커밋, git 커밋 생성, 커밋 메시지 작성 시 또는 사용자가 "commit", "git commit", "commit message", "기존 커밋", "changelog"를 언급하거나 버전 제어 메시지 구성에 도움이 필요할 때 사용합니다. 커밋이 원자적이고 설명적이며 팀 규칙을 따르도록 보장합니다.
