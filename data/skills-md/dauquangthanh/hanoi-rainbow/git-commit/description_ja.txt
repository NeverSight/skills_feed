従来のコミット標準とベスト プラクティスに従って、適切に構造化された git コミット メッセージを生成します。必要に応じて、適切な型プレフィックス (feat、fix、docs、refactor など)、簡潔な件名、および詳細な本文を備えた明確で説明的なコミットを作成します。コードの変更をコミットするとき、git コミットを作成するとき、コミット メッセージを書くとき、またはユーザーが「コミット」、「git コミット」、「コミット メッセージ」、「従来のコミット」、「変更ログ」について言及するとき、またはバージョン管理メッセージの構造化にヘルプが必要なときに使用します。コミットがアトミックで記述的であり、チームの規則に従っていることを保証します。
