Genera messaggi git commit ben strutturati seguendo gli standard di commit convenzionali e le migliori pratiche. Crea commit chiari e descrittivi con prefissi di tipo corretti (feat, fix, docs, refactor, ecc.), argomenti concisi e corpi dettagliati quando necessario. Da utilizzare quando si effettuano modifiche al codice, si creano commit git, si scrivono messaggi di commit o quando gli utenti menzionano "commit", "git commit", "messaggio di commit", "commit convenzionali", "log delle modifiche" o hanno bisogno di aiuto per strutturare i messaggi di controllo della versione. Garantisce che i commit siano atomici, descrittivi e seguano le convenzioni del team.
