Conçoit des schémas de bases de données complets comprenant des modèles relationnels et NoSQL, la normalisation, les stratégies d'indexation, la modélisation des relations, les types de données, les contraintes et l'optimisation des performances. Couvre les diagrammes entité-relation, les migrations de schémas, le partitionnement et les meilleures pratiques pour PostgreSQL, MySQL, MongoDB et d'autres bases de données. À utiliser lors de la conception de bases de données, de la création de schémas, de la modélisation de données, de l'optimisation de requêtes ou lorsque les utilisateurs mentionnent « conception de base de données », « conception de schéma », « modélisation de données », « ERD », « normalisation », « indexation » ou « architecture de base de données ».
