Guide la migration de l'infrastructure et de la plateforme, y compris la migration de cloud à cloud (AWS vers GCP, Azure vers AWS), la migration de cluster Kubernetes, les modifications de la plateforme CI/CD, la migration de la pile de surveillance, la migration du fournisseur d'identité et la transformation de l'infrastructure réseau. Couvre la migration IaC, l'orchestration de conteneurs, le maillage de services, les outils d'observabilité et les stratégies multi-cloud. À utiliser lors de la migration d'une infrastructure, d'un changement de fournisseur de cloud, d'un déplacement de clusters Kubernetes ou lorsque les utilisateurs mentionnent « migration vers le cloud », « changement de plate-forme », « migration d'infrastructure », « migration K8s », « multi-cloud » ou « changement de fournisseur de cloud ».
