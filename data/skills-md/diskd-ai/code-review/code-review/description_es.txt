Revise el código con un proceso de revisión estructurado y de alta señal. Úselo cuando se le solicite revisar código, revisar un PR/MR (GitHub PR a través de `gh`, GitLab MR a través de `glab`), revisar una base de código, brindar comentarios sobre una lista de cambios, auditar la calidad del código o realizar una revisión de código. Admite revisiones de diferencias/PR y revisiones completas de la base de código. Produce hallazgos procesables con etiquetas de gravedad, citas exactas de archivos y líneas y un veredicto final. Cubre diseño, funcionalidad, complejidad, pruebas, nombres, comentarios, estilo, documentación, seguridad y rendimiento.
