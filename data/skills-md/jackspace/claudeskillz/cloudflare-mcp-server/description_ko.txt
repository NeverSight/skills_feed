Cloudflare Workers에 MCP(Model Context Protocol) 서버를 구축할 때 이 기술을 사용하세요. 이 기술은 TypeScript를 사용하여 원격 MCP 서버를 배포하거나, OAuth 인증(GitHub, Google, Azure 등)을 구현하거나, 상태 저장 MCP 서버에 내구성 개체를 사용하거나, 비용 최적화를 위해 WebSocket 최대 절전 모드를 구현하거나, 이중 전송 방법(SSE + Streamable HTTP)을 구성할 때 사용해야 합니다. 이 기술은 McpAgent 클래스 내보내기 문제, OAuth 리디렉션 URI 불일치, WebSocket 상태 손실, 지속성 개체 바인딩 오류 및 CORS 구성 실수를 포함한 15가지 이상의 일반적인 오류를 방지합니다. 기본 MCP 서버, OAuth 프록시 통합, 내구성 개체가 있는 상태 저장 서버 및 전체 wrangler.jsonc 구성을 위한 프로덕션 테스트를 거친 템플릿이 포함되어 있습니다. 토큰 유효성 검사, DCR을 통한 원격 OAuth, OAuth 프록시(workers-oauth-provider) 및 전체 OAuth 공급자 구현의 4가지 인증 패턴을 모두 포함합니다. 작업자 및 내구성 개체 기본 사항이 포함되어 있습니다. 토큰 효율성: ~87% 절감(40,000 → 5,000개 토큰). Cloudflare의 공식 MCP 서버에서 프로덕션 테스트를 거쳤습니다.
