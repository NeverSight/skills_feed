Utilisez cette compétence lors de la création de serveurs MCP (Model Context Protocol) sur Cloudflare Workers. Cette compétence doit être utilisée lors du déploiement de serveurs MCP distants avec TypeScript, de la mise en œuvre de l'authentification OAuth (GitHub, Google, Azure, etc.), de l'utilisation d'objets durables pour les serveurs MCP avec état, de la mise en œuvre de l'hibernation WebSocket pour l'optimisation des coûts ou de la configuration de méthodes de transport doubles (SSE + HTTP Streamable). La compétence évite plus de 15 erreurs courantes, notamment les problèmes d'exportation de classe McpAgent, les incompatibilités d'URI de redirection OAuth, la perte d'état WebSocket, les erreurs de liaison d'objets durables et les erreurs de configuration CORS. Comprend des modèles testés en production pour les serveurs MCP de base, l'intégration du proxy OAuth, des serveurs avec état avec Durable Objects et des configurations wrangler.jsonc complètes. Couvre les 4 modèles d'authentification : validation de jeton, OAuth distant avec DCR, proxy OAuth (workers-oauth-provider) et implémentation complète du fournisseur OAuth. Autonome avec les bases de Worker et Durable Objects. Efficacité des jetons : ~87 % d'économies (40 000 → 5 000 jetons). Production testée sur les serveurs MCP officiels de Cloudflare.
