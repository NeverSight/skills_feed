콘텐츠 컬렉션을 위한 프로덕션 테스트 설정 - 변환하는 TypeScript 우선 빌드 도구
자동 유효성 검사를 통해 로컬 콘텐츠 파일(Markdown/MDX)을 유형이 안전한 데이터 컬렉션으로 변환합니다.

사용 시기: Vite + React를 사용하여 블로그, 문서 사이트 또는 콘텐츠가 많은 애플리케이션을 구축할 때,
React 구성 요소로 MDX 콘텐츠 설정, Zod로 유형이 안전한 콘텐츠 스키마 구현,
Contentlayer에서 마이그레이션하거나 콘텐츠 컬렉션에서 TypeScript 가져오기 오류가 발생합니다.

내용: Vite 플러그인 설정, tsconfig 경로 별칭, Zod 검증이 포함된 컬렉션 스키마,
compileMDX를 사용한 MDX 컴파일, 계산된 속성에 대한 변환 함수, 재하이핑/설명 플러그인,
MDXContent, Cloudflare Workers 배포 및 프로덕션 빌드 최적화를 통한 React 구성 요소 통합.

키워드: 콘텐츠 컬렉션, @content-collections/core, @content-collections/vite,
@content-collections/mdx, MDX, 마크다운, Zod 스키마 유효성 검사, 유형이 안전한 콘텐츠,
머리말, compileMDX, 정의Collection, 정의구성, Vite 플러그인, tsconfig 경로,
.content-collections/generated, MDXContent 구성 요소, rehype 플러그인, 댓글 플러그인,
콘텐츠 스키마, 문서 변환, allPosts 가져오기, 정적 사이트 생성,
블로그 설정, 문서, Cloudflare Workers 정적 자산, 콘텐츠 유효성 검사 오류,
모듈을 찾을 수 없습니다. 콘텐츠 컬렉션, 경로 별칭이 작동하지 않음, MDX 유형 오류,
변환 함수 비동기, 컬렉션이 업데이트되지 않음
