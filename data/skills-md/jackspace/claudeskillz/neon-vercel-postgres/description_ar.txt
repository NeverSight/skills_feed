توفر هذه المهارة معرفة شاملة لدمج Postgres بدون خادم Neon وVercel Postgres (المبني على بنية Neon الأساسية) في تطبيقات الويب. يجب استخدامه عند إعداد قواعد بيانات Postgres بدون خادم، أو تكوين تجمع الاتصالات للبيئات الطرفية وبدون خادم، أو تنفيذ سير عمل تفريع قاعدة البيانات، أو استكشاف مشكلات اتصال Postgres وإصلاحها في Cloudflare Workers، أو Vercel Edge Functions، أو وظائف Node.js بدون خادم.

استخدم هذه المهارة عندما:
- إعداد Neon Postgres لعمال Cloudflare أو Vercel Edge أو البيئات التي لا تحتوي على خادم
- تكوين Vercel Postgres لتطبيقات Next.js
- تنفيذ سير عمل تفريع قاعدة البيانات (فروع قاعدة البيانات المشابهة لـ git)
- دمج Drizzle ORM أو Prisma مع Neon/Vercel Postgres
- تصحيح أخطاء تجمع الاتصال أو مهلة المعاملة أو مشكلات تكوين SSL
- الترحيل من D1/SQLite إلى Postgres أو من Postgres التقليدي إلى Postgres بدون خادم
- إعداد الاستعادة في الوقت المناسب (PITR) أو النسخ الاحتياطية لقاعدة البيانات
- مواجهة أخطاء مثل "استنفاذ تجمع الاتصال"، أو "اتصالات TCP غير مدعومة في حالة عدم وجود خادم"، أو "وضع ssl مطلوب"

الكلمات الرئيسية: نيون بوستجرس، @neondatabase/serverless، @vercel/postgres، بوستجرس بدون خادم، حافة بوستجرس، تفرع نيون، قاعدة بيانات فيرسيل، http بوستجرس، websocket بوستجرس، اتصال مجمع، رذاذ نيون، بريزما نيون، بوستجرس كلاود فلير، بوستجرس فيرسيل إيدج، علامة قالب SQL، نيونكتل، فروع قاعدة البيانات، استعادة نقطة زمنية، هجرات بوستجرس، SQL بدون خادم، قاعدة بيانات الحافة، واجهة برمجة تطبيقات النيون، Vercel SQL
