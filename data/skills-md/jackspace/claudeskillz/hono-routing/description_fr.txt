Cette compétence fournit des connaissances complètes pour créer des API de type sécurisé avec Hono, en se concentrant sur les modèles de routage, la composition des middlewares, la validation des demandes, les modèles client/serveur RPC, la gestion des erreurs et la gestion du contexte.

À utiliser lorsque : la création d'API avec Hono (n'importe quel environnement d'exécution), la configuration de la validation de demande avec les validateurs Zod/Valibot/Typia/ArkType, la création d'une communication client/serveur RPC sécurisée, l'implémentation d'un middleware personnalisé, la gestion des erreurs avec HTTPException, l'extension du contexte Hono avec des variables personnalisées ou la rencontre de problèmes d'inférence de type middleware, de confusion avec les hooks de validation ou de problèmes de performances RPC.

Mots clés : hono, routage hono, middleware hono, hono rpc, validateur hono, validateur zod, validateur valibot, API de type sécurisé, contexte hono, gestion des erreurs hono, HTTPException, c.req.valid, composition middleware, hooks hono, routes typées, client hono, réponse middleware non tapée, échec de la validation hono, inférence de type hono rpc
