Cette compétence fournit des connaissances complètes sur la création de sites Web statiques avec le générateur de sites statiques Hugo. Il doit être utilisé lors de la configuration de projets Hugo (blogs, sites de documentation, pages de destination, portefeuilles), de l'intégration de Tailwind CSS v4 pour un style personnalisé, de l'intégration de systèmes CMS sans tête (Sveltia CMS ou TinaCMS), du déploiement sur Cloudflare Workers avec des actifs statiques, de la configuration de thèmes et de modèles et de la prévention des erreurs de configuration Hugo courantes.

Utilisez cette compétence lorsque vous rencontrez ces scénarios : échafaudage de nouveaux sites Hugo, choix entre les éditions Hugo Extended et Standard, intégration de Tailwind CSS v4 avec Hugo Pipes, configuration des fichiers Hugo.yaml ou Hugo.toml, intégration de PaperMod ou d'autres thèmes via des sous-modules Git, configuration de Sveltia CMS ou TinaCMS pour la gestion de contenu, déploiement sur Cloudflare Workers ou Pages, dépannage de la configuration de baseURL, résolution des erreurs d'installation de thème, résolution des problèmes de format de premier plan (YAML vs TOML), empêchant les échecs de build liés à la date, configurant PostCSS avec Hugo ou configurant CI/CD avec GitHub Actions.
