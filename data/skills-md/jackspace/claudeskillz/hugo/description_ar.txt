توفر هذه المهارة معرفة شاملة لإنشاء مواقع ويب ثابتة باستخدام منشئ المواقع الثابتة من Hugo. يجب استخدامه عند إعداد مشاريع Hugo (المدونات ومواقع التوثيق والصفحات المقصودة والمحافظ)، ودمج Tailwind CSS v4 للتصميم المخصص، ودمج أنظمة CMS مقطوعة الرأس (Sveltia CMS أو TinaCMS)، والنشر إلى Cloudflare Workers باستخدام الأصول الثابتة، وتكوين السمات والقوالب، ومنع أخطاء إعداد Hugo الشائعة.

استخدم هذه المهارة عند مواجهة هذه السيناريوهات: دعم مواقع Hugo الجديدة، والاختيار بين إصدارات Hugo Extended وStandard، ودمج Tailwind CSS v4 مع Hugo Pipes، وتكوين ملفات Hugo.yaml أو Hugo.toml، ودمج PaperMod أو سمات أخرى عبر وحدات Git الفرعية، وإعداد Sveltia CMS أو TinaCMS لإدارة المحتوى، والنشر إلى عمال أو صفحات Cloudflare، واستكشاف أخطاء تكوين baseURL وإصلاحها، وحل المشكلات. أخطاء تثبيت القالب، أو إصلاح مشكلات تنسيق الواجهة الأمامية (YAML vs TOML)، أو منع فشل البناء المرتبط بالتاريخ، أو إعداد PostCSS باستخدام Hugo، أو إعداد CI/CD باستخدام GitHub Actions.

الكلمات الرئيسية: Hugo، Hugo-Extend، static-site-generator، ssg، go-templates، Papermod، goldmark، markdown، blog، document، docs-site، صفحة الهبوط، sveltia-cms، tina-cms، headless-cms، cloudflare-workers، Workers-static-assets، wrangler، Hugo-server، Hugo-build، frontmatter، yaml-frontmatter، toml-config،hugo-themes،hugo-modules، متعدد اللغات، i18n، github-actions، عدم تطابق الإصدار، خطأ baseurl، السمة غير موجودة، tailwind، tailwind-v4، tailwind-css، Hugo-pipes، postcss، css-framework، فائدة-css، Hugo-tailwind، تكامل الرياح الخلفية، أصول Hugo
