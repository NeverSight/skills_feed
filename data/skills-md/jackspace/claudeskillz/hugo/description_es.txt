Esta habilidad proporciona conocimientos completos para crear sitios web estáticos con el generador de sitios estáticos Hugo. Debe usarse al configurar proyectos Hugo (blogs, sitios de documentación, páginas de destino, portafolios), integrar Tailwind CSS v4 para un estilo personalizado, integrar sistemas CMS sin cabeza (Sveltia CMS o TinaCMS), implementar en Cloudflare Workers con activos estáticos, configurar temas y plantillas y prevenir errores comunes de configuración de Hugo.

Utilice esta habilidad cuando se encuentre con estos escenarios: creación de andamios para nuevos sitios de Hugo, elección entre las ediciones Hugo Extended y Standard, integración de Tailwind CSS v4 con Hugo Pipes, configuración de archivos hugo.yaml o hugo.toml, integración de PaperMod u otros temas a través de submódulos de Git, configuración de Sveltia CMS o TinaCMS para la gestión de contenidos, implementación en Cloudflare Workers o Pages, resolución de problemas de configuración de baseURL, resolución de errores de instalación de temas, corrección del formato de frontmatter problemas (YAML vs TOML), prevención de fallas de compilación relacionadas con la fecha, configuración de PostCSS con Hugo o configuración de CI/CD con GitHub Actions.
