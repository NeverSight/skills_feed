Cette compétence fournit des connaissances complètes sur le développement de plugins WordPress, couvrant les modèles de base, les meilleures pratiques de sécurité, les interactions avec les bases de données, les hooks/filtres, l'API de paramètres, les types de publication personnalisés, l'API REST et AJAX. Cette compétence doit être utilisée lors de la création de plugins WordPress, du dépannage de problèmes de sécurité, de la mise en œuvre de types de publication/taxonomies personnalisés, de la création d'interfaces d'administration ou de l'utilisation de la base de données WordPress.

À utiliser lorsque : création de nouveaux plugins WordPress, implémentation de nonces/sanitization/escape, utilisation de $wpdb et d'instructions préparées, création de pages d'API de paramètres, enregistrement de types de publication ou de taxonomies personnalisés, implémentation de points de terminaison d'API REST, gestion des requêtes AJAX, débogage des problèmes d'activation/désactivation de plugin, prévention des vulnérabilités d'injection SQL/XSS/CSRF.

Mots clés : développement de plugin wordpress, sécurité wordpress, hooks wordpress, filtres wordpress, base de données wordpress, préparation wpdb, sanitize_text_field, esc_html, wp_nonce, type de publication personnalisé, register_post_type, paramètres api, rest api, admin-ajax, injection wordpress sql, wordpress xss, wordpress csrf, en-tête de plugin, hook d'activation, hook de désactivation, normes de codage wordpress, architecture de plugin wordpress
