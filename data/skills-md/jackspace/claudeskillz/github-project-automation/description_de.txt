Dieser Skill bietet eine umfassende Automatisierung für die Einrichtung und Konfiguration des GitHub-Repositorys.
Es sollte beim Erstellen neuer Projekte, beim Einrichten von CI/CD-Pipelines und beim Konfigurieren von Problemen verwendet werden
Vorlagen erstellen, Sicherheitsscans aktivieren oder vorhandene Projekte zur GitHub-Automatisierung migrieren.

Der Skill verhindert 18 dokumentierte Fehler in der YAML-Syntax von GitHub Actions, der Workflow-Konfiguration,
Issue-Vorlagenstruktur, Dependabot-Einrichtung und CodeQL-Sicherheitsscan. Es umfasst 12
Produktionsgetestete Workflow-Vorlagen, 4 Problemvorlagen, Sicherheitskonfigurationen und Automatisierung
Skripte für eine schnelle Projekteinrichtung.

Verwendung bei: Einrichten von GitHub Actions CI/CD, Erstellen von Issue-/PR-Vorlagen, Aktivieren von Dependabot,
Konfigurieren des CodeQL-Scannens, Automatisieren der GitHub-Repository-Einrichtung, Beheben von YAML-Syntaxfehlern,
Integration von Sicherheitsscans, Bereitstellung für Cloudflare Workers über GitHub Actions oder
Implementierung von Multi-Framework-Testmatrizen.

Schlüsselwörter: Github-Aktionen, Github-Workflow, CI/CD, Issue-Vorlagen, Pull-Request-Vorlagen,
Dependabot, Codeql, Sicherheitsscan, Yaml-Syntax, Github-Automatisierung, Repository-Setup,
Workflow-Vorlagen, Github-Aktionsmatrix, Geheimnisverwaltung, Zweigschutz, Codeeigentümer,
Github-Projekte, kontinuierliche Integration, kontinuierliche Bereitstellung, Workflow-Syntaxfehler,
Fixierung der Aktionsversion, Runner-Version, Github-Kontext, Yaml-Einrückungsfehler
