Nutzen Sie diese Fähigkeit, wenn Sie Code zu Open-Source-Projekten beisteuern. Die Fertigkeit umfasst die ordnungsgemäße Erstellung von Pull-Anfragen, die Vermeidung häufiger Fehler, die Betreuer ärgern, das Bereinigen persönlicher Entwicklungsartefakte vor der Einreichung, das Verfassen effektiver PR-Beschreibungen, das Einhalten von Projektkonventionen und die professionelle Kommunikation mit Betreuern. Es verhindert 16 häufige Beitragsfehler, darunter das Arbeiten am Hauptzweig, das Nichttesten vor der PR-Einreichung, einschließlich nicht damit zusammenhängender Änderungen, das Einreichen von Planungsdokumenten, Sitzungsnotizen, temporären Testdateien, Screenshots und anderen persönlichen Artefakten. Enthält 3 wichtige Workflow-Regeln, die NIEMALS übersprungen werden dürfen: (1) Arbeiten Sie immer an Feature-Zweigen, (2) Testen Sie vor der PR gründlich anhand von Beweisen, (3) Konzentrieren Sie PRs auf ein einzelnes Feature. Der Skill umfasst Automatisierungsskripte zur Validierung von PRs vor der Einreichung, Vorlagen für PR-Beschreibungen und Commit-Nachrichten sowie umfassende Checklisten. Diese Fähigkeit sollte immer dann verwendet werden, wenn Pull-Requests für öffentliche Repositorys erstellt, zu Community-Projekten beigetragen oder Code an Projekte übermittelt werden, die Ihnen nicht gehören.
