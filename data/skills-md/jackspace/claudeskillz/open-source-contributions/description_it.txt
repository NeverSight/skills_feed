Utilizza questa competenza quando contribuisci con il codice a progetti open source. Le competenze riguardano la corretta creazione di pull request, l'evitare errori comuni che infastidiscono i manutentori, la pulizia degli artefatti dello sviluppo personale prima dell'invio, la scrittura di descrizioni PR efficaci, il rispetto delle convenzioni del progetto e la comunicazione professionale con i manutentori. Previene 16 errori comuni di contributo, tra cui lavorare sul ramo principale, non eseguire test prima dell'invio di PR, comprese modifiche non correlate, invio di documenti di pianificazione, note di sessione, file di test temporanei, screenshot e altri artefatti personali. Include 3 regole critiche del flusso di lavoro che non devono MAI essere ignorate: (1) Lavorare sempre sui rami delle funzionalità, (2) Testare accuratamente con prove prima del PR, (3) Mantenere i PR concentrati su una singola funzionalità. La competenza include script di automazione per convalidare le PR prima dell'invio, modelli per le descrizioni delle PR e i messaggi di commit e liste di controllo complete. Questa competenza dovrebbe essere utilizzata ogni volta che crei richieste pull per repository pubblici, contribuisci a progetti della comunità o invii codice a progetti di cui non sei proprietario.
