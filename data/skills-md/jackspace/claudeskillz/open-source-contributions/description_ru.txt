Используйте этот навык при написании кода для проектов с открытым исходным кодом. Навык охватывает правильное создание запросов на включение, избежание распространенных ошибок, которые раздражают сопровождающих, очистку артефактов личного развития перед отправкой, написание эффективных PR-описаний, соблюдение соглашений проекта и профессиональное общение с сопровождающими. Это предотвращает 16 распространенных ошибок при вкладе, включая работу над основной веткой, отсутствие тестирования перед отправкой запроса на участие, включая несвязанные изменения, отправку документов планирования, заметок о сеансах, временных тестовых файлов, снимков экрана и других личных артефактов. Включает 3 важных правила рабочего процесса, которые НИКОГДА нельзя пропускать: (1) Всегда работайте над ветвями функций, (2) Тщательно тестируйте доказательства перед PR, (3) Сосредоточьте PR на одной функции. Навык включает в себя сценарии автоматизации для проверки заявок на отправку перед отправкой, шаблоны для описаний заявок и сообщений о фиксации, а также подробные контрольные списки. Этот навык следует использовать всякий раз, когда вы создаете запросы на включение для общедоступных репозиториев, участвуете в проектах сообщества или отправляете код в проекты, которыми вы не владеете.
