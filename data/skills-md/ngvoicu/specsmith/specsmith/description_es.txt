Gestión de especificaciones estructurada para flujos de trabajo de codificación de IA. Convierte planes efímeros en especificaciones persistentes y reanudables con fases, tareas y seguimiento del progreso que sobreviven a lo largo de las sesiones. Utilice esta habilidad siempre que el usuario: salga del modo de plan (ofrezca automáticamente guardar el plan como una especificación), diga "reanudar" o "en qué estaba trabajando", quiera cambiar entre proyectos, mencione especificaciones/fases/tareas, diga "especificación nueva/lista/resume/estado/pausa/activar", diga "forjar", "investigar", "crear una especificación para X", "plan X", diga "generar openapi", "actualizar especificación de API", "crear API docs", "openapi" o cualquier flujo de trabajo que implique una planificación estructurada que deba persistir. También se activa cuando el usuario inicia una nueva sesión en un proyecto que tiene un directorio `.specs/`: verifique si hay una especificación activa y ofrezca reanudarla.
