Aplique una gestión segura de secretos en todas las plataformas. Nunca codifique secretos de OAuth2, claves API, tokens, contraseñas o credenciales en el código fuente. Almacene todos los secretos en archivos .env, cárguelos desde variables de entorno y asegúrese de que .env se ignore. Utilice esta habilidad cuando: (1) escriba cualquier código que utilice claves API, secretos de cliente OAuth2, tokens o credenciales, (2) configure autenticación o integraciones de terceros, (3) cree nuevos proyectos que necesiten configuración del entorno, (4) revise el código para problemas de seguridad relacionados con secretos, (5) configure canalizaciones de CI/CD o implementaciones de Docker con secretos. Activadores: clave API, OAuth, secreto de cliente, token, credenciales, .env, variables de entorno, secreto, contraseña, configuración de autenticación, integración de terceros.
