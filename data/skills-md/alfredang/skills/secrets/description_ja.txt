すべてのプラットフォームにわたって安全なシークレット管理を強制します。 OAuth2 シークレット、API キー、トークン、パスワード、または認証情報をソース コードにハードコーディングしないでください。すべてのシークレットを .env ファイルに保存し、環境変数からロードして、.env が gitignore されていることを確認します。このスキルは、(1) API キー、OAuth2 クライアント シークレット、トークン、または資格情報を使用するコードを作成する場合、(2) 認証またはサードパーティ統合をセットアップする場合、(3) 環境構成が必要な新しいプロジェクトを作成する場合、(4) シークレットに関連するセキュリティ問題についてコードをレビューする場合、(5) シークレットを使用して CI/CD パイプラインまたは Docker デプロイメントを構成する場合に使用します。トリガー: API キー、OAuth、クライアント シークレット、トークン、資格情報、.env、環境変数、シークレット、パスワード、認証セットアップ、サードパーティ統合。
