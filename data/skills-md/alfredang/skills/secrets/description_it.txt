Applica la gestione sicura dei segreti su tutte le piattaforme. Non codificare mai segreti OAuth2, chiavi API, token, password o credenziali nel codice sorgente. Archivia tutti i segreti nei file .env, caricali dalle variabili di ambiente e assicurati che .env sia gitignorato. Utilizzare questa competenza quando: (1) si scrive codice che utilizza chiavi API, segreti client OAuth2, token o credenziali, (2) si configura l'autenticazione o integrazioni di terze parti, (3) si creano nuovi progetti che richiedono la configurazione dell'ambiente, (4) si esamina il codice per problemi di sicurezza relativi ai segreti, (5) si configurano pipeline CI/CD o distribuzioni Docker con segreti. Trigger: chiave API, OAuth, segreto client, token, credenziali, .env, variabili di ambiente, segreto, password, configurazione di autenticazione, integrazione di terze parti.
