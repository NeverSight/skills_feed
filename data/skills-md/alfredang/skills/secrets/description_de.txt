Erzwingen Sie eine sichere Verwaltung von Geheimnissen auf allen Plattformen. Kodieren Sie OAuth2-Geheimnisse, API-Schlüssel, Token, Passwörter oder Anmeldeinformationen niemals fest im Quellcode. Speichern Sie alle Geheimnisse in .env-Dateien, laden Sie sie aus Umgebungsvariablen und stellen Sie sicher, dass .env gitignoriert wird. Verwenden Sie diese Fertigkeit, wenn: (1) Code geschrieben wird, der API-Schlüssel, OAuth2-Client-Geheimnisse, Token oder Anmeldeinformationen verwendet, (2) Authentifizierung oder Integrationen von Drittanbietern eingerichtet wird, (3) neue Projekte erstellt werden, die eine Umgebungskonfiguration erfordern, (4) Code auf Sicherheitsprobleme im Zusammenhang mit Geheimnissen überprüft wird, (5) CI/CD-Pipelines oder Docker-Bereitstellungen mit Geheimnissen konfiguriert werden. Auslöser: API-Schlüssel, OAuth, Client-Geheimnis, Token, Anmeldeinformationen, .env, Umgebungsvariablen, Geheimnis, Passwort, Authentifizierungseinrichtung, Integration von Drittanbietern.
