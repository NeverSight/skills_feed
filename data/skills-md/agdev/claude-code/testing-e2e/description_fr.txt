Conseils d’experts pour rédiger des tests résilients de bout en bout qui couvrent plusieurs processus et composants. Utilisez cette compétence lors de la révision, de la rédaction ou de la refactorisation de tests e2e à l’échelle du système. Couvre les scénarios destinés aux utilisateurs, les fonctions d'assistance, les usines de données, les sélecteurs basés sur ARIA et les assertions auto-récupérables. PAS pour les tests unitaires/d'intégration - utilisez plutôt la compétence de test d'intégration d'unité. Voir references/playwright-resilience.md pour les modèles de sélecteur détaillés.
