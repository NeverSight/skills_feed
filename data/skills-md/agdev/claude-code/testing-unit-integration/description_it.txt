Guida esperta per la scrittura di test di unità, integrazione, componenti, microservizi e API puliti, semplici ed efficaci. Utilizzare questa capacità durante la revisione dei test esistenti per eventuali violazioni, la scrittura di nuovi test o il refactoring dei test. NON per test end-to-end che si estendono su più processi: utilizza invece la competenza testing-e2e. Copre il modello AAA, le data factory, le strategie di mocking, i test DOM, i test dei database e le migliori pratiche di asserzione.
