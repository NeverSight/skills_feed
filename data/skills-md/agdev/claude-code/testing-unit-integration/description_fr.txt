Conseils d'experts pour rédiger des tests unitaires, d'intégration, de composants, de microservices et d'API propres, simples et efficaces. Utilisez cette compétence lorsque vous examinez les tests existants pour détecter les violations, rédigez de nouveaux tests ou refactorisez des tests. PAS pour les tests de bout en bout qui couvrent plusieurs processus - utilisez plutôt la compétence testing-e2e. Couvre le modèle AAA, les usines de données, les stratégies de simulation, les tests DOM, les tests de bases de données et les meilleures pratiques d'assertion.
