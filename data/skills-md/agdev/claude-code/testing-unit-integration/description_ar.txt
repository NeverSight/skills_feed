إرشادات الخبراء لكتابة اختبارات الوحدة والتكامل والمكونات والخدمات الصغيرة وواجهة برمجة التطبيقات (API) النظيفة والبسيطة والفعالة. استخدم هذه المهارة عند مراجعة الاختبارات الموجودة بحثًا عن الانتهاكات، أو كتابة اختبارات جديدة، أو إعادة هيكلة الاختبارات. ليس للاختبارات الشاملة التي تشمل عمليات متعددة - استخدم مهارة اختبار e2e بدلاً من ذلك. يغطي نمط AAA، ومصانع البيانات، واستراتيجيات السخرية، واختبار DOM، واختبار قاعدة البيانات، وأفضل ممارسات التأكيد.
