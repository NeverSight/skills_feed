Garde-corps critique pour les opérations de base de données Supabase garantissant une isolation multi-tenant avec le filtrage publication_id, une utilisation appropriée de supabaseAdmin, évitant SELECT *, les modèles de gestion des erreurs et un accès sécurisé à la base de données côté serveur. À utiliser lors de l'écriture de requêtes de base de données, de l'utilisation de supabase, de l'accès à des newsletter_campaigns, des articles, des rss_posts ou à toute donnée relative au client.
