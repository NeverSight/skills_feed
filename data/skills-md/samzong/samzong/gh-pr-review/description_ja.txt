「gh」 CLI を使用して、責任あるプロジェクト所有者として GitHub プル リクエストを確認します。ユーザーが PR URL (例: https://github.com/ORG/REPO/pull/N) または現在の git リポジトリの PR 番号 (アップストリームを優先、そうでない場合はオリジン) を提供し、中国語で書かれ、コピー＆ペースト可能な GitHub コメントが英語で書かれたオーナーグレードのレビュー ドキュメント `review-N.md` を必要とする場合に使用します。レビューの範囲を PR によって変更された行に絞ります (無関係な既存のコードを細かく指摘しないでください)。ただし、ベスト プラクティスを適用し、変更によって引き起こされた明らかなバグ、セキュリティ上の問題、または CI の障害にフラグを立てます。
