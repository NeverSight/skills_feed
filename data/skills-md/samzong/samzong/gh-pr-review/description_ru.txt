Просмотрите запрос на извлечение GitHub как ответственный владелец проекта, используя CLI `gh`. Используйте, когда пользователь предоставляет URL-адрес PR (например, https://github.com/ORG/REPO/pull/N) или номер PR для текущего репозитория git (предпочитает восходящий поток, иначе источник) и хочет получить документ проверки уровня владельца `review-N.md`, написанный на китайском языке с возможностью копирования комментариев GitHub на английском языке. Охватите обзор строками, измененными PR (не придирайтесь к несвязанному ранее существующему коду), но применяйте лучшие практики и отмечайте любые явные ошибки, проблемы безопасности или сбои CI, вызванные изменением.
