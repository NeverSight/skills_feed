Examinez une Pull Request GitHub en tant que propriétaire de projet responsable à l'aide de la CLI `gh`. À utiliser lorsque l'utilisateur fournit une URL PR (par exemple https://github.com/ORG/REPO/pull/N) ou un numéro PR pour le dépôt git actuel (préférez l'amont, sinon l'origine) et souhaite un document de révision de niveau propriétaire « review-N.md » écrit en chinois avec des commentaires GitHub copiables en anglais. Étendez l'examen aux lignes modifiées par le PR (ne pinaillez pas le code préexistant sans rapport), mais appliquez les meilleures pratiques et signalez tous les bogues évidents, les problèmes de sécurité ou les échecs de CI causés par le changement.
