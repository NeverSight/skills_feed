Esamina una richiesta pull di GitHub come proprietario responsabile del progetto utilizzando la CLI `gh`. Da utilizzare quando l'utente fornisce un URL PR (ad esempio https://github.com/ORG/REPO/pull/N) o un numero PR per il repository git corrente (preferire upstream, altrimenti origine) e desidera un documento di revisione di livello proprietario "review-N.md" scritto in cinese con commenti GitHub copiabili e incollabili in inglese. Limita la revisione alle righe modificate dal PR (non selezionare il codice preesistente non correlato), ma applica le migliori pratiche e segnala eventuali bug evidenti, problemi di sicurezza o errori CI causati dalla modifica.
