Überprüfen Sie einen GitHub-Pull-Request als verantwortlicher Projekteigentümer mit der „gh“-CLI. Verwenden Sie diese Option, wenn der Benutzer eine PR-URL (z. B. https://github.com/ORG/REPO/pull/N) oder eine PR-Nummer für das aktuelle Git-Repo angibt (bevorzugt Upstream, sonst Herkunft) und ein auf Chinesisch verfasstes, für den Eigentümer geeignetes Überprüfungsdokument „review-N.md“ mit kopier- und einfügbaren GitHub-Kommentaren auf Englisch möchte. Beschränken Sie die Überprüfung auf Zeilen, die vom PR geändert wurden (wählen Sie keinen nicht verwandten, bereits vorhandenen Code aus), sondern wenden Sie Best Practices an und kennzeichnen Sie alle offensichtlichen Fehler, Sicherheitsprobleme oder CI-Fehler, die durch die Änderung verursacht wurden.
