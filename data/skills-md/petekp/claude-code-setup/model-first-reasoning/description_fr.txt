Appliquez le raisonnement basé sur le modèle (MFR) aux tâches de génération de code. À utiliser lorsque l'utilisateur demande "modèle d'abord", "MFR", "modélisation formelle avant codage", "modèle puis implémentation", ou lorsque les tâches impliquent une logique complexe, des machines à états, des systèmes de contraintes ou toute implémentation nécessitant des garanties formelles d'exactitude. Impose une séparation stricte entre les phases de modélisation et de mise en œuvre.
