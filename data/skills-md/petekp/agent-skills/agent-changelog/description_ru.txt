Составьте оптимизированный для агента журнал изменений, сопоставляя историю git с планами и документацией. Используйте, когда вас просят «обновить журнал изменений», «собрать историю», «документировать развитие проекта» или заранее после важных этапов, архитектурных изменений или при обнаружении устаревшей/устаревшей информации, которая может сбить с толку агентов по кодированию.
