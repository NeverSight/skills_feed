Prepara i rami delle richieste pull memorizzando le modifiche, creando rami di funzionalità dal principale, rivedendo le modifiche, eseguendo controlli di qualità del codice solo sui file modificati, generando messaggi di commit convenzionali ed effettuando il push. Da utilizzare durante la preparazione di un PR, la creazione di un ramo per una richiesta pull o il commit di modifiche in seguito a commit convenzionali.
