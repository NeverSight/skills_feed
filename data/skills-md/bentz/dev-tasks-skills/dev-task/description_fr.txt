Cette compétence guide le « processus de développement d'exigences complexes », en mettant l'accent sur les étapes abstraites et les méthodes de mise en œuvre réutilisables plutôt que sur l'analyse post-mortem d'une exigence spécifique. Il permet de clarifier les objectifs dans les conversations à plusieurs tours, d'organiser les tâches et les tâches pendant l'exécution, et de terminer les tests et la gestion des cas extrêmes dans la phase finale.
