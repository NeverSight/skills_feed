Modèles Go idiomatiques pour les erreurs, les interfaces, la concurrence et les packages. À utiliser lorsque :
- Rédaction ou révision du code Go
- Concevoir des interfaces ou la structure d'un package
- Implémentation de modèles de concurrence
- Gestion des erreurs et propagation du contexte
- Structuration de projets Go
Mots-clés : Go, golang, emballage d'erreurs, conception d'interface, goroutine, canal,
contexte, conception de package, injection de dépendances, condition de concurrence critique
