Principes de conception, gestion des versions et documentation de l'API REST. À utiliser lorsque :
- Conception de nouveaux points de terminaison API
- Choisir entre REST, GraphQL ou gRPC
- Implémentation du versioning API
- Rédaction des spécifications OpenAPI
- Gestion des erreurs API
Mots clés : REST, API, OpenAPI, Swagger, versioning, méthodes HTTP,
codes d'état, pagination, gestion des erreurs
