로컬 개발 시 Stripe webhook 400 오류를 수정합니다. 문제: `스트라이프 리슨`
시작할 때마다 새로운 비밀을 생성하지만 앱은 오래된 비밀을 사용합니다.
.env.local. 이 기술은 개발 서버가 시작되기 전에 임시 비밀을 자동 동기화합니다.

자동 호출: 스트라이프 웹후크가 400을 반환하고 "서명 확인 실패",
"일치하는 서명을 찾을 수 없습니다.", 웹후크가 로컬에서 작동하지 않고, 체크아웃이 성공합니다.
하지만 구독이 업데이트되지 않습니다. STRIPE_WEBHOOK_SECRET 불일치, 설정 중
스트라이프 청취, 스트라이프로 pnpm dev 구성.
