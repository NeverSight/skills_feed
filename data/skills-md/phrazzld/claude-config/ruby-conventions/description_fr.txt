Meilleures pratiques Ruby et Rails pour les services, les tests et l'architecture. À utiliser lorsque :
- Rédaction ou révision de code Ruby/Rails
- Concevoir des objets de service ou organiser la logique métier
- Mise en place des tests RSpec avec FactoryBot
- Création d'API avec sérialisation JSON
- Optimisation des performances Rails (N+1, cache)
Mots clés : Ruby, Rails, RSpec, FactoryBot, objet de service, paramètres forts,
Requête N+1, Sidekiq, RuboCop, Thor CLI
