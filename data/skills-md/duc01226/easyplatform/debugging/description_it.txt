Framework di debug sistematico per l'indagine delle cause principali prima delle correzioni. Utilizzare per bug, errori di test, comportamenti imprevisti, problemi di prestazioni. Utilizza il flag `--autonomous` per il debug headless strutturato con gate di approvazione. Trigger: debug, bug, errore, correzione, diagnosi, causa principale, analisi dello stack, investigazione del problema. NON per: revisione del codice (utilizzare la revisione del codice), semplificazione (utilizzare il semplificatore di codice).
