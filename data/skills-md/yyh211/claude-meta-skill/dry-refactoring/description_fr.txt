Guide la refactorisation systématique du code selon le principe DRY (Don't Repeat Yourself). À utiliser lorsque l'utilisateur demande d'éliminer la duplication de code, de refactoriser le code répétitif, d'appliquer le principe DRY ou de mentionner que le code sent le copier-coller, les nombres magiques ou la logique répétée. Implémente un flux de travail en 4 étapes depuis l'identification de la répétition jusqu'à la refactorisation vérifiée.
