Leitet systematisches Code-Refactoring nach dem DRY-Prinzip (Don't Repeat Yourself) an. Verwenden Sie diese Option, wenn der Benutzer darum bittet, Codeduplizierung zu beseitigen, sich wiederholenden Code umzugestalten, das DRY-Prinzip anzuwenden oder Code erwähnt, der nach Kopieren und Einfügen, magischen Zahlen oder wiederholter Logik riecht. Implementiert einen 4-stufigen Workflow von der Identifizierung von Wiederholungen bis zur verifizierten Umgestaltung.
