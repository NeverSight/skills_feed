Debug di composizioni, funzioni (KCL, modelli Go, patch e trasformazione, predisposizione automatica) e risorse gestite del crossplane. Da utilizzare per la risoluzione di problemi di rendering della composizione, errori di funzione, errori di creazione di risorse, problemi di dipendenza o problemi di stato delle rivendicazioni. Supporta il debug del cluster remoto quando i file di composizione non sono disponibili localmente. Si attiva su parole chiave come "crossplane", "composition", "XR", "claim", "function-kcl", "risorsa gestita" o durante il debug delle risorse Kubernetes create da Crossplane.
