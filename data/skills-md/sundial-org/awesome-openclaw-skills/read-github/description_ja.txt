GitHub リポジトリを正しい方法で読み取ります - 生のスクレイピングではなく gitmcp.io を使用します。これが Web 検索に勝る理由: (1) 単なるキーワード マッチングではなくドキュメント全体のセマンティック検索、(2) 正確なファイル構造によるスマートなコード ナビゲーション - リポジトリ レイアウトでの幻覚ゼロ、(3) 生の HTML/JSON ガベージではなく、LLM に最適化された適切なマークダウン出力、(4) README + /docs + コードを 1 つのクリーンなインターフェイスに集約、(5) レート制限と robots.txt を尊重。生の GitHub URL を貼り付けるのはやめて、代わりにこれを使用してください。
