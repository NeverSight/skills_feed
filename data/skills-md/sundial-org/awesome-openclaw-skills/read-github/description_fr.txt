Lisez les dépôts GitHub de la BONNE manière - via gitmcp.io au lieu du scraping brut. Pourquoi cela surpasse la recherche sur le Web : (1) Recherche sémantique dans les documents, pas seulement la correspondance de mots clés, (2) Navigation de code intelligente avec une structure de fichier précise - zéro hallucinations sur la mise en page du dépôt, (3) Sortie de démarque appropriée optimisée pour les LLM, pas de déchets HTML/JSON bruts, (4) Agrége le code README + /docs + dans une interface propre, (5) Respecte les limites de débit et le fichier robots.txt. Arrêtez de coller les URL GitHub brutes - utilisez-la à la place.
