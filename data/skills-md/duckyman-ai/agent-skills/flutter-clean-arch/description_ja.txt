機能優先構造のクリーン アーキテクチャ、Riverpod 状態管理、ネットワーク用の Dio + Retrofit、および fpdart エラー処理を使用して Flutter アプリケーションを生成します。このスキルは、Flutter アプリの作成、クリーンなアーキテクチャ パターンでの機能の実装、Riverpod プロバイダーのセットアップ、機能エラー処理のためのいずれかのタイプでのデータの処理、タイプセーフな API クライアントでの HTTP リクエストの作成、またはドメイン/データ/プレゼンテーション層を使用したプロジェクトの構築を行うときに使用します。トリガーには、「Flutter アプリ」、「クリーン アーキテクチャ」、「Riverpod」、「機能ファースト」、「状態管理」、「API クライアント」、「Retrofit」、「Dio」、「REST API」、または懸念を分離して Flutter 機能を構築するリクエストが含まれます。
