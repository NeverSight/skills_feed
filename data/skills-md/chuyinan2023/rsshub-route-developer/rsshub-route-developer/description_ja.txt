任意の Web サイト URL から RSSHub ルートを開発します。ユーザーが Web サイトの RSS フィードを作成したり、RSSHub ルートを構築したり、RSSHub プロジェクトにルート PR を送信したりする場合に使用します。完全なワークフローを処理します: ターゲット ページの分析、ルート コード (namespace.ts、ルート ハンドラー、rader.ts) の生成、ローカル開発サーバーの実行とテスト、フォーマット/lint、DIYgod/RSSHub への PR の送信。ユーザーが RSSHub ルート開発パターン、ルートの問題のデバッグ、または RSSHub プロジェクトの規則について質問する場合にも使用します。
