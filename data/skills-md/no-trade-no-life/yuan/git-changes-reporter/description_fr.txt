Générez des rapports de modifications Git structurés (JSON + Markdown). Utilisez cette compétence lorsque l'utilisateur mentionne « modifications git », « résumé de validation », « révision du code », « note de version », « modifications récentes », « résumé quotidien » ou a besoin d'analyser les modifications de code dans une plage de validation spécifiée. Comprend un rapport sémantique à triple structure (intention de conception, code principal, portée de l'impact), adapté aux révisions de code, aux notes de version, aux synchronisations d'équipe, au CI/CD et à d'autres scénarios.
