Genera report strutturati sulle modifiche git (JSON + Markdown). Utilizza questa abilità quando l'utente menziona "modifiche git", "riepilogo del commit", "revisione del codice", "nota di rilascio", "modifiche recenti", "riepilogo giornaliero" o ha bisogno di analizzare le modifiche del codice entro un intervallo di commit specificato. Include un report semantico a tripla struttura (intento di progettazione, codice principale, ambito di impatto), adatto per revisioni del codice, note di rilascio, sincronizzazioni del team, CI/CD e altri scenari.
