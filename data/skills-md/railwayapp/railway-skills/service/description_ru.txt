Этот навык следует использовать, когда пользователь спрашивает о состоянии службы, хочет переименовать службу, изменить значки служб, связать службы или создать службы с изображениями Docker. Для создания сервисов с использованием локального кода отдайте предпочтение навыку «новый». Для источников репозитория GitHub используйте навык «new», чтобы создать пустой сервис, а затем навык «environment», чтобы настроить источник.
