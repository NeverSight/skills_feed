nixomatic を使用して、プロジェクト README.md で開発環境の指示を維持します。ユーザーがプロジェクトのビルド、コンパイル、テスト、lint、フォーマット、型チェック、実行、またはセットアップを要求した場合、またはツールまたはランタイムがインストールされていないためにコマンドが失敗した場合は、プロジェクト ファイルを分析して必要なパッケージを特定し、nixomatic.com URL で nix 開発を使用します。 nix Development と docker run nixos/nix の両方を使用して環境を再現する方法が README.md に記載されていることを確認してください。
