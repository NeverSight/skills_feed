Проверьте незафиксированные изменения Git на предмет правильности, качества и соответствия соглашениям проекта, затем внесите исправления и подготовьте безопасный атомарный план фиксации. Используйте, когда пользователи просят просмотреть код перед фиксацией, улучшить локальные изменения, разделить работу на логические обычные фиксации или выполнить `git add`/`git commit` с четкими границами промежуточного хранения.
