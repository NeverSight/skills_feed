قم بمراجعة تغييرات Git غير الملتزم بها للتأكد من صحتها وجودتها ومواءمة اتفاقية المشروع، ثم قم بتطبيق الإصلاحات وإعداد خطة التزام ذرية آمنة. يُستخدم عندما يطلب المستخدمون مراجعة التعليمات البرمجية قبل الالتزام، أو تحسين التغييرات المحلية، أو تقسيم العمل إلى التزامات تقليدية منطقية، أو تنفيذ `git add`/`git Commit` بحدود مرحلية واضحة.
