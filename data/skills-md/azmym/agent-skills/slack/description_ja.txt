Web API を介して Slack と対話します。メッセージの読み取り、要約、検索、投稿、反応、ピン留め、チャネルの管理を行います。ユーザーが (1) Slack URL を共有する場合、(2) チャネルの読み取りまたは概要を要求する場合、(3) Slack メッセージを検索する場合、(4) メッセージの送信/投稿を要求する場合、(5) メッセージへの反応またはピン留めを要求する場合、(6) ユーザーを検索する場合、または (7) Slack チャネルを名前で言及する場合 (例: "#channel-name") に使用します。また、Slack スレッド、毎日のスタンドアップ、会話ダイジェスト、またはあらゆる Slack インタラクションのトリガーにもなります。
