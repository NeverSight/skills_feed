CLI pour déboguer les processus backend Node.js avec inspection non bloquante. À utiliser lorsque l'utilisateur doit se connecter aux processus Node.js (par PID, nom, Docker ou port), définir des points de trace/points de journalisation/points d'exception, capturer des piles d'appels et des variables locales, exécuter JavaScript dans le contexte du processus ou inspecter les journaux de la console. Nécessite un démon ; connectez-vous avant les autres commandes de débogage.
