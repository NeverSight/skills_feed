Protocolo de pruebas estricto. Hacer cumplir el principio de "si no se prueba, no se hace".

Momentos desencadenantes:
- Al corregir un error (debe agregar prueba de regresión)
- Al implementar nuevas funcionalidades (debe contar con las pruebas correspondientes)
- Al modificar el código existente (asegúrese de que las pruebas aún pasen)
- Cuando el usuario menciona "probar", "probar" o "verificar funcionalidad"
