Protocollo di test rigoroso. Applicare il principio “Se non è testato, non è fatto”.

Momenti scatenanti:
- Quando si corregge un bug (è necessario aggiungere il test di regressione)
- Quando si implementano nuove funzionalità (devono avere test corrispondenti)
- Quando si modifica il codice esistente (assicurarsi che i test continuino a passare)
- Quando l'utente menziona "test", "test" o "verifica funzionalità"
