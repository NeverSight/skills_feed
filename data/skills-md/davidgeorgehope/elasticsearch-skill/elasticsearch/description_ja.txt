Curl を使用して REST API 経由で Elasticsearch および Kibana と対話します。クエリ、インデックス作成、インデックスの管理、クラスターの健全性のチェック、集計の書き込み、ダッシュボードのデプロイ、または Elasticsearch のトラブルシューティングを行うときに使用します。クラスター URL と API キーが必要です。対象範囲: 検索 (クエリ DSL)、CRUD 操作、インデックス管理、マッピング、集計、クラスターの健全性、ILM、ES|QL、Kibana API (ダッシュボード、データ ビュー、保存されたオブジェクト)、OpenTelemetry データ パターン、および一般的なトラブルシューティング パターン。
