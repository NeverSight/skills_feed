Verwenden Sie es, wenn Sie eine Softwarearchitektur entwerfen, die Codestruktur umgestalten, wiederkehrende Designprobleme lösen oder wenn Code Symptome wie enge Kopplung, starre Hierarchien, verstreute Verantwortlichkeiten oder schwer zu testende Komponenten aufweist. Verwenden Sie es auch, wenn Sie zwischen Architekturansätzen wählen oder Code auf strukturelle Verbesserungen überprüfen.
