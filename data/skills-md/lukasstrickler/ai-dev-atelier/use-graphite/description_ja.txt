git Push/gh pr create の代わりに Graphite CLI (gt) を使用してスタックされた PR を管理します。 Graphite リポジトリを自動検出し、競合するコマンドを有用な代替手段でブロックします。次の場合に使用します: (1) Graphite リポジトリで git Push または gh pr create を実行しようとしているとき、(2) 機能の新しいブランチを作成するとき、(3) レビューのためにコードを送信するとき、(4) レビュー可能なチャンクに分割する必要がある大きな変更、(5) フックが git コマンドをブロックし、gt と同等のコマンドを提案するとき。対象外: Graphite で初期化されていないリポジトリ、git add/commit/status/log。トリガー: git プッシュのブロック、gh pr create のブロック、ブランチの作成、PR の送信、スタックされた PR、大規模な PR の分割、gt create、gt submit、グラファイト ワークフロー。
