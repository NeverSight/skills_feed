Workflow de résolution de problèmes GitHub de bout en bout à l'aide de gh, des modifications de code local, des builds/tests et de git push. À utiliser lorsqu'on vous demande de prendre un numéro de problème, d'inspecter le problème via gh, d'implémenter un correctif, d'exécuter des builds/tests XcodeBuildMCP, de valider avec un message de clôture et de pousser.
