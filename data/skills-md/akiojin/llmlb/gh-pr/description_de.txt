Erstellen oder aktualisieren Sie GitHub-Pull-Requests mit der gh-CLI, einschließlich der Entscheidung, ob ein neuer PR erstellt oder nur ein Push basierend auf dem bestehenden PR-Mergestatus durchgeführt werden soll. Verwenden Sie diese Option, wenn der Benutzer eine PR öffnen/erstellen/bearbeiten, einen PR-Text/eine PR-Vorlage generieren möchte oder „PRを出して/PR作成/gh pr“ sagt. Standardeinstellungen: base=develop, head=aktueller Zweig (nur derselbe Zweig; niemals Zweige erstellen/wechseln).
