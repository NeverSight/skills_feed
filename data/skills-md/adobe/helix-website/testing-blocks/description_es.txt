Guía para probar cambios de código en proyectos de AEM Edge Delivery, incluidos bloques, scripts y estilos. Utilice esta habilidad después de realizar cambios en el código y antes de abrir una solicitud de extracción para validar la funcionalidad. Cubre pruebas unitarias para utilidades y lógica, pruebas de navegador con Playwright/Puppeteer, linting, validación de rendimiento y orientación sobre qué pruebas mantener o usar como validación desechable.
