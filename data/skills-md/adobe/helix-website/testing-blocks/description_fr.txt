Guide pour tester les modifications de code dans les projets AEM Edge Delivery, y compris les blocs, les scripts et les styles. Utilisez cette compétence après avoir apporté des modifications au code et avant d'ouvrir une pull request pour valider la fonctionnalité. Couvre les tests unitaires pour les utilitaires et la logique, les tests de navigateur avec Playwright/Puppeteer, le peluchage, la validation des performances et des conseils sur les tests à maintenir ou à utiliser comme validation jetable.
