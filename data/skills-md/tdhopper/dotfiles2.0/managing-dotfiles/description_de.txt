Verwenden Sie diese Fähigkeit, wenn Sie mit persönlichen Dotfiles arbeiten, die von yadm verwaltet werden. Dazu gehören das Abrufen von Remote-Änderungen, das Festschreiben und Übertragen von Dotfile-Änderungen, das Ändern von Konfigurationsdateien (Shell, Editor, Terminal, Git usw.), das Anzeigen nachverfolgter Dateien, das Lösen von Zusammenführungskonflikten und die Pflege des Dotfiles-Repositorys. Verwenden Sie für Arbeitspunktdateien stattdessen den Skill „managing-work-dotfiles“.
