Standardizza tutte le interazioni GitHub tramite la CLI GitHub (`gh`) anziché URL ad hoc, clic sull'interfaccia utente o chiamate dirette all'API REST. Da utilizzare quando è necessario leggere o modificare lo stato di GitHub (repos, problemi, richieste pull, revisioni, verifica dello stato, flussi di lavoro/esecuzioni di azioni, versioni, etichette, traguardi, discussioni, nozioni) e si desidera un output deterministico (preferire `--json` + `--jq`). Da utilizzare anche quando l'utente fornisce un URL GitHub, inclusi link diretti come `https://github.com/OWNER/REPO/pull/123`, `.../issues/123`, `.../pull/123/files` o permalink di commento come `#issuecomment-...` e devi recuperare il PR/problema/thread sottostante e rispondere. Torna a "gh api" solo quando non esiste un comando "gh NOUN VERB" di prima classe.
