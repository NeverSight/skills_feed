Standardisez toutes les interactions GitHub via la CLI GitHub (`gh`) au lieu d'URL ad hoc, de clics sur l'interface utilisateur ou d'appels directs d'API REST. À utiliser lorsque vous devez lire ou modifier l'état de GitHub (dépôts, problèmes, demandes d'extraction, révisions, vérifier l'état, workflows/exécutions d'actions, versions, étiquettes, jalons, discussions, points essentiels) et que vous souhaitez une sortie déterministe (préférez `--json` + `--jq`). À utiliser également lorsque l'utilisateur fournit une URL GitHub, y compris des liens profonds comme `https://github.com/OWNER/REPO/pull/123`, `.../issues/123`, `.../pull/123/files`, ou des liens permanents de commentaires comme `#issuecomment-...`, et que vous devez récupérer le PR/issue/thread sous-jacent et y répondre. Revenez à `gh api` uniquement lorsqu'il n'y a pas de commande `gh NOUN VERB` de première classe.
