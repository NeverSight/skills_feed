アドホック URL、UI クリック、または直接 REST API 呼び出しではなく、GitHub CLI (「gh」) を介してすべての GitHub インタラクションを標準化します。 GitHub の状態 (リポジトリ、問題、プル リクエスト、レビュー、ステータスの確認、アクション ワークフロー/実行、リリース、ラベル、マイルストーン、ディスカッション、要点) を読み取りまたは変更する必要があり、確定的な出力が必要な場合に使用します (`--json` + `--jq` を推奨します)。また、ユーザーが「https://github.com/OWNER/REPO/pull/123」、「.../issues/123」、「.../pull/123/files」などのディープリンク、または「#issuecomment-...」などのコメントパーマリンクを含む GitHub URL を提供し、基になる PR/issue/thread を取得して返信する必要がある場合にも使用します。第一級の `gh NOUN VERB` コマンドがない場合にのみ、`gh api` にフォールバックします。
