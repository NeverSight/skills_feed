Стандартизируйте все взаимодействия с GitHub через GitHub CLI («gh») вместо специальных URL-адресов, кликов по пользовательскому интерфейсу или прямых вызовов REST API. Используйте, когда вам нужно прочитать или изменить состояние GitHub (репозитории, проблемы, запросы на включение, обзоры, статус проверки, рабочие процессы/запуски действий, выпуски, метки, этапы, обсуждения, суть) и вам нужен детерминированный вывод (предпочитайте `--json` + `--jq`). Также используйте, когда пользователь предоставляет URL-адрес GitHub, включая глубокие ссылки, такие как `https://github.com/OWNER/REPO/pull/123`, `.../issues/123`, `.../pull/123/files`, или постоянные ссылки комментариев, такие как `#issuecomment-...`, и вам нужно получить базовый PR/issue/thread и ответить. Возвращайтесь к `gh api` только в том случае, если нет первоклассной команды `gh NOUN VERB`.
