Estandarice todas las interacciones de GitHub a través de la CLI de GitHub (`gh`) en lugar de URL ad hoc, clics en la interfaz de usuario o llamadas directas a la API REST. Úselo cuando necesite leer o cambiar el estado de GitHub (repositorios, problemas, solicitudes de extracción, revisiones, verificar estado, flujos de trabajo/ejecuciones de acciones, lanzamientos, etiquetas, hitos, discusiones, esencias) y desee una salida determinista (prefiera `--json` + `--jq`). Úselo también cuando el usuario proporciona una URL de GitHub, incluidos enlaces profundos como `https://github.com/OWNER/REPO/pull/123`, `.../issues/123`, `.../pull/123/files`, o enlaces permanentes de comentarios como `#issuecomment-...`, y necesita recuperar el PR/issue/thread subyacente y responder. Recurra a `gh api` solo cuando no haya un comando `gh NOUN VERB` de primera clase.
