قم بتوحيد جميع تفاعلات GitHub عبر GitHub CLI (`gh`) بدلاً من عناوين URL المخصصة أو نقرات واجهة المستخدم أو استدعاءات REST API المباشرة. استخدمه عندما تحتاج إلى قراءة حالة GitHub أو تغييرها (الاتفاقيات، المشكلات، طلبات السحب، المراجعات، حالة التحقق، سير عمل/تشغيل الإجراءات، الإصدارات، التسميات، المعالم، المناقشات، العناصر) وتريد مخرجات حتمية (يفضل `--json` + `--jq`). يُستخدم أيضًا عندما يقدم المستخدم عنوان URL لـ GitHub، بما في ذلك الروابط العميقة مثل `https://github.com/OWNER/REPO/pull/123`، `.../issues/123`، `.../pull/123/files`، أو الروابط الدائمة للتعليق مثل `#issuecomment-...`، وتحتاج إلى جلب PR/issue/thread الأساسي والرد. ارجع إلى `gh api` فقط في حالة عدم وجود أمر `gh NOUN VERB` من الدرجة الأولى.
