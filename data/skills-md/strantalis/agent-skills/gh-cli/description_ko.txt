임시 URL, UI 클릭 또는 직접 REST API 호출 대신 GitHub CLI('gh')를 통해 모든 GitHub 상호 작용을 표준화합니다. GitHub 상태(저장소, 문제, 끌어오기 요청, 검토, 상태 확인, 작업 워크플로/실행, 릴리스, 레이블, 마일스톤, 토론, 요점)를 읽거나 변경해야 하고 결정적인 출력을 원할 때 사용합니다(`--json` + `--jq` 선호). 또한 사용자가 `https://github.com/OWNER/REPO/pull/123`, `.../issues/123`, `.../pull/123/files`와 같은 딥 링크 또는 `#issuecomment-...`와 같은 댓글 영구 링크를 포함하여 GitHub URL을 제공할 때 사용하고 기본 PR/문제/스레드를 가져와 응답해야 합니다. 일류 `gh NOUN VERB` 명령이 없는 경우에만 `gh api`로 대체됩니다.
