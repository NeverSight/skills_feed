Automatically generate a HarmonyOS code framework from design documents, including IDL interface definitions, directory structure, header files, source file templates, and build configurations. Before generation, it analyzes existing OpenHarmony code to learn current code style and architectural norms. Suitable for user requests: (1) generate code framework, (2) create IDL interfaces, (3) generate BUILD.gn configurations, (4) create service code, (5) initialize project structure, (6) reference existing code style. Keywords: code generation, IDL, BUILD.gn, bundle.json, SA profile, OpenHarmony code style, code generation, interface definition, build configuration, OH code style
