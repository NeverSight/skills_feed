Compétence spécialisée pour la révision du code de sécurité dans les systèmes distribués OpenHarmony. Déclenché lorsque l'utilisateur demande une « révision de la mise en œuvre de la sécurité du code », une « révision de la sécurité du code », une « révision du code de sécurité » ou des demandes similaires de révision de la sécurité du code d'un système distribué. Cette compétence fournit des conseils de révision détaillés pour 18 règles de conception de sécurité d'entreprise distribuée OpenHarmony, couvrant le contrôle des autorisations, les machines à états, le transfert de données, la gestion des autorisations, les relations de confiance et d'autres domaines de sécurité. Utilisez cette compétence pour effectuer des examens de sécurité ciblés pour les systèmes distribués OpenHarmony en plus des règles générales de cybersécurité.
