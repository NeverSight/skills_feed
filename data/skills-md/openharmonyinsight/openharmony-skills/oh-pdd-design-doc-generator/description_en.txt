Automatically generate HarmonyOS design documents—including architecture and functional design documents—based on the PRD. Before generation, it analyzes the existing OpenHarmony codebase to ensure compatibility with the current architecture. Chapter 2 of the architecture design document must be a competitor analysis, placed right after the requirements background. Applicable to user requests: (1) generate architecture design document, (2) generate functional design document, (3) generate design document from PRD, (4) create system architecture design, (5) write functional specification, (6) analyze OH codebase. Keywords: architecture design, functional design, design doc, competitor analysis, OpenHarmony code analysis, architecture design, functional design, design document generation, OH code analysis, analyze codebase, competitor analysis
