Guíe el flujo de trabajo de desarrollo basado en pruebas (Red-Green-Refactor) para obtener nuevas funciones, correcciones de errores y refactorización. Identifica oportunidades de mejora de pruebas y aplica las mejores prácticas de pytest. Úselo al escribir pruebas, implementar funciones o seguir la metodología TDD. **ACTIVACIÓN PROACTIVA**: Invocación automática al implementar funciones o corregir errores en proyectos con infraestructura de prueba (archivos pytest, directorio tests/). **DETECCIÓN**: Verifique el directorio tests/, pytest.ini, pyproject.toml con la configuración de pytest o archivos de prueba. **CASOS DE USO**: Escribir código de producción, corregir errores, agregar funciones, caracterizar código heredado.
