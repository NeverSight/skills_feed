Guidez le workflow de développement piloté par les tests (Red-Green-Refactor) pour les nouvelles fonctionnalités, les corrections de bogues et la refactorisation. Identifie les opportunités d'amélioration des tests et applique les meilleures pratiques pytest. À utiliser lors de l'écriture de tests, de la mise en œuvre de fonctionnalités ou du suivi de la méthodologie TDD. **ACTIVATION PROACTIVE** : appel automatique lors de l'implémentation de fonctionnalités ou de la correction de bugs dans des projets avec une infrastructure de test (fichiers pytest, répertoire tests/). **DÉTECTION** : recherchez le répertoire tests/, pytest.ini, pyproject.toml avec la configuration pytest ou les fichiers de test. **CAS D'UTILISATION** : Écriture de code de production, correction de bugs, ajout de fonctionnalités, caractérisation de code existant.
