Simplificación de código basada en TDD que preserva el comportamiento a través de pruebas. Utilice ciclos Red-Green-Refactor para simplificar el código, un cambio verificado por prueba a la vez. **DISTINTO DE**: Revisión general de código o reescritura de IA: esta habilidad requiere pruebas existentes y solo continúa cuando las pruebas confirman que se conserva el comportamiento. **PROACTIVO**: Invocación automática cuando el código cubierto por pruebas tiene complejidad (funciones >50 líneas, alta complejidad ciclomática, duplicación) y el usuario desea simplificarlo de forma segura. Frases desencadenantes: 'limpiar código', 'simplificar el código', 'reducir la complejidad', 'ayuda para refactorizar'. **NO PARA**: Agregar funciones o corregir errores; en su lugar, use la habilidad /tdd.
