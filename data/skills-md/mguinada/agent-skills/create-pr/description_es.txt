Crea solicitudes de extracción de GitHub con títulos con el formato adecuado. Úselo al crear relaciones públicas, enviar cambios para revisión o cuando el usuario diga /pr o solicite crear una solicitud de extracción. Analiza los cambios en la rama actual y utiliza la plantilla de solicitud de extracción de la carpeta .github. **ACTIVACIÓN PROACTIVA**: Invocación automática cuando existen confirmaciones en la rama de funciones y el usuario menciona PR, revisión o envío. **DETECCIÓN**: Verifique confirmaciones no enviadas, rama de funciones (no principal/maestra), el usuario solicita compartir/revisar cambios. **CASOS DE USO**: El código está listo para su revisión; después de completar el trabajo de funciones, el usuario desea compartir los cambios.
