Crea richieste pull GitHub con titoli formattati correttamente. Da utilizzare quando si creano PR, si inviano modifiche per la revisione o quando l'utente dice /pr o chiede di creare una richiesta pull. Analizza le modifiche sul ramo corrente e utilizza il modello di richiesta pull dalla cartella .github. **ATTIVAZIONE PROATTIVA**: richiamo automatico quando esistono commit sul ramo della funzionalità e l'utente menziona PR, revisione o push. **RILEVAMENTO**: verifica la presenza di commit non inviati, ramo funzionalità (non principale/master), l'utente chiede di condividere/revisionare le modifiche. **CASI D'USO**: il codice è pronto per la revisione, dopo aver completato il lavoro sulle funzionalità, l'utente desidera condividere le modifiche.
