Erstellt GitHub-Pull-Requests mit ordnungsgemäß formatierten Titeln. Verwenden Sie diese Option, wenn Sie PRs erstellen, Änderungen zur Überprüfung einreichen oder wenn der Benutzer /pr sagt oder darum bittet, eine Pull-Anfrage zu erstellen. Analysiert Änderungen am aktuellen Zweig und verwendet die Pull-Request-Vorlage aus dem .github-Ordner. **PROAKTIVE AKTIVIERUNG**: Automatischer Aufruf, wenn Commits im Feature-Branch vorhanden sind und der Benutzer PR, Review oder Push erwähnt. **ERKENNUNG**: Auf nicht gepushte Commits prüfen, Feature-Zweig (nicht Haupt-/Masterzweig), Benutzer bittet darum, Änderungen zu teilen/zu überprüfen. **ANWENDUNGSFÄLLE**: Der Code ist zur Überprüfung bereit. Nach Abschluss der Funktionsarbeiten möchte der Benutzer Änderungen teilen.
