適切にフォーマットされたタイトルを使用して GitHub プル リクエストを作成します。 PR を作成するとき、レビューのために変更を送信するとき、またはユーザーが /pr と言うか、プル リクエストの作成を要求するときに使用します。現在のブランチの変更を分析し、.github フォルダーのプル リクエスト テンプレートを使用します。 **プロアクティブ アクティベーション**: 機能ブランチにコミットが存在し、ユーザーが PR、レビュー、またはプッシュに言及したときに自動的に起動します。 **検出**: プッシュされていないコミット、機能ブランチ (メイン/マスターではない)、ユーザーが変更の共有/レビューを要求しているかどうかを確認します。 **使用例**: コードはレビューの準備ができており、機能の作業が完了した後、ユーザーは変更を共有したいと考えています。
