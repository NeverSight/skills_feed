Crée des requêtes pull GitHub avec des titres correctement formatés. À utiliser lors de la création de PR, de la soumission de modifications pour examen ou lorsque l'utilisateur dit /pr ou demande de créer une pull request. Analyse les modifications sur la branche actuelle et utilise le modèle de demande d'extraction du dossier .github. **ACTIVATION PROACTIVE** : appel automatique lorsque des validations existent sur la branche de fonctionnalités et que l'utilisateur mentionne PR, révision ou push. **DÉTECTION** : vérifiez les validations non poussées, la branche de fonctionnalités (pas principale/maître), l'utilisateur demande de partager/réviser les modifications. **CAS D'UTILISATION** : le code est prêt à être révisé. Après avoir terminé le travail sur les fonctionnalités, l'utilisateur souhaite partager les modifications.
