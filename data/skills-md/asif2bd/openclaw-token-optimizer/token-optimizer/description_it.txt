Riduci l'utilizzo dei token OpenClaw e i costi delle API attraverso il routing del modello intelligente, l'ottimizzazione dell'heartbeat, il monitoraggio del budget e le funzionalità 2026.2.15 native (eliminazione delle sessioni, limiti delle dimensioni del bootstrap, allineamento TTL della cache). Utilizzalo quando i costi dei token sono elevati, vengono raggiunti i limiti di velocità API o quando si ospitano più agenti su larga scala. I 4 script eseguibili (context_optimizer, model_router, heartbeat_optimizer, token_tracker) sono solo locali: nessuna richiesta di rete, nessuna chiamata di sottoprocesso, nessuna modifica di sistema. I file di riferimento (PROVIDERS.md, config-patches.json) documentano strategie multi-provider facoltative che richiedono chiavi API esterne e accesso alla rete se scegli di utilizzarle. Vedi SECURITY.md per un'analisi completa.
