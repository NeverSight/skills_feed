Árbol de decisiones de recuperación de datos de Next.js App Router centrado en TTFB-first y una experiencia de usuario honesta. Úselo al agregar o cambiar la lógica de recuperación de datos (búsqueda de RSC, almacenamiento en caché, SSG/ISR, transmisión de suspenso), cuando los metadatos o generar metadatos dependen de datos remotos, cuando se trata de datos de sesión/autenticación específicos del usuario, cuando se implementa la captación previa del servidor híbrido más hidratación del cliente (consulta TanStack) o cuando se revisan regresiones TTFB/LCP.
