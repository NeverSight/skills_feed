Esplora le codebase Python leggendo i file __init__.py prima di aprire altri file in un pacchetto. Mantieni ogni __init__.py come documentazione vivente: una docstring di livello superiore contenente una descrizione del pacchetto di una riga, un albero di file di file .py e sottodirectory e un breve scopo per ogni voce. Da utilizzare quando: (1) esplorare o navigare in una codebase Python, (2) creare nuovi pacchetti o moduli Python, (3) aggiungere, rimuovere o rinominare file all'interno di un pacchetto Python, (4) eseguire l'onboarding in un progetto Python sconosciuto.
