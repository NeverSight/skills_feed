패키지의 다른 파일을 열기 전에 __init__.py 파일을 읽어 Python 코드베이스를 탐색하세요. 모든 __init__.py를 살아있는 문서로 유지하십시오: 한 줄짜리 패키지 설명, .py 파일 및 하위 디렉터리의 파일 트리, 각 항목에 대한 간략한 목적을 포함하는 최상위 수준 문서 문자열입니다. (1) Python 코드베이스 탐색 또는 탐색, (2) 새로운 Python 패키지 또는 모듈 생성, (3) Python 패키지 내의 파일 추가, 제거 또는 이름 바꾸기, (4) 익숙하지 않은 Python 프로젝트에 온보딩할 때 사용합니다.
