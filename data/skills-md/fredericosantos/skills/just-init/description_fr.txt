Parcourez les bases de code Python en lisant les fichiers __init__.py avant d'ouvrir d'autres fichiers dans un package. Conservez chaque __init__.py comme une documentation vivante : une docstring de niveau supérieur contenant une description du package sur une seule ligne, une arborescence de fichiers et de sous-répertoires .py et un bref objectif pour chaque entrée. À utiliser lorsque : (1) l'exploration ou la navigation dans une base de code Python, (2) la création de nouveaux packages ou modules Python, (3) l'ajout, la suppression ou le renommage de fichiers dans un package Python, (4) l'intégration dans un projet Python inconnu.
