パッケージ内の他のファイルを開く前に、__init__.py ファイルを読んで Python コードベースをナビゲートします。すべての __init__.py を生きたドキュメントとして維持します。これは、1 行のパッケージ説明、.py ファイルとサブディレクトリのファイル ツリー、および各エントリの簡単な目的を含むトップレベルの docstring です。次の場合に使用します: (1) Python コードベースの探索またはナビゲート、(2) 新しい Python パッケージまたはモジュールの作成、(3) Python パッケージ内のファイルの追加、削除、または名前変更、(4) なじみのない Python プロジェクトへのオンボーディング。
