Debug sistematico in 4 fasi: comprendere l'errore, formulare ipotesi, testare una modifica alla volta, risolvere con sicurezza. Attivalo quando i test falliscono in modo imprevisto, si verificano errori, il comportamento è sbagliato o qualcosa che funzionava prima ora non funziona più. Si attiva su: "debug", "perché non funziona", "fallimento del test", "errore imprevisto", "bug", "rotto".
