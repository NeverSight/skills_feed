Evalúe si un ticket de desarrollo (historia de usuario, solicitud de función, informe de error, etc.) está listo para el desarrollo y proporcione comentarios específicos y procesables si no lo está. Utilice esta habilidad siempre que el usuario solicite clasificar, evaluar, valorar, revisar o verificar la preparación de un ticket, historia, problema o elemento de trabajo. El ticket puede provenir de cualquier lugar: pegado en línea, leído desde un archivo, obtenido de Jira u otro rastreador a través de MCP o cualquier otra fuente. Utilice también esto cuando un usuario pregunte "¿está listo este ticket?" o "¿qué le pasa a este billete?" o quiere mejorar las especificaciones de un ticket.
