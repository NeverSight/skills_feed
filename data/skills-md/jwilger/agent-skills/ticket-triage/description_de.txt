Bewerten Sie, ob ein Entwicklungsticket (User Story, Funktionsanfrage, Fehlerbericht usw.) für die Entwicklung bereit ist, und geben Sie konkretes, umsetzbares Feedback, wenn dies nicht der Fall ist. Verwenden Sie diese Fähigkeit immer dann, wenn der Benutzer ein Ticket, eine Story, ein Problem oder eine Arbeitsaufgabe selektieren, bewerten, bewerten, überprüfen oder die Bereitschaft prüfen möchte. Das Ticket kann von überall kommen: inline eingefügt, aus einer Datei gelesen, von Jira oder einem anderen Tracker über MCP oder einer anderen Quelle abgerufen. Verwenden Sie dies auch, wenn ein Benutzer fragt: „Ist dieses Ticket fertig?“ oder „Was stimmt mit diesem Ticket nicht?“ oder möchte die Spezifikation eines Tickets verbessern.
