Erstellen Sie hochwertige Git-Commits: Überprüfen/stufen Sie beabsichtigte Änderungen ein, teilen Sie sie in logische Commits auf und schreiben Sie klare Commit-Nachrichten (einschließlich konventioneller Commits). Verwenden Sie diese Option, wenn der Benutzer einen Commit durchführen, eine Commit-Nachricht verfassen, Änderungen durchführen oder die Arbeit in mehrere Commits aufteilen möchte.
