كتاب قواعد حيادي اللغة لإنتاج ومراجعة وتحسين التعليمات البرمجية الأنيقة. يُستخدم عند كتابة تعليمات برمجية جديدة، أو إعادة بناء التعليمات البرمجية الموجودة، أو مراجعة جودة التعليمات البرمجية، أو عندما يطلب المستخدم تعليمات برمجية "أنيقة"، أو "نظيفة"، أو "قابلة للصيانة"، أو "منظمة بشكل جيد". ينطبق على أي لغة برمجة أو إطار عمل.
