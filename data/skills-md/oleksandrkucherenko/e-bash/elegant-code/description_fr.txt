Livre de règles indépendant du langage pour produire, réviser et améliorer un code élégant. À utiliser lors de l'écriture d'un nouveau code, de la refactorisation du code existant, de la révision de la qualité du code ou lorsque l'utilisateur demande un code « élégant », « propre », « maintenable » ou « bien structuré ». S’applique à n’importe quel langage ou framework de programmation.
