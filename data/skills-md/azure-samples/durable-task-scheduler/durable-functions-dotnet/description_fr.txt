Créez des workflows durables et tolérants aux pannes à l’aide d’Azure Durable Functions avec un travailleur isolé .NET et un backend Durable Task Scheduler. À utiliser lors de la création d'orchestrations, d'activités, d'entités sans serveur ou de la mise en œuvre de modèles tels que le chaînage de fonctions, le fan-out/fan-in, les API HTTP asynchrones, l'interaction humaine, la surveillance ou les agrégateurs avec état. S'applique aux applications Azure Functions nécessitant une exécution durable, une persistance de l'état ou une coordination distribuée avec les API de gestion HTTP intégrées et l'intégration Azure.
