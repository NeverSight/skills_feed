Regole di sviluppo del plugin Bubble.io, riferimento API e standard di codifica. Da utilizzare quando si lavora su qualsiasi attività in questo repository: scrittura, revisione, refactoring o creazione di inizializzare.js, update.js, anteprima.js, header.html, azioni degli elementi, azioni lato client, azioni lato server (SSA), codice asincrono/await dell'API plug-in v4, JSDoc, file di configurazione, README, CHANGELOG, descrizioni del marketplace o descrizioni comandi dei campi. Utilizzabile anche per controlli di sicurezza, revisione del codice, debug e pubblicazione di plug-in. Copre istanze/proprietà/oggetti di contesto, interfacce BubbleThing/BubbleList, sospensione del caricamento dei dati, regole DOM/canvas, elementi e intestazioni condivise, stati esposti, gestione degli eventi, standard ESLint e limiti rigidi di Bubble.
