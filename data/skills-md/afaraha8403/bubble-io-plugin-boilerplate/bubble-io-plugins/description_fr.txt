Règles de développement du plugin Bubble.io, référence API et normes de codage. À utiliser lorsque vous travaillez sur n'importe quelle tâche de ce dépôt : écriture, révision, refactorisation ou création de initialize.js, update.js, preview.js, header.html, actions d'éléments, actions côté client, actions côté serveur (SSA), code asynchrone/attente de l'API du plug-in v4, JSDoc, fichiers d'installation, README, CHANGELOG, descriptions de marché ou info-bulles de terrain. Utilisez également pour les audits de sécurité, la révision du code, le débogage et la publication de plugins. Couvre les objets instance/propriétés/contexte, les interfaces BubbleThing/BubbleList, la suspension du chargement des données, les règles DOM/canvas, les en-têtes élément/partagé, les états exposés, la gestion des événements, les normes ESLint et les limites strictes de Bubble.
