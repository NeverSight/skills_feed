Esta habilidad es el "motor de levantamiento pesado" para la resolución de problemas complejos.
**LÓGICA DE ACTIVACIÓN CRÍTICA: LEA DETENIDAMENTE:** El IDE debe evaluar la descripción de la solicitud del usuario para decidir si invoca esta habilidad.
**INVOCA ESTA HABILIDAD SI:** - La solicitud implica **Ambigüedad** (p. ej., "Diseñar un sistema...", "Analizar esto...", "En qué piensas..."). - La solicitud requiere **Razonamiento** o **Planificación** antes de la codificación (por ejemplo, diseño de arquitectura, análisis de compensaciones, análisis de causa raíz). - La solicitud implica **Múltiples perspectivas** (por ejemplo, "debate", "crítica", "revisión integral"). - La solicitud solicita explícitamente **Estrategias cognitivas** (cuestionamiento socrático, abogado del diablo, premortem, ingeniería inversa).
**NO INVOCAR SI:** - La solicitud es una acción simple y determinista (por ejemplo, "Corregir el error tipográfico en la línea 10", "Leer el archivo X", "Ejecutar el comando Y"). - La petición es una generación directa de código para una función bien definida sin necesidad de pensamiento arquitectónico.
Cuando se invoca, esta habilidad desencadena un proceso cognitivo de múltiples agentes para manejar la complejidad.
