Этот навык является «тяжеловесным двигателем» для решения сложных проблем.
**КРИТИЧЕСКАЯ ЛОГИКА ТРИГГЕРА — ПРОЧИТАЙТЕ ВНИМАТЕЛЬНО:** Среда IDE должна оценить описание запроса пользователя, чтобы решить, следует ли активировать этот навык.
**ВЫЗВАЙТЕ ЭТОТ НАВЫК, ЕСЛИ:** – Запрос предполагает **Неоднозначность** (например, «Разработайте систему...», «Проанализируйте это...», «Что вы думаете о...»). – Запрос требует **Обоснования** или **Планирования** перед кодированием (например, проектирование архитектуры, анализ компромиссов, анализ первопричин). – Запрос подразумевает **множественные точки зрения** (например, «дебаты», «критика», «всесторонний обзор»). - В запросе явно говорится о **когнитивных стратегиях** (сократовский допрос, адвокат дьявола, предварительное исследование, обратное проектирование).
**НЕ ВЫЗЫВАТЬ ЕСЛИ:** — запрос представляет собой простое детерминированное действие (например, «Исправить опечатку в строке 10», «Прочитать файл X», «Выполнить команду Y»). — Запрос представляет собой прямую генерацию кода для четко определенной функции без необходимости архитектурной мысли.
При вызове этот навык запускает мультиагентный когнитивный процесс, позволяющий справиться со сложностью.
