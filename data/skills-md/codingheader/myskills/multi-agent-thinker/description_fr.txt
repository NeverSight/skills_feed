Cette compétence est le « moteur de levage lourd » pour la résolution de problèmes complexes.
**LOGIQUE DE DÉCLENCHEMENT CRITIQUE - À LIRE ATTENTIVEMENT :** L'EDI doit évaluer la description de la demande de l'utilisateur pour décider s'il doit invoquer cette compétence.
**INVOQUEZ CETTE COMPÉTENCE SI :** - La demande implique une **ambiguïté** (par exemple, "Concevoir un système...", "Analyser ceci...", "À quoi pensez-vous..."). - La demande nécessite un **raisonnement** ou une **planification** avant le codage (par exemple, conception de l'architecture, analyse des compromis, analyse des causes profondes). - La demande implique des **Perspectives multiples** (par exemple, « débat », « critique », « examen complet »). - La demande demande explicitement des **Stratégies cognitives** (questionnement socratique, Devil's Advocate, Pre-mortem, Reverse Engineering).
**NE PAS INVOQUER SI :** - La requête est une action simple et déterministe (par exemple, "Corriger la faute de frappe à la ligne 10", "Lire le fichier X", "Exécuter la commande Y"). - La requête est une génération directe de code pour une fonction bien définie sans nécessiter de réflexion architecturale.
Lorsqu'elle est invoquée, cette compétence déclenche un processus cognitif multi-agents pour gérer la complexité.
