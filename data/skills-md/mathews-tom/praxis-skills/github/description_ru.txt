Операции CLI GitHub через `gh` для проблем, запросов на включение, CI/действий, выпусков, репозиториев, поиска, сути и API REST/GraphQL. Структурированный вывод с `--json` и `--jq` для анализа. Охватывает `gh Issue create/list/view/edit/close`, `gh pr create/review/merge/checks`, `gh run list/view/rerun/watch`, `gh Release create`, `gh search repos/issues/prs/code`, `gh api` для запросов REST и GraphQL и операции `gh gist`. Включает обработку ошибок для HTTP 401/403/404/422/429, устранение неполадок области и управление ограничением скорости. Триггерные фразы: «создать проблему», «сообщить об ошибке», «открыть заявку», «отправить PR», «поднять запрос на включение», «проверить конвейер», «просмотреть результаты тестирования», «сбой CI», «почему CI не удалось», «проверить статус CI», «объединить PR», «управлять выпусками», «запросить GitHub API», «поиск в репозиториях», «сортировка рабочих процессов», «автоматизировать операции GitHub». Также срабатывает, когда пользователь вставляет URL-адрес GitHub.
