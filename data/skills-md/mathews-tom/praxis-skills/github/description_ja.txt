問題、プル リクエスト、CI/アクション、リリース、リポジトリ、検索、要点、および REST/GraphQL API に対する「gh」を介した GitHub CLI 操作。解析用の `--json` と `--jq` を使用した構造化された出力。 `gh issue create/list/view/edit/close`、`gh pr create/review/merge/checks`、`gh run list/view/rerun/watch`、`gh release create`、`gh search repos/issues/prs/code`、REST および GraphQL クエリの `gh api`、および `gh gist` 操作をカバーします。 HTTP 401/403/404/422/429 のエラー処理、スコープのトラブルシューティング、レート制限管理が含まれます。トリガーフレーズ: 「問題を作成する」、「バグを報告する」、「チケットを開く」、「PR を送信する」、「プルリクエストを発行する」、「パイプラインを確認する」、「テスト結果を表示する」、「CI が失敗した」、「CI が失敗した理由」、「CI ステータスを確認する」、「PR をマージする」、「リリースを管理する」、「GitHub API をクエリする」、「リポジトリを検索する」、「ワークフローをトリアージする」、「GitHub を自動化する」作戦」。ユーザーが GitHub URL を貼り付けたときにもトリガーされます。
