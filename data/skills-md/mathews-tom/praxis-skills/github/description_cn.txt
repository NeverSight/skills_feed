通过“gh”进行 GitHub CLI 操作，用于问题、拉取请求、CI/操作、发布、存储库、搜索、要点和 REST/GraphQL API。使用“--json”和“--jq”进行结构化输出进行解析。涵盖“gh issues create/list/view/edit/close”、“gh pr create/review/merge/checks”、“gh run list/view/rerun/watch”、“gh release create”、“gh search repos/issues/prs/code”、“gh api”（用于 REST 和 GraphQL 查询）以及“gh gist”操作。包括 HTTP 401/403/404/422/429 的错误处理、范围故障排除和速率限制管理。触发短语：“创建问题”、“提交错误”、“开票”、“提交 PR”、“提出拉取请求”、“检查管道”、“查看测试结果”、“CI 失败”、“CI 为何失败”、“检查 CI 状态”、“合并 PR”、“管理版本”、“查询 GitHub API”、“搜索存储库”、“分类工作流程”、“自动化 GitHub 操作”。当用户粘贴 GitHub URL 时也会触发。
