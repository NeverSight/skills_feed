MCP(모델 컨텍스트 프로토콜) 서버를 주문형 기술로 변환하여 활성 컨텍스트 창 사용량을 대폭 줄입니다. MCP 도구 스키마는 사용 여부에 관계없이 매 턴마다 토큰을 소비합니다. 스킬은 필요할 때만 로드됩니다. MCP 도구 정의를 분석하고 교체 전략에 따라 각 도구를 분류하며 완전한 기술 패키지를 생성합니다. 사용자가 MCP, MCP를 스킬로 변환, 컨텍스트 크기 감소, 도구가 너무 많음, 도구 토큰 팽창, MCP 축소, 컨텍스트 창이 가득 참, MCP 교체, MCP 마이그레이션을 언급할 때마다 이 스킬을 사용하십시오. 또한 사용자가 활성 MCP 서버를 많이 갖고 있고 컨텍스트 사용을 최적화하려고 할 때, 도구 정의에서 토큰 오버헤드를 줄이는 것에 대해 묻거나 MCP가 활성화될 때 내 컨텍스트가 너무 크거나 컨텍스트가 부족하다고 말할 때 트리거됩니다.
