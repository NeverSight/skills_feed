MCP (Model Context Protocol) サーバーをオンデマンド スキルに変換して、アクティブなコンテキスト ウィンドウの使用量を大幅に削減します。 MCP ツール スキーマは、使用するかどうかに関係なく、毎ターントークンを消費します。スキルは必要な場合にのみロードされます。 MCP ツール定義を分析し、置換戦略によって各ツールを分類し、完全なスキル パッケージを生成します。ユーザーが、MCP の変換、MCP をスキルに変換、コンテキスト サイズの削減、ツールが多すぎる、ツール トークンの肥大化、MCP のスリム化、コンテキスト ウィンドウがいっぱいになった、MCP の置き換え、MCP の移行に言及した場合は常にこのスキルを使用します。また、ユーザーが多数のアクティブな MCP サーバーを持っており、コンテキストの使用を最適化したい場合、ツール定義からのトークン オーバーヘッドの削減について尋ねる場合、または MCP がアクティブなときにコンテキストが大きすぎるかコンテキストが不足していると言った場合にもトリガーされます。
